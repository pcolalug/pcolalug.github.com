666  Club Stuff / Articles / Some useful mySQL commands on: August 25, 2002, 04:41:24 pm
If you have mySQL installed you can start using right away from command line goto your terminal window and type "mysql" this should bring up the prompt mysql, from here you can use the commands to start using the database.

Here are a few of the useful commands
Note! you must always have the ";" after each line.

show database; >should show you any databases that already exist. On a clean install of mySQL it loads a test database called test for you.
create database "name";>will create a database with the name given.
use "name";>set to use the database named.
drop database "name";>will delete the named database.
show tables;>will show you the tables in whatever database you are in.
describe db;>will give you a breakdown description of the tables.

These are all very useful commands if you donnot already have mySQL I would suggest you get it as soon as possible. It is a great enterprise level database and has several GUI based tools on there site also to work with the database. http://www.mysql.org.

667  Distributions / RPM Based Distributions / ext2 to ext3 migration on: August 25, 2002, 04:24:53 pm
from Joshua Jensen @ Redhat

1) Create the ext3 journal on the ext2 filesystem
 -tune2fs -j /dev/ <ext2_filesystem_partition>

2) Change ext2 refrences in /etc/fstab to ext3
 -sed s/ext2/ext3/g /etc/fstab | less> /etc/fstab

3) initrd should contain ext3 kernel modules
 -mkinitrd /boot/initrd-<version>.img<version>
 -bootloader should be configures to use the initrd

4) reboot and verify with "cat /proc/mounts"

More information from Linux System Admin. by Joshua Jensen

668  Distributions / RPM Based Distributions / Package management on: August 25, 2002, 04:12:34 pm
From Joshua Jensen @ redhat
A package is made up of
 *Files
 -/etc/rmt,/sbin/rmt
 *Dependecies
 -libtermcap.so.2
 *Install/Uninstall Scripts
 *Changelogs
 *General PAckage Information

Package benefits
 *Dependecy Checking
 -Upgading
 -Removing
 -Installing
 *Saves time
 *Security

Package commands
 *Install
 -/bin/rpm -i [options] <package name>
 Example-/bin/rpm -ivh tar-1.13.11-1.i386.rpm
 -Network support FTP and HTTP
 -/bin/rpm -ivh http://ftp://ftp.redhat.com/path/*ldap*rpm
 *Erase
 -/bin/rpm -e[options] <package name>
 Example-/bin/rpm -e [options] <package name>
 *Upgrade
 -U, upgrade -F, freshen
 -/bin/rpm <-U|-F> [options] <package file>
 -Upgrade erase old (if there), install new
 -Freshen  Upgrade only if old package present, else do nothing
 *Query
 -/bin/rpm -q [options] <package name>
 -/bin/rpm -qp [options] <package file>
 -File listing -l, package owner -f, General information -i
 -/bin/rpm --querytags
 -/bin/rpm --qf <format> -q <package name>
 -Query for what is not installed
 1) install rpmdb-redhat
 2) rpm -ivh rpmdb-redhat*rpm
 3) rpm -q --redhatprovides libssl.so.2openssl-0.9.6b-8
 *Verify
 -/bin/rpm -V [options] <package name>
 -/bin/rpm -Vp /mnt/cdrom/packagefile
 -Verify package sig.
 -#gpg --import /usr/share/rhn/RPM-GPG-KEY
 -#rpm -K [options] <package file>
 *Build
 -install SRPM sources in /usr/src/redhat
  ./build/ ./RPMS/ ./SOURCES/ ./SPECS/ ./SRPMS/
 -/bin/rpm ib <-a|-p|-c|-i|-b> <spec file>

Hope this helps a little. These are excerpts from my trip to San Diego.

669  Distributions / RPM Based Distributions / GRUB (boot loader) on: August 25, 2002, 03:32:47 pm
From Joshua Jensen @ redhat
Grub The grand unified bootloader
Config file- /boot/grub/grub.conf
grub, unlike other loaders
  * reads files directly from FFS, FAT, ext2, ext3, riser, JFS and XFS
  * Uses salted MD5 passwords for access
  * Can boot from a network-retrived image file
  * Has it's own interactive command-line

Note be sure to set password for GRUB

Grub commands
1) grub provides interactive prompt that allows full grub command line.
    (Can be used to see arbitrary files)
2) grub-install <device> first stage installer.
3) grub-md5-crypt creates MD5 hashed passwords.

Hope this helps you get started using GRUB
