Well, I downloaded RedHat 8.0 (psyche) last night and got it installed this morning.  It's 5 cd's (although my default workstation install only used the first two).

I must say, RedHat has taken GNOME to the next level.  This is nice.  Gone are the days of 5 different printer configuration programs and cumbersome menus.  This bad-boy is streamlined.  And works great!

Installation was a breeze, all hardware was detected perfectly.  I chose to do a default workstation install (which gives you Gnome, Evolution, Open Office, Mozilla, games, development tools, configuration utilities).

I ran into a problem when I first tried to login to my system, it would display the Gnome logo and lockup.  I'm not sure what the problem was, but I went to the console, installed my latest NVidia drivers and it still locked up.  So this time before I hit "ENTER" on my password I manually selected "American English" and "Gnome" from the options, then logged in, and it worked fine.  

So far I'm very impressed with this system.  The custom fonts provided by redhat give gnome a more pollished and professional look.  Much better (IMHO) than KDE on Mandrake.  Also Mozilla is working beautifully, no crashes, no mis-rendered pages.

I wanted to grab a screenshot to show everyone but GIMP is having trouble doing a screen-cap, so I'm gonna install another utility.  I'll post it here once I have it.

I've gotta say, RedHat 8 is definitely my new distro of choice.  Give it a try, I'm sure you'll like it.

Well RedHat has obviously customized it a bit, but generally the menu has always been fairly well organized.  But it was definitely cleaned up and fine-tuned for this release.

Quote from: dpeach
I agree with imnes about the compatability of the two desktops. I am all for two different environments. But why not share resources so that an app that works in one GUI can work in the other (without having to load half of the code from the other environment to make it work)?

The problem is that they are two different *application environments*.  Completely different.  The GUI stuff is just widgets, and it is relatively trivial to port a well-written app from one environment's GUI to the other. You still need, however, all of the libraries that make it work on a functional level.  It is these libraries, I guess, that are at issue.  While KDE uses the Qt libraries, Gnome uses the GTK libraries.  C++ vs. C, and such.  The very modularity of the OS is a blessing and a curse, I guess.  The alternative would to be to write a widgetless, platform agnostic, "desktop services" layer that any environment could use (this would include your sound server, as well as window management and drawing primatives).  Then again, this is called "XFree86" and a "window manager", and IMHO, it should be replaced.

Ultimately I'd like it see the window manager code abstraced from the system code.  Someone mentioned it above.  That way if you want to code in the KDE/QT style that's fine, and if you code in Gnome/GTK that's fine, but under the hood the library calls would be converted to some more generic library that handles actually calling the x server and performing functions.
