PHP5 will include SQLite, an embedded database.  I think this is great, because now I don't have to worry about if the person who will use my scripts is running MySQL or PostGres or Oracle or DB2, I can just use the built-in one.

It has the MS-Access like feature of containing your entire database within a single file on your hard drive, so it's easy to move around, no need to dump data and write creation/insertion scripts.

Some of the features listed on the site:

    * Implements most of SQL92.
    * A complete database (with multiple tables and indices) is stored in a single disk file.
    * ACID (Atomic, Consistent, Isolated, Durable) transactions.
    * Database files can be freely shared between machines with different byte orders.
    * Supports databases up to 2 terabytes (2^41 bytes) in size.
    * Small memory footprint: less than 25K lines of C code.
    * Two times faster than PostgreSQL and MySQL for many common operations.
    * Very simple C/C++ interface requires the use of only three functions and one opaque structure.
    * TCL bindings included. Bindings for many other languages available separately.
    * Simple, well-commented source code.
    * Automated test suite provides over 90% code coverage.
    * Self-contained: no external dependencies.
    * Built and tested under Linux and Windows.
    * Sources are in the public domain. Use for any purpose.

A really neat feature, which makes perfect sense for web developers, is the database is typeless...

From the faq
Quote

What datatypes does SQLite support?

    SQLite is typeless. All data is stored as null-terminated strings. The datatype information that follows the column name in CREATE TABLE statements is ignored (mostly). You can put any type of data you want into any column, without regard to the declared datatype of that column.
...


Of course, this might take a little while to get used to...
