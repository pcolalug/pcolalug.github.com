Title: Dual-boot vs. Virtualization
Post by: geekforGod on May 16, 2006, 11:02:01 am
I was reading the other topic where someone suggested dual-booting a Linux newcomers machine to be able to use iTunes and I was wondering what you guys think. Is it easier to dual-boot or to install Linux with a VMWare Server image of Windows?

Dual-booting is certainly easier to install and configure on any new Linux distribution. Dual-booting gives each OS full control over the hardware at the time and so there are no performance issues. Dual-booting is a pain just for one app though, especially an application you may want running much of the time, since you'll never use Linux that way.

Virtualization allows you to run the Windows apps you want while still in Linux. If the Windows image gets infested with spyware, you just install a new image. I know sometimes Windows can be a pain to make sure it doesn't overwrite your Linux install when you reinstall it. Virtualization would take away from performance, but if the machine is relatively new and all you're running is desktop applications, then I don't think it would be that bad. I don't think I would use it for games.

Anyways, just thought I would throw that out there and see what you guys think.

Todd

Title: Re: Dual-boot vs. Virtualization
Post by: rugmonster on May 16, 2006, 01:23:47 pm
I suppose this could be acceptable as a solution. The only downside is that running a virtualized operating system is resource intensive, especially if all you are wanting to do is run a media player. If you have the memory to do it, then okay. I just don't like to waste that much on a sluggish system. As you said, if you want to run any games, virtualization is not an option.

Even if you reinstall Windows, all it does is overwrite the MBR and set the partition Windows is being loaded to as active. To get booted back into Linux, you just need a rescue disk to reset the boot flag on the bootable drive. Not such a big deal.

I think virtualization has its purposes, but unless you have 2gigs of memory and a good processor, it's going to cause a serious hit on your systems performance. I'll leave virtualization to workstations and servers for testing and server consolidation.
