Title: Re: Ubuntu 7.04 (fiesty fawn) Released
Post by: rugmonster on April 19, 2007, 05:10:46 pm
I've been using pre-release Feisty for the past month or so at work and I like it. One of the most impressive features is the upgrade tools they have come out with for Edgy to Feisty upgrades. I did one system at work with dist-upgrade and another with do-release-upgrade. Where I had to do some tweaking following the dist-upgrade, do-release-upgrade was absolutely seemless.

Note to those thinking of upgrading. You may want to wait a bit. I'm running the upgrade right now on my desktop and I'm getting very slow package downloads. I'm averaging about 20KB/s, which translates to almost 12 hours to download the new packages for my system. The server installs were considerably less, but those have far fewer packages than a desktop install.

Oddly enough, the CD image download I have going is clipping right along at about 500KB/s. I guess the package repos are getting hit harder.

Post by: rugmonster on April 21, 2007, 07:47:53 am
Further notes on Feisty upgrade.

I have an AD, internal DNS domain, rugmonster.local, that I use for authentication. Since Avahi uses a .local namespace, I could not resolve any of my internal machine's names by FQDN due to Avahi's mdns being checked prior to dns in nsswitch.conf. To fix the issue (and since I don't use Avahi for crap), I changed my nsswitch.conf to the following:

hosts:          files dns mdns4_minimal [NOTFOUND=return] mdns4

The impact of all of that went beyond not being able to resolve names properly via DNS. I could not authenticate against the AD on my Ubuntu machine because Kerberos could not talk to the KDC since it depends on DNS for name resolution.

So if you're having problems where you can resolve a host by hostname alone, but not by FQDN and have a domain with .local as the TLD, then you may want to try this.

Other than that, it's been smooth sailing. The home file server is getting the treatment now, followed by the business's web server.
