I have no real firm answer, but here goes.

I am running MPlayer on my Mandrake 10 box with KDE 3.2. It works for almost everything. On their site they have the codecs and tell you how to install them. Title: Okay, win32 codecs
Post by: imnes on October 27, 2004, 12:10:36 pm
Find out if your player (kaboodle?) is using xine on the back-end.  If not - switch to something that does (like kaffeine as mentioned).

You'll want to grab
http://www.mplayerhq.hu/MPlayer/releases/codecs/all-20040916.tar.bz2

Extract it, move the contents to /usr/lib/win32 I believe.

Then any xine player should be able to handle pretty much any video format.  

Good luck!

Post by: dpeach on October 29, 2004, 09:05:31 am
Yes, the MPlayer essentials package is the list of codecs you need for normal stuff. Then there is another set of codecs for more obscure things.

Look in the MPlayer documentation and see where you are supposed to put those. You may need to dig into a config file in your installed MPlayer to see where it is really looking though. I remember having issues with my original install when I wanted to put more codecs in. I found out MPlayer was not looking in the default directory to find the codecs. And for some reason I still run Mandrake. :twisted:

I don't remember how I found where it was looking. But I found it once, so I am sure you can. If it does not seem to be working right after you put the codecs where they are supposed to be, then check your MPlayer configuration.

Someone may be able to enlighten us here. Can he use the codecs that are making the other programs work for MPlayer? I would think that a codec is the same for all players, but that is my guess, not based on any fact.
