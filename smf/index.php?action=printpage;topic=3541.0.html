Title: Creating strong passwords
Post by: dpeach on June 19, 2009, 07:22:35 am
I know many of you probably already have a formula for creating strong passwords, but here is one I learned a few years ago. I am going to teach this method to a group of people next week who are not necessarily computer savvy. I think this is a good, simplistic way to create unique passwords that are also relatively strong.

Particularly I use this on websites. This is not exactly the way I do mine, but here is the basic concept. We are going to create a 10 character password with this. You can make it longer, but I would not make it shorter. The shortest password you should have would be 8 characters for on-line stuff.

  • Take the first 5 letters in the website. Let's use Facebook.com as an example. faceb
  • Now take a 5 letter word that you will always use to do this. water
  • Upper case and munge the special word. W4T3R
  • Now start with the website name and put the 2 words together like so: fWa4cTe3bR

Now you have a unique password for that site that if someone hacks will not necessarily give them access to everything else in your life like they would have if you used the same password everywhere. You don't really have to remember the password since you can re-create it every time. If a site has fewer than 5 letters in its name you can add what comes after the ".". For example, if a site were dog.net you could do your hash with dogne.

This is just the basics. You can build from this to make other seemingly random passwords for different security levels. If you are creating a password for a program instead of a website you can use the name of the program in your hash.

Do you have any other suggestions for creating unique passwords?
