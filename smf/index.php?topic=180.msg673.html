I definitely agree with Crowe here.  I used to use Slackware religiously (up until version 4.0).  Then I switched to Redhat.  I was completely lost, things were setup differently, a lot of the things I was used to manually setting up now had to be done via redhat's Setup program.

After using RedHat for a while, I find myself completely lost in slackware.  But I have come to really like the convenience RPM gives you (although it is frightening to most users who are first confronted with it).

Using Debian was completely different than slackware or RedHat.  apt is powerful, and shares a lot of features with RPM, but wasn't the same to use.  

Then FreeBSD, it's sort of a take on the old way of doing things.  You switch to /usr/ports, find the program you want to install (such as networking/gaim or whatever).  In the directory you type "make install", it downloads the latest sources, compiles and installs it.  I think if you do a "make uninstall" it will remove it.  Also if you're missing any dependencies it will put that build on hold, go get the missing software, then continue.  Pretty nice.  Not really a "package manager", but solves the problem in it's own way.

I guess the reason I use RedHat / Mandrake primarily is that it's what I"m used to and feel most comfortable with.  A friend of mine uses Slackware, and won't even consider switching to RH/MDK (although I nag him about it every chance I get).  

RedHat and MDK did make life a little difficult for us lately though.  They've both recently reorganized their distributions, switched up / changed configuration programs, locations.  So it's become a bit of a learning curve lately, but not bad.

I guess it's a move to make it more friendly to the average user, but RH (at least in Gnome) has taken the terminal program off thte quick-launch toolbar and burried it in the "System Tools" menu.
