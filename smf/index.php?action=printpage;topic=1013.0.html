Title: lean slack old 66mhz
Post by: imnes on September 19, 2003, 12:07:34 pm
Firebird probably won't run because it may have been compiled for a 586 or 686 CPU.  If you have a 486 then that could be it.

You might want to run an older version of slackware, maybe slackware 8 or older.  I think XFree86 version 3.x runs better on old hardware then XFree86 4 does.

Also you could recompile your kernel, optimize it for your CPU, remove any and all unnecessary parts (for example if you don't use SCSI remove it, if you don't use a sound card, remove all that).  Dunno if that will offer you any performance boost but the kernel would use less ram.

Nick

Post by: rugmonster on September 20, 2003, 11:13:46 am
You have to remember that most of the software and most of the distributions are being compiled for Intel 586 and 686. You might want to try a distribution like Debian since it is still compiled for 386 compatibility. Firebird is also one of those binaries compiled for 686, but you can get the source and compile your own.

I would suggest you go with a bare bones Debian with Fluxbox, Blackbox or XFCE4 with Firebird and Thunderbird or another lightweight email client.

Basically, you aren't going to get much out of that box. I wouldn't run X on it if it were mine. Command line is much faster and probably the way to go with that thing...but nothing says you can't do it the other way.
