First, to get your rig back, use your rescue disk that you made when you installed MD (you *did* make a rescue disk, didn't you?). If you didn't make a rescue disk, you should be able to boot from the cd and at the prompt (before the purdy grafix start) type
Code:
linux rescue
Just follow the instructions on the loverly ncurses screens. BTW, I am assuming that Mandrake operates this way, similar to RH.
Anyway, it should tell you that the installed filesystem is mounted under /mnt/sysvol or some such. Type the following
Code:
cd /mnt/sysvol/etc/X11
Next, type
Code:
vi XF86Config
Now, find the line that says
Code:
Load glx
and comment it out
Code:
#Load glx
Next change the line that says
Code:
Driver nvidia
back to
Code:
Driver nv
and save the file. You should now be able to exit and restart the machine without the cd.

Now, here's my tried and true meathod of installing NVidia drivers. First, download the source tarballs, not the rpms. Now, cd to where you dl'd the tars and type the following
Code:
tar -zxvf nvidia-kernel...
where the ... indicates the rest of the filename (in bash, you should be able to type the "NVidia-ke" part and just hit tab to autocomplete). cd into the new Nvidia-kernel... directory and type (as root)
Code:
make
You should see a line about tainting your kernel, but hey, that's the price we pay for hardware accellerated 3D. Now, repeat the above steps for the NVidia-GLX tarball. Once all that is done, su to root and enter the following
Code:
cd /etc/X11
cp XF86Config XF86Config.working
We do this so that if it doesn't work, we can just overwrite the broken config with one that works. Now,
Code:
vi XF86Config
and uncomment the line saying
Code:
Load glx
and change the line that says
Code:
Driver nv
back to
Code:
Driver nvidia
and save the file. Finally, type
Code:
reboot
This should do it. You know you've won when the NVidia logo flashed briefly before the grafical login. If you have any more problems, pm or email me and we can discuss it in depth.

Success!!!

   I have gotten the Nvidia drivers to work, although the problem was not what I had thought...  After trying again on a fresh RH8 install, using the source tarball method suggested by rburt3, I was exceedingly disappointed to see the EXACT same results again....  This time however, I looked at the logfile output for some clues....  in there, I saw messages saying things like "cannot find an IRQ for your nvidia card!", and "check BIOS - plug & play should be set to NO", and "check BIOS - set 'allocate IRQ to VGA to YES'"....  so, I made those changes in the BIOS, and on the ensuing reboot, it worked!  

   So, my video looks MUCH nicer now, and have played TuxRacer for the first time.  Next step is to get RTCW installed, and to fine-tune video settings some more.  Anyway, I'm very happy to have it going, after being very close to giving up completely.  THANK-YOU to all who helped me get through this one!!!

Hi,
Yes, I've gotten RTCW to work, and it was fairly easy.  Here's a summary of what to do:
1.  Download the Linux main installer (current ver is:  wolf-linux-1.4-full.x86.run), and the update patch (wolf-linux-update-1.41.x86.run).  They are about 70Mb and 5Mb respectively.
2.  Run these two to install the game and patch.  Default location is /usr/local/games/wolfenstein.
3.  Next, you have to copy some of the map paks from an installed Windows version (from the /main directory under RTCW directory).  Filenames are:  mp_pak0.pk3, mp_pak1.pk3, mp_pak2.pk3, pak0.pk3, sp_pak1.pk3, sp_pak2.pk3.  Total for these is about 650Mb...  They get copied into the /main directory under the wolfenstein directory.
4.  Assuming you allowed the installer to create shortcuts in the menu structure, you should now be able to start up the game.  Next things to do are to configure your controls, video, etc...  As a reminder, the game requires GL accelerated video.

As an aside, I also have a working RTCW server on another machine.  That setup is somewhat more involved, and if anyone would like some help with that, I'm available.

I just found this(Gentoo) also. Make sure you make a copy of your /etc/X11/XF86Config-4 and then add "DefaultFbBpp 32" under your "DefaultDepth 24" in the Screen Section. It should look something similar to this
Code:
   Section "Screen"
      Identifier  "Screen"
      Device      "nVidia GeForce 256 DDR"
      Monitor     "Vision Master Pro 17"
      Option      "NvAgp" "2"

      DefaultDepth 24
      DefaultFbBpp 32

      Subsection "Display"
        Depth  16
        Modes  "1152x864" "1024x768" "800x600" "640x480"
      EndSubsection

      Subsection "Display"
        Depth  24
        Modes  "1152x864" "1024x768" "800x600" "640x480"
      EndSubsection
    EndSection


I may have spoken too soon earlier. However, if this doesn't work, copy your known good XF86Config-4 back over so your X isn't completely hosed beyond repair. These forums probably aren't too nice to navigate in lynx.

*above sample XF86Config-4 from Gentoo documentation.
