Post by: Brent on April 22, 2005, 08:00:36 pm
Actually, If you don't have it you should just visit the site that it gave you in ./configure output.

Four simple steps to compiling software:

tar -xzf foo.tar.gz or tar -xjf bar.tar.bz2
./configure (setting up compile options)
make (actual compiling)
make install (putting everything in it's nessesary place)

Of course from time to time you run across packages that may need other packages to build correctly, but fortunately this is rare in slackware.

I took a look at the wx packages on linuxpackages.net, there are several version of them.   This one looks closest to what you need, so download and install it http://www.linuxpackages.net/download.php?id=6231

It still won't work correctly because the library your amule package wants is called libwx_gtk2_xrc-2.5 and the one in the package is slightly different: libwx_gtk2u_xrc-2.5.  But this will be the right version (2.5).

I have a feeling making a link to the expected filename will work with this, so after installing that package, type these commands as root:

Code:
cd /usr/lib
ln -s libwx_gtk2u_xrc-2.5.so.4.0.0 libwx_gtk2_xrc-2.5.so.3
ldconfig


That will create a link to the library you installed with the name that amule is expecting.  

Now install your amule package and try to run it.  The above steps *may* need to be done again for any of the other wx libraries that are named wrong (look in /usr/lib/libwx_*u* for other possibilities) if amule is looking for more.
