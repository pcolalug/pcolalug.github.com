Title: Re: Learn 10 good UNIX usage habits
Post by: rugmonster on December 18, 2006, 02:37:40 pm
grep is used to find a pattern(s) in one or more files.

English translation, lets you search for a word, number, or combination of numbers, letters and characters.

So say you want to see what your home directory is in /etc/passwd without opening the whole file, you can quickly grep it.

Code:
$ grep "givensdj" /etc/passwd
givensdj:blah blah blah:/home/givensdj

Often times, if I know there is a configuration option in a file, but not entirely sure which file it's in, I'll go to the directory that holds all of the configuration files and run a grep, like so...

Code:
$ grep -R "configure_option" /etc

If the pattern is found in a file, it will tell me which file it is and then I can do what I need to do with it. By the way, the -R flag tells it to search recursively through all subdirectories in /etc.

Another thing I find myself using it for a lot is finding where a function is defined in source code, or where a CSS class or id is defined. grep, cat, sed, awk, and find are my most used "advanced" command line tools above all others. There is so much you can do with them, especially when you string them together. A lot of the stuff you might use perl for can be done with these commands.
