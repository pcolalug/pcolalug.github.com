Ed,

Here are a couple of ideas that might help in case you haven't already tried them:

1. Check the lts.conf file located in /opt/ltsp/i386/etc directory.  There should be an entry as follows:
SOUND = Y

2.  Try one of your own machines as a client.  With my own pc as a client, the sound worked with Edbuntu.  I haven't tried that computer with K12LTSP.  The P166 client I'm currently using with K12LTS doesn't have a sound card.  Perhaps there is some incompatibility issue with these Camp Fire sound cards

3. You have probably already tried this, but if you haven't, go into System > Administration > Soundcard Detection and follow the instructions.

I am happy to report I finally got sound working on the Campfire client machine with K12LTSP. Google is your friend. After much googling I found a post that said to make this change in
/opt/ltsp/i386/etc/rc.sound go to line 67 which looks like this
/bin/esd -nobeeps -tcp -port 16001 -as 1 &

and make it look like this
/bin/esd -nobeeps -tcp -port 16001 -as 1 -public &

Using the ESD sound server sound now works for me on the client. There is still much work to be done. Right Dennis? I am doing the install on my test computer to learn. The real work is being done by Dennis on the real server.
