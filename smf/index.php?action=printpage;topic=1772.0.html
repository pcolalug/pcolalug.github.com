Situation 2
You wrote a script in windows and uploaded it to a *nix server

You always get the ctrl-M and it is always anoying.  They have a few things to fix this like dos2unix and all but I love my sciprts, and this one tends to do the job. Not to mention some systems don't even have dos2unix installed by default (debain for example). Works on windows and *nix.. So you can run this before you upload your script. Hope it helps those headaches go away for you scripters.

editor undos

#!/usr/bin/perl -w
use strict;

if ((! defined $ARGV[0])||($ARGV[0] eq ".")) {
        print "undos [file]: remove dos line breaks from a script";
        exit;
}
if (!-f $ARGV[0]) {
        print "$ARGV[0] is not a file!";
        exit;
}

open (MODIFY, "+<$ARGV[0]") or die "Opening: $@\n";
        my @file = <MODIFY>;
        map { s/\r//g; } (@file);
        seek (MODIFY, 0, 0) or die "Seeking: $@\n";
        print MODIFY @file or die "Printing: $@\n";
        truncate (MODIFY, tell(MODIFY)) or die "Truncating: $@\n";
close (MODIFY) or die "Closing: $@\n";
