Usually, a script tries to do some work in a database.  The database spits out an error and the PHP script sends that information to the browser window (do you see any database related errors?  Hit view source, maybe they show up there).  Or something is causing text to be displayed to the browser.  

Then the script tries to redirect you to another page, or setup session information (or anything that requires modifying the HTML head tags).  It can't do that though because when the error text got sent to the browser, so did any HTML headers, so it can't modify them, they've already gone out.  

Since you're using phpnuke, I think their code is probably good.  You probably have misconfigured something, or setup the database incorrectly.

If you can't get it figured out, go to the page giving you the error in your browser, hit 'view source', and copy->paste the entire output into a code block here so I (we) can get a better look at it.

Good luck!
