Title: ScummVM on Linux
Post by: SiliconFOXX on December 05, 2004, 04:33:43 am
http://www.scummvm.org/

It's an emulator for the scripting environment, Scumm.  Basically it allows you to play some of the greatest adventure titles of all time including my favorites, The Dig, Full Throttle, Sam and Max Hit the Road, Day of the Tentacle, and others.

Anyway, it turns out they've got verisons for lots of things (even Dreamcast!) and just so happen to have a Slackware package.  It runs smooth as silk, just like the Windows version, and I'm using it to play The Dig right now (manager just happened to have a copy, bonus).

If you've never heard of these games then do yourself a favor:  go to eBay, grab both of the aforementioned, and load 'em up in ScummVM.

I'm not an expert at seeing how things in a Slack package are put in the system but so far it seems like only one file, the ScummVM executable itself, has been used and that's put in /usr/bin.  Anyone know offhand how I can see what other files were in the package, where they were put, or what files this executable needs to run?  Thanks.

Oh, if you're on Slack the older version, 6.0, works just fine.  No reason to build a new one.
