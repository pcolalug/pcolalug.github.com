csh should have tab completion it's just not on by default, I think you have to do "set filec 1" in your .cshrc or whatever file it is, and then it uses the ESC key not TAB to do completion.  (Although I don't know that just turning on completion in cshell will actually give it to you inside of oracle's sqlplus also).

In Oracle's command line you don't have full history, but you can recall the last statement and edit it and run it again.  But I'd suggest keeping a text editor open, do all your SQL in that and copy-paste to the command window as needed.  That's probably the best approach.

All of the SQL servers are different. They are similar, but definitely different. I would just work on the server at school and work around it. Even if you install the same version, the setup may be different and what might work on your install may not work on the school server. Besides, last time I checked, only the beta release of MySQL supported stored procedures and triggers, which is probably a big part of Oracle's bread and butter.

I will also second imnes' suggestion and keep a text editor open. That will make it easier to script it all out as you go. Not exactly the same, but with Cisco devices, when you create an access control list, you have to apply them in the exact order you want the rules to be evaluated. You can't insert statements, they are just appended. That means I will do my entire access list in a text editor and copy and paste it, test it, should it not work, blow it out of there, edit, copy and paste, test again. Once I have that working access list, I save that file I was working on, should I ever need it.
