Title: GNU Java Compiler
Post by: imnes on January 18, 2005, 09:36:52 pm
Okay Java programmers.  The GNU Java Compiler has been around for a while and works really well, allowing you to compile java source or bytecode to a native executable.

It was missing Swing / AWT so to get a GUI app you had to use something like native widget bindings (qt/gtk) or some other library (swt).  

The gnu 'classpath' project finally seems to have a decet working swing/awt implementation.   You can build the gjc compiler to generate xlib or gtk code out of swing source.  So what this means is you can take your typically slow / ugly interpreted java swing app and just recompile it to be a completely native gtk executable :)

I've just been reading about it but haven't gotten around to trying it yet but if it works as well as people are claiming it should be sweet.  I'd like to try recompiling something like LimeWire on it.
