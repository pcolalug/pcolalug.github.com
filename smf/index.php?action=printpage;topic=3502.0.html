Title: PS3 Linux
Post by: Yossarian on April 11, 2009, 10:43:30 am
If you follow my facebook then you'll have already seen this, but last night was spent installing Linux to my friend's PS3.  There are a million and one tutorials on how to do this, so I won't go into great detail unless asked.  We used PSUbuntu, a specially modified version of Ubuntu 8.10 for PowerPC architectures which comes with the appropriate files and drivers to install to the PS3.

GNOME lags on it, however LXDE flys on it, and provides the same functionality.  There aren't many PPC packages out there, so most of the programs we want to install will have to be compiled from source, which shouldn't be a major problem.  OpenOffice, GIMP, Firefox, all work fine and seems just like a regular desktop.  I installed VLC and there was some video output problems which was easily fixed, that works just fine.

My first problem I ever had with linux, corrupting the boot record, is actually no longer a problem with the PS3 as apparently one of the newer updates has a fail-safe to return to Sony's XMB.  However, this was only a problem once, but I was able to fix it by using the shell with Kboot, a special boot loader.

Getting the six-axis to work with the PS3 was surprisingly straightforward, but the button mapping has become a problem when playing games.

If you have any questions, feel free to ask.

Title: Re: PS3 Linux
Post by: Yossarian on April 11, 2009, 12:56:37 pm
Yes, this is a dual boot.  To get from the Sony OS you have to set the default system to the "Other OS" and reboot, and if you want to get back to the Sony OS (the XMB) you use the command "boot-game-os" which restarts the system into the XMB.

You need a USB keyboard & mouse to do this of course.

As for games, so far I've installed an Atari 2600 emulator, a NES emulator and a Super Nintendo emulator.  They run amazingly and it's a lot more convenient than bringing them wherever we go and hooking them up.
