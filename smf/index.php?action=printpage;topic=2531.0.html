Post by: danc on July 23, 2005, 09:25:52 am
phed,

I'm not clear on whether we've cleared this up for you or not...  here's a little more info that might help.

Quote from: phed

Is it possible to log in as root, go the root directory of the partition and do something like
chmod -R 777
and, with one magic bullet, remove the owner restrictions from all the subdirectories and files on the partiton?

That might work, depending on the filesystem in use on that drive (I've lost track of what that is), and the command should be:  chmod -R 777 *

If that doesn't work, you may need to edit your /etc/fstab file, and add this to the line for your /dev/sda* devices:
umask=0    (that's a "zero" there)

It would go right in with the other arguments, like this:
(umask=0,rw,noexec,nosuid,nodev,sync,procuid,iocharset=utf8)

Let me know if that doesn't work, or isn't clear...  

Dan
