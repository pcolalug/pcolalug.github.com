1384  Distributions / RPM Based Distributions / RedHat 8.0 - First Impressions on: October 10, 2002, 03:28:41 pm
Okay, the side-by-side comparison is a little scary.  RedHat has customized KDE and GNOME to the point they look way-too-similar.  Have a look.

They even took out the standard KDE apps and replaced them with GNOME apps.  Instead of KOffice you get OpenOffice.  Instead of KMediaPlayer, you get XMMS.  And Konqueror isn't even in the menus, you get mozilla.  (I fired up Konqueror via the Run dialog).

I guess this is great for the average user who doesn't know / care what Gnome / KDE are and just needs to be able to know how to get around and use it.  No matter which windowmanager they pick, menus are layed out identically.  And without having the actual program names in the menus, it's easier to "figure out" how to use it.  Instead of clicking on "Konqueror" or "Mozilla", you click on "Internet->WebBrowser".  I like that.

1389  Distributions / RPM Based Distributions / XMMS won't play MP3's! on: October 10, 2002, 02:21:20 pm
Due to legal threats (according to the newsgroups), RedHat has removed MP3 capabilities from their latest distribution.  This is easily solved, just download and install this package.

http://www.gurulabs.com/files/xmms-mp3-1.2.7-13.p.i386.rpm

To install, become root and run:
rpm -ivh xmms-mp3-1.2.7-13.p.i386.rpm

Things should then work as expected.  Alternatively, you could also just rebuild XMMS from sources.  There is supposedly a message you have to confirm about using "Unsupported MP3 plugin" or something.

I'm not sure if other media players were affected, I use gnome which uses XMMS.  If I get a chance I'll install KDE and see if they hacked up KMediaPlayer or whatever it is.
1390  Club Stuff / Articles / CD Burning in Linux on: October 10, 2002, 01:56:21 pm
mkisofs is a good program, but what if you want an exact copy of a CD?  Like copying a bootable linux CD for someone? (Unless I'm wrong, mkisofs doesn't do exact copies of CD's, just dumps files into an iso image, am I correct?)

I use a utility called "readcd" (it comes installed on most systems, it's part of the cdrecord package).

Easy to use, run it as root:
readcd dev=x,x,x f=filename

Then follow the directions above for burning it back to cd, such as:
cdrecord -v speed=x dev=x,x,x -data filename

Good luck.

1394  Distributions / Debian Based Distrtibutions / Nobody loves debian. on: October 09, 2002, 06:51:32 am
I definitely agree with Crowe here.  I used to use Slackware religiously (up until version 4.0).  Then I switched to Redhat.  I was completely lost, things were setup differently, a lot of the things I was used to manually setting up now had to be done via redhat's Setup program.

After using RedHat for a while, I find myself completely lost in slackware.  But I have come to really like the convenience RPM gives you (although it is frightening to most users who are first confronted with it).

Using Debian was completely different than slackware or RedHat.  apt is powerful, and shares a lot of features with RPM, but wasn't the same to use.  

Then FreeBSD, it's sort of a take on the old way of doing things.  You switch to /usr/ports, find the program you want to install (such as networking/gaim or whatever).  In the directory you type "make install", it downloads the latest sources, compiles and installs it.  I think if you do a "make uninstall" it will remove it.  Also if you're missing any dependencies it will put that build on hold, go get the missing software, then continue.  Pretty nice.  Not really a "package manager", but solves the problem in it's own way.

I guess the reason I use RedHat / Mandrake primarily is that it's what I"m used to and feel most comfortable with.  A friend of mine uses Slackware, and won't even consider switching to RH/MDK (although I nag him about it every chance I get).  

RedHat and MDK did make life a little difficult for us lately though.  They've both recently reorganized their distributions, switched up / changed configuration programs, locations.  So it's become a bit of a learning curve lately, but not bad.

I guess it's a move to make it more friendly to the average user, but RH (at least in Gnome) has taken the terminal program off thte quick-launch toolbar and burried it in the "System Tools" menu.
