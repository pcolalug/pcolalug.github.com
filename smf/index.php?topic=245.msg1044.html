Well, the pain started when I tried to emerge kde. It seems that to call 1.4 a release canidate is really optimistic. Many of the ebuilds have libs masked or have confilicting requirements with other programs. Folks seem to be working hard to get things straight, but I just don't know enough about their system to be of any help right now.

I will say this, however, gentoo makes your machine *scream*. I don't have hard numbers, but compiles (when they finished) flew by. Boot time seemed to be cut by 2/3 (though I did compile a lot into the kernel instead of modules).

Another handy feature that may not get a lot of notice is their rc-update script. Instead of adding scripts to the appropriate run level, one does something like
Code:
rc-update add apache default

This would add apache to the default run level. It is smart enough to know that the network should be up before apache, so it makes sure that happens for you. Really handy. So, to have a graphical workstation when you boot, you would have to do something like
Code:
emerge -p gnome
emerge gnome
rc-update add gdm default

Simple, no?

This all having been said, however, I'm putting RH8 on my primary partition tonight so I can be productive again, rather than trying to track down broken dependencies in ebuild scripts. I'll give it another go when 1.4 is released.
