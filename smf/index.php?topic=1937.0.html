I was hunting around for a new distro after rumors of Slackware dropping Gnome and gave this new distro a try.  So far I really like it.  It's based on Debian, running the latest 2.6 kernel and latest version of Gnome desktop.

I haven't really used Debian so I don't know how much of the installer and config tools are directly used and how much is specific to ubuntu.  The install is text based but very straightforward.  It even scanned my wireless network for access points and let me configure the wep security key so it could grab updates durring the install.

It comes with the typical Gnome applications (Gnome 2.8.1), it replaces Epiphany web browser with FireFox 0.9.3.  It also has Evolution 2, and OpenOffice 1.1.2.

So far everything "just works" out of the box.  The distro is free, if anyone else is looking for a clean Gnome based distro I suggest you give this one a try.  The current release is a single 500mb .iso and you can either download it directly from them, or use BitTorrent to get it.



http://www.ubuntulinux.org/

Since we were talking about it last night:

Most Gnome users know that you can burn CD's by opening a file browser window to your CD/DVD burner and dragging+dropping files onto it.  New to Gnome is the ability to work with ISO files.


Click for a full-size screenshot (93kb).

From your file browser, right-click on an ISO file.  You'll notice you have the option of opening the file with the Archive Manager, or Writing to Disc.

The write to disc feature is nice and simple, you select which burner you want to send the file to, and then you get a small window showing a progress bar of the burning process.

The interesting part here is Gnome is not using cdrecord on the back-end.  cdrecord isn't even working on my system right now which is the whole reason I dug around for a way to burn from Gnome and noticed the new iso features.

Now, a nice program for managing software (this being added by Ubuntu / Debian I think, not from Gnome) is the Synaptic Package Manager.  This is a graphical way to add and remove programs, so you don't have to sudo apt-get from the command line.


Click for a full-size screenshot (60kb).

Simple to use, descriptions for any package, ability to search for software.  If you're using Ubuntu give it a try.  It's available at Computer -> System Configuration -> Synaptic Package Manager.

Something I wasn't used to with Ubuntu is the way they setup permissions on the system.  For one thing there is no way to login to the root account.  You never even setup a root password on this sytem.  To run programs / edit files that require root privileges you have to be part of the system admin group (which the first user you create is put into by default), then it prompts for your password (not a root password) whenever you try to access priveleged programs.  You can play around as root by doing "sudo su" but that's the only way I've found.  You can't even get in as root by booting into single user mode.

For compiling software a user has to belong to the 'src' group.  This includes using the java compiler.  These things can be annoying if you don't know about them but as long as  you get yourself in the groups you need to be in everything works great.

Another usability touch-up that I really found helpful was moving everything off of the desktop.  Under Ubuntu, your desktop is empty, no icons whatsoever.  They moved all the typical device and folder links into a "Computer" menu on the taksbar, and they put the waste basket onto the taskbar as well.



This saves you from having to move / close windows to get to things usually hidden on your desktop.
