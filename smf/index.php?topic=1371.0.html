Ping is a lower level function than Samba. Ping will tell you it is on the network and your path is good, and Samba is dependent on that network connectivity being good, but ping could care less about Samba.

Quote from: SiliconFOXX
I cannot use S.W.A.T...


In your xinetd.d file for swat, make sure you have "disable = no". Here is an example of my /etc/xinetd.d/swat.

Code:

service swat
{
        port            = 901
        socket_type     = stream
        wait            = no
        only_from       = 0.0.0.0
        user            = root
        server          = /usr/sbin/swat
        log_on_failure += USERID
        disable         = no
}
/etc/xinetd.d/swat (END)

In your smb.conf, there is an option to set the description, which is what you are probably seeing:

Quote from: Samba Documentation from SWAT

server string (G)

    This controls what string will show up in the printer comment box in print manager and next to the IPC connection in net view. It can be any string that you wish to show to your users.

    It also sets what will appear in browse lists next to the machine name.

    A %v will be replaced with the Samba version number.

    A %h will be replaced with the hostname.

    Default: server string = Samba %v

    Example: server string = University of GNUs Samba Server

The part that caught my attention was:

# Uncomment this if you want a guest account, you must add this to /etc/passwd
# otherwise the user "nobody" is used
; guest account = pcguest


Nobody more than likely has no permissions on /home/foxx/UWF

Easy fix (and bad for security) is to chmod 777 /home/foxx/UWF which should allow anything and everybody to access it.

Other fix (also bad for security) is to set guest account=foxx in the smb.conf

Best fix (good as it gets with security in samba) is to create a user (I used to use samba) and the share resides in /var/samba/sharename, with samba as the owner.

Personally, I gave up on Samba just because it poked to many holes into my Linux boxes (just my opinion), and when I need to transfer files between Linux & Winders, I use SCP.
