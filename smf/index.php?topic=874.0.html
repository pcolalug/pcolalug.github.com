It stinks! Let me go on record as being the first person on this board to actually word, what I am sure others feel about Gentoo, it is stupid.

There, I feel better.

First of all, I downloaded the stage3 image. But no one gives directions on how to use it. Even following the directions on the web site, it is assuming that you have a connection to the Internet. If you have followed my rants in the past, you know that I don't like distros that assume that you are always on-line.

So I downloaded the stage 3 so that I would not have to do any installing from the Internet, it should have all been on the CD. Apparently it was, but when I went to emerge my files in, it always looked to the internet and not where the stage 3 tarball was extracted. Does the documentation tell you where to point anything to use the tarball that I spent several minutes extracting? No.

Before I go much further, I do offer hope. I did get a working root and user account eventually.

During the install, I was able to use my USB to 10/100 NIC. (But I should not have needed to since I was doing the stage 3 install.) When I got to the point of custom compiling my kernel, I had no clue as to which module I needed to compile in for the USB card to work. Well, thought I, since it works during the install, then it must be one of the ones turned on by default. Wrong again. So after the final kernel compile, I no longer had network connectivity. I checked out the kernel and found that the driver that I think I need should have been loaded as a module (pegasus.c). Doing a "modprobe pegasus.c" would only tell me that it could not find the module (but it is sitting at /usr/src/linux/usb).

I have to admit, I don't know what I am doing for loading modules, but that is what I read should do the trick. Still nothing.

Let's back up to the kernel compile. I guess if you have done many kernel compiles, then you know about how long it should take. I have rarely done it, and that was way back in my early days with Linux. I realize that it is a text based install, but there could have been an indicator telling me how far I was to being done. I know it all depends on speed of computer and what is being compiled in, but it could have at least told me a percentage complete.

I was so excited when I saw the first status indicator. I thought, "wow, this is going quickly," until I realized that it was the first of about 200! (OK, maybe a little exageration).

So now I have a nice command line of Gentoo. Which I am sure works just fine. But I need OO.o, Firebird and Thunderbird to make my computing complete. A command line doesn't get that for me. All of these require X. I cannot install it, 'cause I don't have any instructions on how to install from the huge tarball that I downloaded. And I cannot connect through my network device because it is not compiled in. Of course I can re-compile, which I was planning to do on occasion anyway, but I only have a good internet connection (something more than dial-up) when I go to someone else's house. I wasted all of my time that I could have been emerging X to figure out that my network card would not work anymore. Even though it worked great during the initial install.

My recommendations:
Give instructions on how to use the stage 3 tarball .
Have some kind of status indicator to tell how much kernel compile is done.
When doing a menuconfig, load the kernel that I used to install the system and let me tweak from there. Instead of loading some generic kernel that I have to completely configure, not just optimize.

It is the first distro that I have gotten to run on this notebook. But, I am now going on a 4 month trip without any Linux because in it's current state it is unuseable for my needs.

Now, David, lets not start a flamewar. I use Gentoo because I thought that most of the "boxed" distros (Mandrake included) did a lot of things that were "stupid". I think what you mean is "I don't have the time or the interest to build my system for myself and don't understand the mindset of a person who would." Fine, but lets not get nasty, shall we?

This just confirms that Gentoo aint for everyone. To be fair, though, it never claims to be, and can you really say you weren't warned? Folks used to the ease, convienience, and hassles of the "big name" distros may not have a very good time with Gentoo.

I have to heartily disagree about the install documentation. Throughout, it says stuff like "If you are starting from stageX, you can ingnore this section and skip to step Y". Also, there is separate documentation for desktop setup. Also, I've found answers to just about every problem I've had by searching the forums. Note that if you are to the point of being "Linux-savy", these forums are a good resource for Linux in general, though you will likely have to adapt the solutions to your distro. Also, if you used the 1.4 rc 4, there is no Gentoo Reference Platform which contains the binaries for the packages you mention. What you need is for some kind soul to download the ebuilds you want and then show you how to inject them into your portage tree. Let me know if you want to do this.

Gentoo is something you might want to install with a buddy the first few times (Rugmonster helped me with the kernel config - ah, BSD, how I miss your simplicity sometimes...). Otherwise, folks, you're in for a long, likely frustrating, few days. That being said, however, Gentoo is the perfect setup for me. I love the control over the system, the easy software management, and the endless "tweakability" of the thing.

When you get back (November?) and feel up to it, you are more than welcome to come over to the house and I'll help you with the install.

Quote from: dpeach
Isn't that what the 250MB tarball is for?

Nope. The big (-3stages) images just include the stage tarballs on the cd. From stage3, you have a GNU toolchain (make, automake, autoconf, etc), as well as the system stuff (libs and the like) emerged for you. Some kernel sources may be included, I dunno. However, you should still have to emerge, config, and install a kernel source tree, system logger, boot loader, as well as many other programs you would want to use. All of this requires an internet connection. Now, the GRP should provide you with pre-built binaries of many popular packages, once that's done, but its not. Again, you could have a buddy
Code:
emerge --fetchonly
the packages you want and inject them into your tree via cd or something.

Quote from: rburt3
Quote from: dpeach
And if you can't install without an Internet connection, then I whole heartedly disagree with that requirement.

Well, there's always Debian or Slackware... Ain't choice grand?

I appreciate choice. And I like the idea of being able to install from the net. I don't have to worry as to whether the package is the latest one that works, or if it was one built way back in the 20th century.

But "being able to install from the net" and requiring me to install from the net are two very different things. With no option to go without Internet connection is just not reasonable. I can think of millions of people in third world countries that are installing Linux for the benefits, and yet they do not / cannot have an Internet connection. If Gentoo is good for rich Americans with broadband, why shouldn't our friends in Botswana be able to play?

It is a requirement that I just do not like.

Does that mean that I will never run Gentoo? No. Actually, I really like the idea behind it and think that it is generally a good distro. Will I promote it with any kind of passion? Probably not.

We are just a group of too many nice guys. I can't even get a flame war started.

BTW, Gentoo still stinks!
