2  Software / Programming/Development / Linux answer to DreamWeaver? on: September 09, 2004, 09:10:34 am
Hi imnes and Rugmonster,

    This is a very complex issue and I'm certainly no expert.  As I understand it, the short answer seems to be:  amd64 machines will run either 32 bit or 64 bit code . . . if they are in that particular mode.  The critical issue seems to be with things like pointers, default word lengths and the like.  Obviously, if your pointer suddenly has 32 extra low-order bits it isn't going to be pointing anywhere near where you want it to.

     So, if you are cruising along in 64 bit mode and you want to run a 32 bit binary you have to switch modes . . . which probably means pushing everything on a stack somewhere, switching modes, poping things back and continuing on.  I'm just guessing here, of course, but there has to be some mechanism to make the transition.  You also need to be using the appropriate libraries, so these need to be pointed to appropriately.  Obviously this can be done, but some effort has to be made to do it.  Which is to say, you can't just throw some arbitrary binary at it with out providing for it in advance.

     Which is one of the main reasons I wanted to go with Gentoo.  It would seem that you could simply compile all your source code in 64 bit mode and away you go . . . and in many cases this is true.  On the other hand, in many cases it is not true as well.   I've been following the amd64 forum on Gentoo.org for months now and the general impression I get (and some folk have stated) is that if you need to get serious work done stick to 32 bit for a while longer.  Even relatively simple programs like GRUB have to be used as a 32 bit binary, it won't compile in 64 bit mode.  GRUB doesn't matter, of course, because you haven't actually begun anything when you use it, but the same is true of Open Office and several others.  Vmware apparently remains too flakey to be useful, for another example.

      Some people are reporting smooth working systems under 64 bit (Gentoo), but they are invariably far more experienced than me!  And I have to admit that the solutions and "workarounds" that some of the folks on the Gentoo forum have come up with are nothing less than astounding!  I can't understand most of them even after they are presented to me in words of less than 4 letters!

      All of which is NOT a reason to forgoe a 64 bit system of some sort.  I have no doubt at all that all of this will get sorted out eventually . . . it is just a very nasty and horribly complicated problem.  AND the 64 bit processors are really, really fast in both 64 bit and 32 bit modes!  I did do a complete stage 1 Gentoo install before deciding to give it up for a while, and my dual 2GHz Opteron system can compile the entire kernel in less than 90 seconds, for example.  The entire stage 1 install took about 3 hours, and a good deal of that was spent downloading source code and portage scripts from Cox cable.

     The Gentoo amd64 forum is very helpful, and I would strongly recommend you go snoop around there to see what is happening.  People are extremely good about answering questions (as here!) and the 32 v 64 bit question comes up fairly often.
