This brings up an interesting point about Linux on the desktop that I don't think has been raised before, so here goes.

I've stopped reading distro reviews on sites other than this one because it seems that the experience you get varies widely depending on hardware/configuration. Other than performance issues, I don't think this is the case with Windows(TM), and certainly isn't an issue with Apple(TM). I've seen fairly good results with RedHat, SuSE, and (of course) Gentoo, but there have been differences based on the hardware I've installed on (stability, certain configs not working, etc).

Linux really can't go the Mac way, and I think the user-base is too diverse to really achieve consistancy across hardware the way Winderz does (besides, I don't think we want to shoot for "Performs equally badly on any supported hardware" ). Sure, boxed distros can compile for the lowest common denominator and if more folks start using the Knoppix hardware detection/auto config set up, things should get better. What do you think?

Well if you think about it, windows doesn't really support any hardware.  It has a lowest common set of drivers to get a basic system going, then all your fresh drivers are installed later.

Linux has just about everything built right into the kernel, and  they are all at various stages of bugginess.  And you never know when someone working on one part of the kernel will unknowingly break something else in the kernel.

In all fairness I've got an old Motorolla SM56 winmodem that works great in linux but won't work in any MS platform newer than Win98  Also true of a lot of the windows-only printers.  

Another thing, one of the systems I own is flawless under Linux.  It's an AMD Thunderbird 850mhz, FIC Motherboard, Matrox Millenium MGA 400 16mb dual head video, 1.5gb PC-133 ram, 3x6gb IDE drives, SCSI DVD-Rom, CDRom, and CDRW (courtesy of Steve).  Perfect under Linux or BSD.  But it will not work in any version of windows.  Crashes for no apparent reason under NT4.0, 2000, XP.  I'm not sure what the deal is.  All hardware tests fine...  

On drivers again, under the Windows OS all your drivers come from the people who built the hardware, who know what they're doing and have all the specs.  A lot of the drivers in the Linux kernel though have been reverse engineered / hacked without access to hardware specs and engineering docs.  So the fact that it even boots on most systems is a pretty good acheivement
