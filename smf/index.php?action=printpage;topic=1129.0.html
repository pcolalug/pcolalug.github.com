Title: FreeBSD 4.9
Post by: imnes on November 04, 2003, 10:31:33 pm
Ok so far all my hardware works in FreeBSD 4.9 with no special attention required.  This includes:

DVD+CDRW (burning works through the BSD burncd appl and also cdrecord)
WiFi (Lucent Orinoco/Hermes)
USB
Sound Card - i810 Intel
nVidia GeForce 4 420 Go Video Card (commercial nvidia drivers)
pcmcia - Works (I have a 120mb SanDisk that I use on it)
Keyboard - Actually works great!  Toshiba keyboards have a problem under Linux + X doesn't seem to have a problem here...
Onboard Lan - Intel EtherExpress Pro 100 - Works fine
APM works I can suspend and resume fine.

Some pleasant surprises?  Some things I didn't expect to work (or at lesat work well) have worked great.  Including Gnome 2.4 (I have major stability issues under recent linux distros that just aren't a problem here).

Java SDK.  I didn't think would work at all.  I finally got the official 1.3 SDK installed and working.  I haven't tried 1.4 yet but this is good enough for my school projects.

Sun StarOffice 7.0 - This actually installed perfectly and works great.  A lot faster than the previous version.  I think OpenOffice is better looking with it's gtk2-ness and pretty fonts, but this does work fine.  Even Adabas support.  FYI: This product is completely free if you are a student / faculty at a University...  The link you need is:   http://www.sun.com/products-n-solutions/edu/solutions/staroffice.html

Games work as expected, Unreal 2k3, Quake 3.  They play great.

Some things are just working better under BSD than Linux.  One thing is the USB mouse.  I have a touchpad on the laptop, and use an external USB mouse.  Under Linux if I go into X without the mouse plugged in, and then try to plug it in, it won't work, I have to restart X.  Under BSD whenever I add another mouse to the system, they all route back to the same device, so X doesn't notice anything going on, I can plug, unplug all day and things work.

Another important area is sound.  I'm sure some of you have perfectly working sound drivers in Linux, but you are still plauged by the kernel only handling one sound source at a time.  So you run ESD or ArtsD and hope everything is happy.  BSD mixes multiple sound sources at the same time, so I can run ESD, ArtsD, Quake 3, and XMMS concurrently and they all work.  I wish the linux sound system would work similarly...

If you guys have questions, want me to try something out in BSD let me know...
