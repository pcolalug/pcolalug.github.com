Title: ASUS, Get your act together
Post by: steve on January 19, 2005, 06:35:41 pm
Well, I'm typing from my Windows machine as my Linux system is dead. Trying to troubleshoot a memory problem last night, I decided to flash the bios to the latest version. Per Asus' instructions I downloaded the bios file to a floppy, inserted it into the drive and booted the system. I was really happy because Asus has an Eazy-Flash utility where all you have to do is hit "Alt-F2" during post and it'll update the bios from the floppy. I did that. The system said "Reading Rom image.  Erasing Rom.   ROM Update Failed!"

Well, I started getting nervous. I left it at the last line for about a half hour, praying that it would come back and say "SIKE!". It didn't. I reset the machine and guess what? Nothing! No display. No Boot. Nothing. I then used my other (Windows) system to search the net. It seems that others have had this same problem. Solution: Don't use the Asus Eazy-Flash upgrade. BTW: I HAD and Asus K8VSE Deluxe AMD64 board.
If you check Asus site, on the ROM version before this one, there is a not to use the Bios upgrade software for that upgrade. However, there's no such note for the latest upgrade.. "Thanks Asus"

Anyway, went to NewEgg today and ordered an AOPEN AK86-L motherboard shipped overnight express. It’s pretty much the same but for about 1/3 the cost. It also has some good reviews in the Linux world.

Well, it looks like I've got another Motherboard manufacturer to stay away from....

Steve

Title: ASUS, Get your act together
Post by: rugmonster on January 19, 2005, 07:24:36 pm
I had a similar issue. I used the Windows utility for upgrading the BIOS on my Asus K8V Deluxe. There was one version that said not to use that utility, but that was once you dug into the site. Well, I applied it and rebooted the system expecting the best. I had a very similar problem. Nothing happened. Me and my friend bought our systems at the same time and have nearly identical pieces, so we did this both at the same time. After a little brainstorming, we pulled the CMOS battery and unplugged the system to try to reset the values there. After replacing the battery and turning it back on, we had to reconfigure the settings in the CMOS, but all was fine. Turns out, there were several changed values in the CMOS that somehow messed up the boot process. Clearing the values for the older BIOS made it use the defaults and all was well in the world.

I think my next motherboard will have a second, on board BIOS, just for times like that.
