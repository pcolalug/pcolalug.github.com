43  Club Stuff / Club discussions / Kernel Compile clinic. on: December 31, 2004, 10:42:14 am
Way to go, peach! Anyway, yes, the make mrproper isn't really needed, but I do it the first time I build a new kernel version. This should give me a pristine build environment sans any artifacts from packaging and whatnot. Isn't nessisary, but doesn't hurt.

For making kernels in the 2.6 tree, you really only need the following (you need to have your boot partition mounted and have write privs on it for the install target):

if you've configured any modules:
Code:
make menuconfig && make && make modules_install && make install


if you've compiled everything in the kernel (which has some problems if you want to use third party drivers like nvidia), you can omit the modules_install target.
