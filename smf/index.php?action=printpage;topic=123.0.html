Mandrake is designed more as a Desktop OS than a Server OS. Granted, you can use it as a server but it is very gui oriented. So, if your using RH as a server and happy with it, I'd say stick with it. If you'd like to try out a Desktop OS then try Mandrake. It's very easy to setup and usually work right the first time. Mandrake is also very big on including cutting edge software in their distro. That's why they release a new version every 6 months.

Remember, you won't read much about the succesful installs in a forum so don't use them to guage how good a Distro is. I go by the rule of thumb that if I want to see the good points about a product I'll look for reviews. If I want to see what kind of problems people have with it, I go to the forums.

Title: So what's the deal with everyone and Mandrake :P
Post by: steve on August 20, 2002, 08:17:38 am
What a lot of people forget, or don't realize, is, it doesn't matter which distro you're running. The difference between RH, SuSE, Mandrake, Slackware, etc. is what programs they package with Linux and the user interface for managing the system.

Linux IS the kernel and it's pretty much the same in all the distros. There may be some minor differences but nothing that the user really needs to worry about.

So, if I'm running Mandrake from the console with my apache web server running and sendmail/qmail/whatever and dhcpd and squid, and snort, etc. It isn't any different than running RH with the same stuff.

IMHO when it comes to servers, the difference of a good distro vs a not so good distro is how fast they release security updates for the packages.

But Apache is still Apache whether it is running on RH or Mandrake.

Think of it like a book. If I write a book and it's published by doubleday it may have one cover but if it's published by O'reilly it may have another cover. The book is still the same, it's just the cover/wrapping that has changed.

However, when it comes to desktop distros, you need to look at who has the easiest interface to work with. Again all the software is pretty much the same but the interface and management tools will be different. Then it comes down to which gui management tools and interface you like better.

Steve

I've run every distro I could get my hands on. In fact, I'm known for formatting my computer several times a week  ;)

Right now I'm testing out SuSE 8.0 and I really like it. But generally Mandrake (at least up to 8.2) was my distro of choice.

The problems I've had with my latest computer though have given me reason to stop using Mandrake (and RedHat). First thing is the kernel they include gives me major problems with my sound card. I can compile a custom kernel to fix that, but also the compiler that comes with recent versions of RH and MDK give me problems. (XMMS, mplayer are almost unusable for me).

I've tried Slackware, it's okay. I've run ever version of RH since 5, and I like it. Mandrake just has the nicest installer (and most robust set of desktop apps IMHO). I must say though, I made the trip into FreeBSD territory recently. I downloaded and installed FBSD 4.6.2 and it is very nice. It uses a lot less resources than recent versions of linux, and seems to support my hardware better out of the box.. But it has it's problems too, mainly VMWare and Kylix don't run on it (two programs I like to play around with). If not for that, I would still be running it.

SuSE seems nice, but it's a pain to install. I couldn't find any iso images to download so I had to do a (2.6GB) install via FTP. Not too terrible over DSL, but it still took a few hours. SuSE seems to support my hardware perfectly, but I ran into a problem with it too. The custom kernel they use (it includes memory patches from the 2.5 series) causes VMWare not to work. So I had to compile a custom kernel and disable >4gb memory support. It actually took me about 4 tries to get the kernel compiled and still have all my hardware work, but it's up and running fine now.

Back on MDK though, I had some stability problems with KDE3 (especially Konqueror) that I'm not seeing in SuSE. I'm gonna take a guess and say it's the compiler that comes with MDK. Anyone else have problems with konqueror crashing frequently?
