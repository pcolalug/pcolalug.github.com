After much delay by the seller, who has earned a solid negative feedback due to the run around he gave us, we got the server and got it working tonight. Below are some pictures, but first, let me tell you a story about the bootprom password we had to overcome.

The server shipped with no OS loaded. If you are familiar with Sun Sparcs, you know about the bootprom. To boot another device besides the default, you type boot cdrom or boot net. The default order was disk (hard drive) and  then net. When we tried to type in anything we were faced with "firmware password". We couldn't even probe the scsi or boot off of the dvd-rom.

We knew that we had to overwrite the value in the NVRAM to disable the password. First, we got a boot image for Debian. Since the box would boot across the network, we setup a rarpd and tftpd to load this image across the network. That boiled down to no way to get create the /dev/openprom device file so that the tool eeprom could overwrite that value.

Going on a tip from #solaris on freenode, we did something very drastic. I pulled the bootprom from the motherboard which is located beneath the very large power supply. After turning on the system and allowing it to fully initialize, I reinserted the bootprom. Then we reset the values to factory defaults which worked fine. We then had to manually set the MAC address for the NIC since that was apparently stored on that eeprom. Once we did that, the system worked like a champ and my partner in crime is installing Solaris 9 on the machine now. Once that worked, you had better believe we were relieved, because at first we were getting errors because there was a checksum error due to the MAC address not being loaded on the bootprom.

I'll post pictures in the morning. I'm tired.
