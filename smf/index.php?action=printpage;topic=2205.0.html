Title: Web Development on Linux
Post by: imnes on January 23, 2005, 05:11:22 pm
Okay, for all of us doing Web / HTML type work under Linux.  Being able to test cross-browser function of your page is usually necessary.  I've got Mozilla FireFox 1.0, Internet Explorer 6.0, and Opera installed on here.  But being a Gnome person I really don't like having to install kde just to test for konqueror users.  And having to install a mac emulator + OSX to test on Safari.  

Maybe this is a solution:

I stumbled on this project today called GTK WebCore.  If you guys remember the announcement (last year?) Apple took the KDE rendering core from konqueror (Khtml) and ported it to use in Safari.  So as far as I know safari and konqueror should render pages similarly.  Now someone's taken the Safari code and ported it back to Gtk.  It comes with a reference browser implementation, so now you can supposedly get Safari / Konqueror browsing natively in GTK.  

I'm going through the build process right now on Ubuntu and saving the debian packages I create.  If anyone wants to test them out I can make the packages available, but building the thing is pretty straightforward.

You can check out their project at: http://gtk-webcore.sourceforge.net/

But for loading and previewing pages it's fine.
