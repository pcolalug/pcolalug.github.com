Title: Netcat Tricks
Post by: bhoff on June 24, 2006, 08:42:57 pm
Need to copy an entire directory tree from one computer to another as root?
Well normally I would try ssh but without ssh logins this is pretty darn hard plus you
would have to compress the dirctpries and all.
Try netcat.
On machine A:

# tar -cf - . | nc -w 3 -l -p 5432

The command on machine A sets up a tar process on machine A that waits for a
TCP connection on port 5432. When a client (such as machine B) connects,
tar simply sends all the data through that TCP port, thanks to netcat.

On machine B:

# nc -w 5 a.example.com 5432 | tar -xvf -

No temporary files are required. On machine B, another tar process reads
from the network via netcat, writing the data to disk as it streams over the network.

Don't think I am this smart, Thank them guy's and girl's over at Linux Mag.
I was looking thru my online back issues and found these tricks Justthought I would pass them on.

My only concern with this is that it is completely insecure. You might as well be using rcp. No authentication and at most netfilter rules protecting you. Key based scp is much better and takes minimal setup. If you do use something like this, it is best to only have use it for one off sorts of transfers; not something you have running all of the time.
