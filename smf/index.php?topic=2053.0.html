You're thinking SMS, which I have used, but it is fairly resource intensive and practically requires it's own server, which in the package I work on doesn't have an extra to spare. SUS is for Windows Critical Updates, and only those. They are coming out with Windows Update Services (WUS), but that will only cover Windows, Office and other MS product critical updates. That still is not everything that we cover with are responsible for keeping up to date.

Supposedly, you can use SUS (and soon WUS) on networks that are not connected to the Internet, but I have yet to see it work. I don't want to invest energy and server resources, which are a commodity in our setup, to something that won't do everything we need it to. Another issue I have with it is there is no reporting or logging, meaning I can't know if all of the systems have been updated. This is vital considering compliance is mandatory and we have to account for this on our networks.

Sorry about the heavy MS talk. This is one of those times that I can think of how easy software management would be in a Unix environment with tools such as shell scripting, cronjobs, and the ssh set of tools. So, now no one can say I think everything is easier in a Windows environment.
