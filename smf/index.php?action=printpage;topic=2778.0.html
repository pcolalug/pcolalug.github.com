Title: MPlayer on Solaris 11
Post by: imnes on January 29, 2006, 01:21:45 am
Compiling MPlayer under Solaris 11 (and Solaris 10) works, but win32 codecs cause it to crash.  Here's a fix (don't know if this will cause other problems or not, but it's working so far for me).

Edit the source file: ldt_keeper.c

Change:
#define NUMSYSLDTS 6 

to:
#define NUMSYSLDTS 7

Recompile and win32 codecs should now work fine.

I tried to use it and it crashed everytime it loaded a windows dll.  I was researching the error and found some talk I think on a WINE (Windows Emulator) mailing list that mentioned Solaris now reserves the 6th LDT (I'm not sure what that is though) so I bumped it up to 7 and compiled and it worked fine.

I just did a google search and looks like Nexenta did a similar patch but they bumped it all the way up to 10, not sure if that's better / safer.

I was just posting my fix here for future reference incase I need to install it again.
