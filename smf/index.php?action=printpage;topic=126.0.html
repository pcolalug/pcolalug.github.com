Title: Web interface to our Library
Post by: rburt3 on August 21, 2002, 07:28:05 pm
I've talked it  over with David and Steve and I propose to create a mod for phpBB that will allow us to interact with our "library" via these forums.  So far the specs are as follows:

phpBBLibrary_Mod would be a mod to phpBB, implemented as a new forum type,Library, and some associated additions to the admin console and user
priveleges.

1. use amazon api to tie isbn to image, title, description, link to amazon
2. users allowed to use library can check out materials that are available,
get on a waiting list if they are not, and see what they have and when its
due back. Can optionally receive reminders when something is due. Users with overdue materials cannot check out additional materials.
3. Admins can configure who uses the library, how long things are kept, how
and how often reminders are sent to users about returning/overdue materials.
4. Librarian (Library moderator) will confirm that each trasaction (check in
and out) actually took place before committing to the library.
5. Librarian can see at a glance what is due back and when and by who. Can optionally recieve periodic reminders when due dates are close.
6. Catalog can be broken into categories to ease browsing/searching.  Books may, however, appear in multiple catagories.

This thread is for you to post comments, sugguestions, and feature requests, as well as chime in if you want to lend a hand.  I tentatively plan to have this thread up for a week or so, after which time we'll have a feature freeze and start designing and coding the monster.

I went ahead and added the capability to the library.  Now when you input an ISBN number of a new book, it attempts to pull in all the information it needs (except for the Category since that's specific to our library) from the Books-A-Million server.  If the book isn't cataloged by them or doesn't have an ISBN you will still have to enter all the information manually.

It will pull in the info from books a million and give you an opportunity to change anything you like before adding the book.  Otherwise it still operates the same as before.  Give it a try, let me know if you encounter difficulties.  

Ultimately this should be changed over to the XML API that Amazon.com provides.  Since this relies on the way information is presented on the BAM pages, if they change the wording much this feature would break.  I may tackle the amazon stuff later when I know something about XML.  If you are an XML guru feel free to take on this project yourself.

Okay people as you're using the library keep an eye out for this oddity.  Quote marks " and apostraphes ' showing up with a backslash in front of them \" or \'.  This is a side-effect of PHP + MySQL.  I have to use a function called addslashes when inserting information into the database, but I forgot to use the corresponding stripslashes function when displaying that data on a web page.  I didn't catch this when I was testing but saw it once some other members posted books.  I've made the changes on a few pages but let me know the URL of the pages that still need to be fixed as you notice them, just post the URL to here please.
