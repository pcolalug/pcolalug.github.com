Title: Re: w32codecs
Post by: imnes on March 14, 2006, 08:50:38 pm
Another option, download this archive (it's 12MB compressed)
http://www.mplayerhq.hu/MPlayer/releases/codecs/all-20050412.tar.bz2

Extract the contents to /usr/lib/win32 by running
Code:
sudo mkdir /usr/lib/win32
cd /usr/lib/win32
sudo tar jxvf /path/to/all-20050412.tar.bz2
sudo mv all-20050412/* .
sudo rm -rf all-2005412

That will give you the latest win32 codecs (incase the ubuntu package isn't available, or you just want something newer).  Also you'll have the best luck installing totem-xine for getting all your videos to work. 
Code:
sudo apt-get install totem-xine

And for your web browser
Code:
sudo apt-get install totem-xine-firefox-plugin

If that browser plugin gives you any trouble try the mplayer one.
Code:
sudo apt-get remove totem-xine-firefox-plugin
sudo apt-get install mozilla-mplayer

If you're having trouble getting your /etc/apt/sources.list setup correctly, here's what I currently have:
Code:

Title: Re: w32codecs
Post by: rugmonster on March 15, 2006, 05:00:24 pm
I just went to the mirrormax.net backports mirror and you need to change hoary-backports to hoary-extras. mplayer and the w32codecs are in the standard repositories, you just need to add multiverse and restricted to your sources.list. What this means is that you would find the line that says...

Code:
deb http://us.archive.ubuntu.com/ubuntu hoary universe main

...make it look like...

Code:
deb http://us.archive.ubuntu.com/ubuntu hoary universe main restricted multiverse

Have you run this since changing your sources.list?

Code:
$ sudo apt-get update

One more thing, why are still using Hoary when Breezy is available? All it should take is changing all instances of hoary to breezy in your sources.list and running...

Code:
$ sudo apt-get update && sudo apt-get dist-upgrade

Next month, the next version of Ubuntu, Dapper will be released and I believe Hoary support will go away.
