Hehe.  The dpeach man underestimates himself...  

Silver, I don't know much about Fedora, but like dpeach said, you need to properly configure your "X server" which is what provides the graphical interface on Linux.  Try this:

1.  Do the Ctrl-Alt-F2 sequence again, and at the login prompt, log in as "root", using the root password you specified during the install.
2.  Type the command:  xorgconfig
3.  If #2 does nothing, try this command:  XF86config (or XF86Config) (note capital letters, things in Linux are case-sensitive)
4.  If either #2 or #3 start a configuration program, just go through and answer all the questions.  The important ones are which video card driver you use, and what resolution (horiz and vertical frequencies) you set the display at.  If you don't know those specs, try Googling for your laptop specifications to get them (should be a range, like 30-90 or 50-120).
5.  If you get that finished, log out ("exit" command) and then log in as your normal user name (NOT root).
6.  Type "startx"
7.  Tell us what happened...

The cyberblade chipset apparently has serious issues with X, and the suggested "fix" is to use the VESA driver. (See http://www.linuxquestions.org/questions/history/344844)

My opinion is that solution bites, as the VESA driver is "the driver of last resort". My guess would be that since it is using shared memory, that the kernel is somehow "stepping" into video space and therefore is attempting to deal with corrupted RAM. My suggestion, to test this idea, is to go into the BIOS and reduce the "shared memory" footprint to either 1 or 2 meg vs. 8meg. This could make it possible to at least get you past the install, and at that point you'll just be missing color depth. Another option is to pass the option to the kernel (don't remember right it off the top of my head) to explicitly tell it how much memory is available (Total Memory - Video Shared Memory). My experience is that shared video cards take up the tail end of memory space, and then the BIOS reports the reduced about. While this may work for Windows, Linux "sees" that extra memory and is probably trying to use it for scratch memory. Another option (provided that Presario is a desktop) is to disable the onboard video and put another video card that has its own memory onboard.
