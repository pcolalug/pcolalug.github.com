I've gone through the first 100 pages of this book today.  This book is slow going, just learned how to set the graphics mode, load a bitmap, and display it on the screen.  Lots of introductory stuff about the game industry, discussion about Linux, the programming tools, compilers, debuggers, text editors, etc.

Game Programming was the reason I got into programming (and was my whole motivation for going to college, to get a job as a game programmer).  Done a few lame games in the past as school / hobby projects, so I'll definitely finish this book and try out the tools / techniques offered.  

My favorite game toolkit in linux is Allegro http://alleg.sourceforge.net.  This book touts SDL (Simple DirectMedia Library) which is much more low-level, but not too hard.

I'll post a better review when I finish it.  Anybody know a graphics designer that could draw me some artwork for a game project when I get done?

Yeah, you kinda have to wade through some intro material to get to the good stuff (code). All in all, though, I thought it's a good intro to the basic development tools available in Linux. I especially appreciated the debugger information, as this hasn't always been my strong point.

Quote from: imnes
Game Programming was the reason I got into programming (and was my whole motivation for going to college, to get a job as a game programmer).  Done a few lame games in the past as school / hobby projects, so I'll definitely finish this book and try out the tools / techniques offered.

Me too, sorta. I got into computers because of gaming, though I don't hold high hopes of breaking into the industry. I've toyed with mods and such, though.  Its hella fun. Just wait, imnes, once I get that fat wad of cash, we'll code us a game.

Quote from: imnes
My favorite game toolkit in linux is Allegro http://alleg.sourceforge.net.  This book touts SDL (Simple DirectMedia Library) which is much more low-level, but not too hard.

Yeah, SDL's for writing engines rather than a full game API.  In fact, there's not much you couldn't write using SDL, multimedia-wise. For game engines, I've been looking at Torque by Garage Games ($100 per license, per developer, not too bad) and the Nebula Device from Radeon Labs (free). It'd be cool to build the engine from scratch, but John Carmak, I aint!
