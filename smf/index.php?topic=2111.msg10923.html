Quote from: dpeach
I have never done the step: make bzImage. What does this do for you? Isn't the bzImage from a previous compile useable?

The new bzImage that you compile *is* your new kernel.  In my example, we move it to the /boot directory, and rename it to "vmlinuz-2.6.7".  You can actually name it anything you want, as long as you call it that way in the kernel line in your bootloader config file.  But anyway, using a bzImage from a previous compile would defeat the purpose of doing a new compile...

I'm pretty sure that you can also use just the command "make" at this step, instead of the two steps I used ("make bzImage" and "make modules").  Perhaps that's what you've been doing, and actually making a new bzImage without knowing it.  The term "bzImage" itself is a "target" of the Makefile which the compiler uses to produced a bzip'd (compressed) kernel image.

Quote from: dpeach
And what does the mv System.map line do for you? I know what the command does, but why do it? I don't know what the System.map is.

The System.map file is a somewhat mysterious beast, and your system can operate without it.  It has to do with very low-level symbol calls in the compiled machine code, and helps the kernel know where things are addressed in memory.  Also it is somehow involved in system logging, which can be important when debugging a problem.  The way I understand it, it is not actually required, but is a "good thing" to do.  Here is a link with some very good info if you want to read more: http://www.dirac.org/linux/system.map/

Hopefully that explains a little for you.  Kernel compiling is a big hairy monster that I only claim to know a little about...

Way to go, peach! Anyway, yes, the make mrproper isn't really needed, but I do it the first time I build a new kernel version. This should give me a pristine build environment sans any artifacts from packaging and whatnot. Isn't nessisary, but doesn't hurt.

For making kernels in the 2.6 tree, you really only need the following (you need to have your boot partition mounted and have write privs on it for the install target):

if you've configured any modules:
Code:
make menuconfig && make && make modules_install && make install


if you've compiled everything in the kernel (which has some problems if you want to use third party drivers like nvidia), you can omit the modules_install target.
