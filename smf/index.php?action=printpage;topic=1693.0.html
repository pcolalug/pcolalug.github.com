Title: Best server distro?
Post by: erc on June 11, 2004, 01:16:28 am
I am in the (un-)enviable position of having to replace a rather aging and non-supported RedHat 7.2 distro with something else.  This is a development server, so (1) it's gotta be Linux, and (2) it's gotta be small, fast, and no X!

The server is an AMD K2/300 with 192MB of RAM, a 10GB root partition (contains OS, home directories, email), and other drives for development and backup use.  It runs headless, so running X would be a huge waste of server time and resources.

Anyone recommend a distro that's (1) painles to install, (2) has a small footprint in terms of disk and memory space, and (3) doesn't have X , or at least is painless to *not* run and include X?  I'm not looking for Mandrake or similar, where the distro is designed for the desktop and getting X and it's associated huge suite of libraries and apps to not install is impossible, or almost so.  The other requirement is that it be able to be automatically updated over the net without under intervention.  This box is designed to sit in the corner and never be touched - it reboots itself once a week or so, early Sunday morning, just because :)

I'd go with FreeBSD for the server (it's otherwise perfect for such an application), but since the development server is used for producing Linux apps, it's gotta run Linux.  My other development server happily runs FreeBSD in about 150MB of disk space :)  (And yes, it's the Escapade development server, thanks for asking!) ;)

Thanks in advance :)

Title: Best server distro?
Post by: steve on June 13, 2004, 08:14:27 pm
I've stayed out of this topic because I don't normally deal with the Servers. However, I have helped our engineers choose Linux servers for projects they've been working on. Usually when they ask what the best server to run is, I answer by asking what programs they plan on running. I then send them to the programs web sites to see what distros those programs are being developped on. In the case of Apache, Samba, Sendmail and other common server apps, you can find them on most server distros. However, if you're going to be running something new/out of the norm, find out what it's being developped on and that would be the way I'd recommend to go. You're going to get better compatability, stability, and support that way.

This also means if the programs are still being developped and tested on RedHat 7.x then stick with it.

I've found that most big server programs are normally at least one distro version behind the current version. So, I wouldn't necessarily plan on going with the latest and greatest version.

Steve
