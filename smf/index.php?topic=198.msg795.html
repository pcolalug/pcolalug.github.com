The forums have been dormant for a while now, see if I can generate some discussion...

What do you guys think about the movement towards a standard linux desktop across distributions?  Some of the larger distros (like RedHat / Mandrake) are working hard to make their particular system more user friendly, but I would like to see some common organization / setup across distributions and window managers.

RedHat's current move gives the user the same menu system across whatever Window Manager he chooses.  That's a step in the right direction, but not enough.  I think it should be standardized across distributions too.  

Why?

What do you say to someone who calls up and says "How do I get on the internet?"

If they are using Windows, you say "Goto the Start Menu, Click on Programs, then click Internet Explorer."

What if they ask the same question on Linux.  Well, you would probably have to start interrogating them.  "What Distribution do you Have?  What Version of that Distribution do you have?  What Window Manager do you have?  What programs did you install?  Do you see a program called "Mozilla" in the menu anywhere?  How about "Konqueror"?  Or " Opera", or "Netscape"?  Or "Lynx"?  Or ...."

And what if they can't find any of those programs?  Time to figure out a way to explain to an average user of Windows about installling a browser, on their particular version of their particular distribution.  Do they know what a "root" password is?  Do they have compilers installed?  What a mess!

Now, I'm not saying that each distribution have to install the same software.  But just use generic names and standard menu locations.  RedHat has done this in their latest release.  Getting on the internet is as easy as "Click the Menu, click on Internet, cilck on Web Browser".  I don't really care if netscape / mozilla / opera / konqueror / whatever starts up, they all work about the same.  And I don't think most users really care what pops up either, after all they just want to browse the web.

What are some of the other obstacles keeping "Joe User" from running Linux?

How about Printer support.  This is something else that is overly complicated in Linux.  I consider myself a fairly knowledgable linux advocate, but even I have trouble figuring out how to cancel a confused print job on my system.  

Package Management is another area.  People need an easy way to install (precompiled) packages and dependencies.  If adopting a single package manager across the linux world could happen, I don't know which I would pick.  RPM is nice, but not very convenient.  I haven't used APT in a while, but doesn't it automatically install missing dependencies?  Maybe APT with a nice easy GUI interface could be the answer?  Something where a user can say "Install an Instant Messanger" or "Upgrade to the Latest Web Browser" and have it done.  I dread having to walk someone through compiling a package and resolving dependencies, and I assume that most users wouldn't have the patience to try to handle it.

And just to reiterate a point made in other discussions, we don't want a dumbed-down version of linux, but a linux desktop that is accessible, useful, and acceptable to the masses.

It's easy to get fellow comp geeks to use linux.  People who get a warm feeling from digging through source code, and compiling customized versions of programs optimized for their CPU.  Building custom kernels to be as small as possible.  All that stuff, well if you know how to do all that you shouldn't be affected by a generic desktop, 'cause you'll probably customized the heck out of it anyways.

I think with a more centralized / standard desktop we would see more of the big software developers release linux apps.  I think many of them stay away becuase they don't want the hassle of trying to provide customer support for a normal user trying to grasp an understanding of "administrating" their computer.  Imagine a customer calling Macromedia about DreamWeaver-MX3 for Linux (wouldn't that be nice).  "Yeah, I just tried to install DW on my machine, but it's not in my menu.  How do I get it?"  "Well, what Window Manager are you running?"  "What?  I don't know, what are you talking about?"  Companies would get tired of answering support calls relating to the desktop that they probably couldn't answer, and customers would get tired of getting told "That's not a question for us, that's something to do with your distribution.  Contact the Window Maker / Debian Tech Support."  People would get fed up and go back to MS-Windows and never give linux another thought.

Back to the Window Managers.  It's clear that people don't want one window manager to dominate.  But why can't they share the same configuration and services?  Every (major) window manager has a menu system.  So why can't they share the configuration information for those?  I want my newly installed program to show up in my menu reguardless of whether I'm in KDE, GNOME, WindowMaker, XFCE, or whatever else!  Would it also be asking too much for them to share common configuration utilities?  A common print manager and other usefuly utilities?

And can I have a standard "DESKTOP" folder across window managers?  Where I can have folders for my documents, pictures, music, downloads, and other stuff?  (maybe that would be too Windows(TM)-like for some of you, I dunno ).

Standardizing things in Linux to the point that the average windows - user could figure it out is the biggest obstacle standing in the way of the ultimate adoption of Linux by users and Software Developers.

From a software developers point of view:  It may be ideal, even easier, to write software for Linux than Windows, but customer support is too prohibitive.  In fact most software currently available for linux doesn't really have tech support.  You have to trolll through IRC channels and mailing lists and try to find answers to your questions.  If I can't get "Konqueror" to work properly, who can I call to walk me through it?  

It looks like a lot of the problems are being addressed by the Linux Standards Base http://www.linuxbase.org/.  They have some of the biggest names in the Linux world backing them, including Caldera, Debian, RedHat, IBM, Intel, SGI, Oracle, and others.

Hopefully things will take shape soon.  While Linux Distros are suffering with central problems like standardizing menu layout, Microsoft is working hard to innovate / change Windows to set it apart from the Linux offerings so that it will be harder for people to see Linux as an alternative, but as 'Old Technology'.  "Why do I want to use something that looks / works like Windows 95?"  

Who knows, I look forward to the day I can install Linux on my mom's computer and she can just use it without help and hand-holding.  It's definitely not there yet.

This is a problem/advantage of Linux. While standardized desktops, menus, printer management, and software installation makes things easier and may attract more people, the biggest attraction for Linux in the past has always been "choice".

Some examples are KDE, WindowMaker, Gnome. These desktops approach desktop management differently and each has it's pros and cons. Some people like one while other people like the others.

Another example is RPM vs. Apt/Get vs. source code install. They all are different and have their advantages.  Source code install allows you to compile for your specific system for more efficient operation while Apt/Get and RPM are precomiled for generic systems.

Printer management seems to be settling on CUPS for a standard but there are still other choices out there.

Remember, one of the biggest complaints the Linux users have against MS is that MS forces you to use a specific desktop with a specific layout and GUI interface. While this makes it easy to help someone over the phone it doesn't allow different folks to enjoy different desktops.

Here's something to think about. What if all houses were layed out the same? It would be easy to help people over the phone making suggestions on how to move things around in the house or how to change something or fix that wall problem in the living room. But, on the other side people would get tired and ask why houses couldn't be designed differently. (That's where Linux comes in.)

  • We will now set up your display renderer [XFree86]...
  • Now lets set up your sound server...
  • Now we will configure your internet access...[/list:u]At each stage, an xml file is being built in /etc/.services.conf or some such that will tell *all* apps what to use for such services. The settings for each service could be kept in its own config file, just like now, though I would like to see those all move to XML as well.
    And it would be nice for the desktops to share menu configs as well. This being said, I think it would be a better solution for distros to go back to supporting a primary desktop, a-la RedHat/Gnome, Mandrake/KDE, Lycoris/KDE. I can, if I so desire, put alternate window managers on these systems, its just the config is up to me.  This is where the distros are *supposed* to distinguish themselves, anyway, isn't it? I've seen Mandrake and SuSE target the middle of the road, technically proficient power-user, while Gentoo and Debian are hard-core.  I'm not really sure *who* RedHat is for these days, but I would guess that it is the business folks.
    I guess that the point of all of this is that I really don't think that all distros should try and be all things to all people. Pick a niche and cater to that kind of user. However, I do think some behind the scenes compatability would be welcome.  Anyone want to hack the xml-config idea?
