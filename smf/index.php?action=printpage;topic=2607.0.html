Title: Sun 'Niagara' Based Servers
Post by: imnes on September 21, 2005, 07:54:51 am
New servers are coming out soon based on the new chips from Sun.  These are non-smp capable chips (SMP version will be out in about a year).  

They've taken the idea of multi-core / multi-threaded chips to the next level.  Each CPU houses 8 processing cores, each running 4 threads of execution.  This gives your system the capability to run 32 threads at once, whereas a typical CPU can run 1, some of the newer Intel / AMD chips can do 2.

http://www.theinquirer.net/?article=19423

(http://blogs.sun.com/roller/resources/jonathan/niagara_chip_small_pic.jpg)

(http://blogs.sun.com/roller/resources/jonathan/niagara_fireball_tm_small.jpg)

One of the interesting features pointed out in the article is that when running on Solaris you have the tools to partition the processor however you want.  

Quote
the same way you can partition a Sun 15K's processors. If you want two cores dedicated to your web server, and four to the JVM, and two others to the database, no problem, you can do it in the same way you could with multiple sockets.


They also mention  "It also has some inherent fault isolation but you can't mirror cores, yet."  Does this mean that part of your CPU could fail, the system sees it and works around it keeping the server running?  That's kinda neat.

Accoring to http://www.commsdesign.com/news/product_news/showArticle.jhtml?articleID=171000271 system prices start at $30k.
