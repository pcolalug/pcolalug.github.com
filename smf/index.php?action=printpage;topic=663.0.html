Well technically (is that spelled right?) you don't have to dual boot.  You could run apache+mysql+php on windows, and you could run an ASP server in linux (chilisoft I think).  Or even better, you could run linux and have it start up a vmware copy of win2k once it boots.  :)

But if you must dual boot...

You have 80g.  You could have gotten by with 6g or less probably.  So you could just split the drive in half give each OS 40g.  But to give yourself some convenience in the future, I would say give linux 10g, windows 10g, then have a 3rd partition of type FAT32 where your /home (and other handy locations) from linux go, and your windows web directories go.  That way if you have something screw up, you could reload linux, windows, or both, and have your web data intact.  Just a thought.

If it were my system, given your massive ram, HD, and quick CPU.  I would actually probably just partition all 80gb out to linux.  Do a base system install that does nothing but have your base software on it (and Xwindows).  Then actually run another Linux server, and your Win2k server from vmware.  That way you could restart / switch servers (or have both running at once) without having to reboot your machine at all.  And as long as vmware runs happily you could reload your vmware'd servers and really never have to actually touch the main software of the machine :)  

Of course I'm not sure how much vmware costs and if you'd want to buy more software.  Sorry, but I like doing geeky stuff like that.
