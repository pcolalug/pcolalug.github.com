Okay, shell scripting is easy, but hard. It is easy syntax wise, but you have to know what the other commands and programs do in the system.

Perl is nice, but can be rather overwhelming with its philosophy of "more than one way to get things done". One advantage of Perl though is, to quote rburt3, if all else fails, you can do it with Perl.

My suggestions (and my own plan of action) is to learn shell scripting enough to get some stuff done, then perl and then php. php can be used for system based scripting, but it really shines in webdev.

Another good scripted language you might want to look at, if you don't know any right now, is Python. It is supposed to be very simple and good for beginners. Green Tea Press (http://www.greenteapress.com/index.html) has an open book series How to Think Like a Computer Scientist. They have three versions; Python (http://www.greenteapress.com/thinkpython/), Java (http://www.greenteapress.com/thinkapjava/) and C++ (http://www.greenteapress.com/thinkcpp/). I plan on reading the Python version soon.

Green Tea also has a book that is currently in the works, Learning Perl the Hard Way (http://www.greenteapress.com/perl/). According to the author, it is aimed at someone who already knows how to program and wants to learn Perl and get to doing cool stuff quickly.

Post by: rburt3 on September 15, 2003, 06:21:39 am
Ok, I'll jump in...

Though I don't know it, really, I think Perl is the way to go. Its ubiquitous on all *nix systems (and easy to set up on others) and is independent of the installed shell (do the BSD's still default to csh or tsch?). It seems to be the sysadmin's "go-to" language.

PHP is nice and all, but you'd have to set it up yourself on many systems.

While bash is nice (why would you use anything else???), there's still no guarentee that it would be available on all systems you may wind up responsable for.

Perl may be a tough row to hoe if you haven't had much programming experience, and the myriad ways of going about expressing your algorithm can be frustrating, but stick with it and you'll have a skill that will *always* come in handy, no matter what platform you are working with.
