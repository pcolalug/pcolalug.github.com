I used to like KDE until I found a nasty bug in it was crashing Linux. I hope they fixed the bug by now but I haven't tried it lately. I found that when in KDE using an Nvidia card and drivers in a multi-desktop environment, KDE would eat up memory and not release it.

Here's what happened:

1. I, like many Linux users, run multiple desktops.
2. When I switched from one desktop to another, the memory used on the system would increase.
3. When I switched back to the original desktop, the memory would not be released.
4. Sooner or later, I would use up all physical memory and the system would go into a never ending memory swapping routine. This would lock up the mouse and keyboard. I would have to hit the reset button on the computer and do a hard reboot to fix it.

I tested and verified this problem in KDE and verfied that it did not exist in Gnome.

So, I switched to Gnome and haven't looked back.
