63  Hardware / Network / Verizon wireless on: March 11, 2009, 07:26:25 am
Wish I could claim some sort of technical expertise for this but I got a Verizon wireless card working on my laptop running Fedora 10 and it was totally by accident.

I got the Verizon wireless a year or so ago because I wanted my own Internet connection while travelling.  Verizon supports only Windows and Mac.  I used the PCMCIA wireless card on a Windows XP machine.  I think that when I installed the Verizon softwate on that laptop my userid, password, and Verizon phone number were written to the PCMCIA card.

Anyway, I had made several unsuccessful attempts to get the card to work on my Linux laptop.

This morning I was bored and I thought I'd give it another try.  On this machine you can disable built-in wireless with a Function + F2 combination.  For the first time I did that.

Then I inserted the Verizon wireless card in the PCMCIA slot.  The Great Penguin smiled upon me.  The machine established a CDMA connection without any configuration by me.  The connection is faster than a scalded dog.

65  Distributions / RPM Based Distributions / Fedora 10 on: December 16, 2008, 12:15:00 pm
Fedora 10 was released 25 November 2008 and I bought an installation DVD from Linux Central http://www.linuxcentral.com/_v3/.

Installation on my laptop was painless.  Connected to a home wireless LAN easily.  Nice assortment of applications --- OpenOffice, GIMP, game assortment, Firefox.  Slicker than owl poop.

One feature that impressed me is how easily I was able to access a couple of external storage devices:  A flash memory card with a PCMCIA card adapter and a USB memory stick.  They appear as icons on the desktop as 1.0 GB media and 256.6 MB media.  Files on the devices can easily be seen with the GUI file manager.

In a terminal they were located in a directory called "media" in the "/" (root) directory as disk and disk-1.

Things have come a long way from when we had to mount devices like floppies and CD ROMs from the command line.

68  Distributions / Security / Obscure hobby on: November 05, 2008, 12:15:12 pm
I've been a Pretty Good Privacy (PGP) user for years and since dabbling in the black arts of Linux one of my goofy obsessions has been getting Linux versions to work.

PGP has fallen into disuse since the Gnu Privacy Guard (GPG) came along and more and more distributions could care less if PGP worked with new distros.

I installed Open SuSE 11.0 recently.  Had trouble getting an .rpm for PGP installed.  Here is a link to what I had to do to get it working:

http://forums.opensuse.org/applications/397874-installing-pgp-6-5-8-a.html

Basically, I had to hunt down and install some missing /lib files.  This is no biggie for you wizards in Linux but for someone in the amateur/wannabe skill level, like me, this is a big deal.

This is of little interest to anyone but that has never slowed me down before.
