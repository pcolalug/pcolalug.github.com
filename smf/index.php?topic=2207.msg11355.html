Aight guys... it's been quite a while since I've been here due to moving and having a baby, so things are still kinda hectic.  I remember when I first came here with a simple question that you guys answered with great expertise.  Time for a little more complicated question.

I just downloaded and installed Mandrake 10.1 Download Edition on my new PC.  
Specs:
AMD Athlon XP 2000+ 1.67GHz processor
512MB PC2100 DDR
Asus A7N8X-X nForce2 mobo - Onboard nForce2 LAN and AC'97
MSI StarForce nVidia GeForce FX 5200 128MB 8x AGP
Mandrakelinux 10.1 - 2.6.8.1-12mdk kernel

Well, I decided to go to nVidia's Website and download all the latest drivers available.  After some issues and about 2 hours of frustration and Googling, I fixed the issues with the display drivers.  When I went to install the LAN/Sound drivers, I started having even more issues.  I finally resolved the LAN issues, but now I can't get my sound to work.  I've followed the instructions to the T, but I now only have sound when using a Media Player that can use the Xine Engine for output.  This is my modprobe.conf file:
# This file is autogenerated from /etc/modules.conf using generate-modprobe.conf command

# alias eth0 forcedeth **
# alias sound-slot-0 snd-intel8x0 **
remove snd-intel8x0 /sbin/modprobe -r snd-pcm-oss; /sbin/modprobe --first-time -r --ignore-remove snd-intel8x0
install snd-intel8x0 /sbin/modprobe --first-time --ignore-install snd-intel8x0 && { /sbin/modprobe snd-pcm-oss; /bin/true; }
install usb-interface /sbin/modprobe ohci-hcd; /sbin/modprobe ehci-hcd; /bin/true
alias eth0 nvnet **
alias eth1 8139too
alias char-major-195* nvidia
alias sound-slot-0 nvsound **

All the **s mean I changed those and only those lines.  That's what the installation instructions say to do.  When X starts up, I get this error:
Sound server informational message:
Error while initializing the sound driver:
device /dev/dsp can't be opened (Cannot allocate memory)
The sound server will continue, using the null output device.

Any ideas??

I would guess that the module "nvsound" isn't loaded (as referenced in the last line of your /etc/modprobe.conf file).  Use the "lsmod" command to see if it is loaded.

If not, (as root):  modprobe nvsound

If your sound then works, add "nvsound" to /etc/modprobe.preload

I did a little research and it's something so easy I didn't even think about it.  It seems that the sound drivers are OSS drivers not ALSA.  Nvidia says they're working on an ALSA driver, but "development can only be done when time is permissable."  As far as the video drivers, that was a total pain in itself.  I use and love my nForce2 sound in Windows.  The only other option is to use the open source driver for an Intel chipset which happens to be compatible.  Oh well.  It'll suffice until Nvidia releases the ALSA driver.
