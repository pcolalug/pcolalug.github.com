Title: need help installing a .gz file
Post by: rugmonster on October 31, 2004, 04:04:50 pm
I'll assume the file ends in tar.gz. Try this in a Terminal.

Code:
tar -xzvf <filename>.tar.gz


Here is what the switches mean:
    x = expand
    z = gzip compressed
    v = verbose
     f = filename[/list:u]
    Tar files are like zip files without the compression. gzip is what provides the compression. There is also bzip2 for compression, usually identified with tar.bz2. You can use -xjvf for those files.

    I don't know if you've learned about manpages yet, but any command you use, should have an associated manpage. You can use the following command to find out what the syntax and options are for any command.

Code:
man <command>


There are GUI tools for dealing with archives such as this, but they vary so much depending on your distro and environment that it's just easier some times to give the command line info.

Title: need help installing a .gz file
Post by: rugmonster on October 31, 2004, 08:53:28 pm
When you work in a shell, you are usually dropped in your home directory (something like /home/username). Apparently, this file is on your thumb drive, so you will have to change your working directory to the mountpoint for that device. Unlike windows where you have seperate drive letters for different physical and logical drives, Unix filesystems work as a single hierarchy. You "mount" a filesystem within the single root. To find out what filesystems are currently mounted and where you can type the following:

Code:
mount


This will give you a listing of what is mounted, where and what type of filesystem it is. Since this is a thumb drive, look for one of those lines that says fat32. You will then want to copy that file from the thumb drive to your home directory unless you want to mess with permissions. So, let's say your thumb drive is being mounted at /mnt/flash. To copy the file over, you would do something like this:

Code:
cp /mnt/thumb/scanmodem.gz /home/ktshaffer


What this does is copy the file residing at /mnt/thumb with the name scanmodem.gz to the directory /home/ktshaffer with the same name as the original file. Keep in mind that Unix shells are case sensitive, so pay attention to what is capitolized and what isn't.

I would check out this (http://linmodems.technion.ac.il/first.html) documentation on gunzipping that file and running the program. Not to be a jerk, but I found that directly off the linmodem.org site. One thing that you have to realize is that most linux users can be elitists and will tell you to RTFM. I'm not saying that, but there is a wealth of information available between manpages (once you figure out the art of reading them), Google, the sites of the project you are working on and the Linux Documentation Project (http://www.tldp.org/). There aren't too many things out there that you will have to reinvent the wheel on at this point in the game, and chances are, someone has written a primer or HOWTO on it. I suggest you spend a few minutes reading up on stuff and if you are still stumped, let us know how we can help you.

Before you get much more into things, may I suggest you check out the Introduction to Linux (http://www.tldp.org/LDP/intro-linux/html/index.html) from the Linux Documentation Project. I think you'll understand things a lot better and have a better idea of what's going on with your brand new install.
