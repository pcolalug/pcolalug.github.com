Title: Ruby on Rails
Post by: imnes on January 25, 2005, 09:07:23 am
I went through some tutorial stuff last night, got my web server at home setup and working with rails.  It's pretty slick.  I still think Python is cleaner / easier than Ruby.  Ruby has two syntax issues I personally don't like:

1.  Name prefixes to determine context.  So you end up with variable names like @myvar @@myvar

2.  I don't like underscores, and a lot of the ruby API and add on libraries are full of function_names_with_underscores.

But otherwise it's pretty nice :)

I was thinking about how to write large web application in Rails as far as laying out the thing.  I will be building a site that has a bulletin board, photo album, and file server area.  They all should share common login, and have access to the users session, etc.  The general concensus in the ruby rails IRC channel was to lay out each piece of the site as a seperate rails application.  Bboard is one app, photo album is  a seperate app.

One of the issues I was thinking about is how can I get those to share common configuration files (like the database config) so it only has to be changed in one place?

Also rails configures html layouts within the application, so I can't have a single site-wide template?  

I'll need to dig in deeper to figure this stuff out.  But getting the server setup and a minimal app working with the tutorial took about an hour total.  

Anybody else playing with this stuff?
