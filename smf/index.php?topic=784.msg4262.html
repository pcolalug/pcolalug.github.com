Debian boasts one of the largest single collection of packages available for a distro. Yes, RPMs are quite numerous, but Debian has the largest central collection. Those CDs are to get every stinkin' last one of them (available at release time). My suggestion is to avoid those completely. There are boot disks and even a network only ISO to install you system. I have always used the network only ISO since it is only 40 or 60 megs and I am only downloading what I am installing. This puts less load on the mirrors and costs them less in the long run. Also, the packages you install from the CD no doubtly be updated the first time you run apt-get upgrade.

I would love to help anyone who is interested in learning about Debian. I think it is going to go back on my other two systems (possibly three now that I have an AUI to RJ-45 transceiver and console cable for the sparc) and maybe Gentoo onto the main workstation. I love Debian for its package management, but Gentoo is great as well and bleeding edge. The only downfall of it is the stinkin' compile time.

Debian offers you several different archives: stable, testing, and unstable
With oldness decreasing in that order.  The debian folks are very conservative with what
they call 'stable'.  If you really want a rock solid, no worries, machine stable + the security source

Okay, Debian is (in)famous for being extremely stable. Stable usually doesn't mean bleeding edge. What you can do is change which version of the distribution you are getting. A lot of people like to run either testing or unstable. Things may break if you go up, but then again, unstable is the level of what most other distros are running at...so it is up to you.

You also have the option of looking around the net for other apt sources that have newer packages compiled for Woody and Sarge. You would just add those sources to your /etc/apt/sources.list.

Now, to change to a newer version of the base distro, you need to go into your sources.list. It should look something like this:

Code:
deb http://ftp.us.debian.org/debian stable main non-free contrib
deb http://non-us.debian.org/debian stable main non-free contrib

See where it says "stable"? Change that to either testing or unstable. Also, should you find other repositories for apt-get, you can add them here. Some sources, such as the deb-src and security site may need to be commented out if you go up to unstable (or that is what I just found yesterday).

Now that you apt has a new place to look, you need to update the database of available packages.

Code:
apt-get update

Update the database and...

Code:
apt-get dist-upgrade

Updates your packages for the new distribution.

Code:
apt-get upgrade

Updates most of your packages. This is what you will use to keep your system up to date. Some packages aren't touched by this, so you may want to run apt-get dist-upgrade from time to time to upgrade those packages.

Code:
apt-cache search <keyword>

This is one command I use constantly. If you want to install a package and know the name of the program, you can find the correct package name with this. There are a lot of packages out there, so to wade through them all by hand can be a pain. Find the package name here and then install it with...

Code:
apt-get install <package> <package> <package>

The beauty of apt and debian. You can specify multiple packages here and every single dependency is taken care of for you. I can't express how much I love apt. Yesterday when I was setting up a debian server, I rediscovered that love. You will get spoiled. You'll understand why I hate rpm's so much soon.

There is also the program aptitude which is supposed to be pretty nice. It is X based, but since I don't run X on my debian box, I can't say much about it. Let me know if you have any other questions. Hope this helps.

Another update:  Got the runlevel thing figured out, with some IM help from Ruggy and some IRC talk.  In case anybody cares, what I learned is this:

Debian uses runlevel 1 for single-user.  Levels 2-5 are all multiuser, and appear to be exactly the same to me.  Level 6 is for reboot.  It defaults to runlevel 2, so I removed the S99gdm and S99xdm scripts from the /etc/rc2.d directory, and it now boots to the CLI.  If I want to use Xwindows, I just use "startx".  This is somewhat different than what I'm used to (Mandrake).  Anyway, nice learning experience and there ya have it.
