Title: What makes Escapade different
Post by: erc on February 20, 2003, 07:12:35 am
Since I wandered into this topic, I feel duty-bound to reply ;)

Why yet another scripting language?  suppose you're a newbie - you know some SQL, you know some HTML, and you want to do dynamic pages.  OK, now what?  ASP, PHP, Perl, etc. are, sad to say, horribly complex, and the ramp-up time is considerable.

Escapade, on the other hand, is EASY.  I'm not going to give you the marketing speech about how it's fast, simple, quick, and easy ;)  but I will say that it has considerable advantages over most other scripting languages.  Plus, we're working on a compiler for it - it's operational, although it doesn't have all the stuff that the engine has yet, the ultimate goal is for you to be able to take an Escapade page, run it through the compiler, and come out on the other end with an executable.

If anyone wants to try out the bleeding edge engine, let me know and I'll give you the URL to the super-secret download site ;) - the one up on the web site is getting old, and so we're preparing for a major new release.

In addition to the engine on the web site, the new beta engine has support for POP and SMTP commands, so it makes it pretty easy to write a web-based mailer.  The first bare-bones mailer I wrote with the new engine took me a couple of hours to code and test, and it took about 5 lines of Escapade code to do it in.  the new engine also has support for ASP-like sessions (which makes life a LOT easier!), and what we call preferences, which is just another word for named sessions.

Yes, the engine is free.  :)  We've got current versions for Linux and FreeBSD, and an older port to Windows.  The Windows port is a bit painful, which is why I'm leaving it for last :P

Title: sample code
Post by: erc on February 20, 2003, 08:27:40 pm
Sure:

#
# Escapade code to pull records from a database and display them on a page
#
<html><head><title>Records in table 'blah'</title></head<body>
<DBOPEN "dbname", "hostname", "login", "password">
<IF "$ERRORTEXT" NE "">
 Error opening database: $ERRORTEXT
 </body></html>
 <STOP
</IF>
<table><tr><th>field1</th><th>field2</th></tr>
<SQL select * from blah>
 <tr><td>$field1</td><td>$field2</td></tr>
</SQL>
</table></body></html>

Another example:

#
# Escapade code to display messages in a POP mailbox
# Gathering login information is up to you :)
#

<POPOPEN "hostname", "login", "password">
<IF "$POPERROR" NE "">
 Error opening mailbox: $POPERROR
 </body></html>
 <STOP>
</IF>
# include a style sheet, if you want...
<INCLUDE "emailstylesheet.css">
<POPMSGS MSGNO>
There are $MSGNO messages in your mailbox<br>
# Show messages in reverse order, last to forst
<FOR I=$MSGNO TO 1 STEP -1>
# just fetch the header at this point
 <POPHEADER $I>
 <IF "$POPSUBJECT" EQ "" THEN POPSUBJECT="[no subject]">
 $I $POPSUBJECT $POPFROMNAME ($POPFROMADDR) $POPLINES $POPDATE<br>
</FOR>
# reset POP so the messages don't appear read
<POPRESET>
<POPCLOSE>
</body></html>

How's that?
