The GUI.  Nothing concrete. I would definately say it is in line with Win2K as opposed to XP. Maybe somewhat in between. Again, nothing concrete and really just the interface.

What don't I like about Mandrake? It could be a drastic misunderstanding of Linux in general, but sometimes when I install RPMs half of my menu items disappear along with any icons (except one generic icon for everything). I tried a Windows trick that takes care of it though. I start like I am going to reinstall the system and then tell it to Upgrade as opposed to Install. When that happens it will rebuild the RPM database and clean things up. I would suspect there is an easier way to do that, but it works for me.

Little things like that bother me. Other little things like that happen. I am sure it happens everywhere in Linux. Who knows, if I was using something a little less familiar looking, I might not have my system back up at all.

Another thing, much of the documentation seems to be written for Red Hat. Interestingly, I am installing a very complicated piece of software right now. It has tons of configuration options. Even though the installation/configuration pages say that Red Hat is the worst system to run this package on, it gives details for using Red Hat. As if Red Hat users are the only ones that need detailed information.

So as a Mandrake user, I feel somewhat slighted by people catering to Red Hat. And even though they are somewhat similar (more before than now), where they place files by default is different, so digging through documentation for RH may not get you to the file you want in Mandrake. Of course, if you use anything other than that, then you probably have little to no help. But smarter people than I run Slack and SuSE.

I think Karen and the crew will be able to work with Mandrake well.
