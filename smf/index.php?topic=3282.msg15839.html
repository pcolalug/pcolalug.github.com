For a server, I would suggest either Debian stable, Ubuntu LTS (8.04 will be the next LTS release), or CentOS/RHEL 5.1. OpenSUSE is a little too flaky for my tastes. I list those because they all provide good, stable platforms. You'll sacrifice the bleeding edge versions for the apps, but I will take stable over new for something I need to be up.

Quote
- What are some free and easy to use virtualization software programs?

Either VMWare Server or VirtualBox. I've moved to using VirtualBox because it seems significantly faster than VMWare Server. I've heard that VMWare Server is purposely held back to encourage people to use VMWare ESX.

Quote
- What are some good web sites, and other reference sources, that can help "dumb down" the process of setting up a virtual environment?

CentOS, Debian, and Ubuntu all have support for Xen, but that is mostly good for doing Linux virtualization. If your processor supports virtualization, I've heard you can run Windows as a guest, but it may not even be worth it. Just keep in mind that you can only use one virtualization technology at a time. Whatever you pick, make sure you keep that in mind and that you likely won't be able to take a VM from one to another.

Quote
- Any other useful information you can think of?

Memory and disk I/O are going to be your biggest constraints. Make sure you've got plenty of memory and if you aren't running RAID now, you might want to look into it. RAID 5 only requires three drives and provides a good balance of redundancy and performance, even using mdadm software RAID.

What is the ultimate goal for your setup? What are you going to be doing with it all? I've recently moved away from using a Win2k3 AD VM server in favor of open-source, linux based solutions. If you're doing some .NET development and need IIS and MS SQL Server or just want to learn/play with AD and the various Windows server tools, I understand. But you may not need Win2k3 and I would be more than happy to recommend some open source options if you have some specific goals in mind.

Quote from: djoed_pcola on January 08, 2008, 11:00:53 pm
For example, the difference between a Virtual Machine and a "Guest Addition".

The guest additions are basically a set of tools that are loaded on the virtual machine for extra functionality, such as allowing you to move your mouse into and out of the guest's GUI environment without having to "capture" and "release" it. Additionally, VirtualBox has a shared drive that allows you to share files between the host and guest, but you need the guest additions installed to use it. It's mostly drivers and enhancements to make using the virtual environment better.
