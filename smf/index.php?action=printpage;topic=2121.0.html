Title: [Solution] ET Won't Start
Post by: dpeach on December 18, 2004, 08:43:39 am
Found a solution for why sometimes your ET (or any game based on the Quake3 engine) will decide to not start. It has to do with the number of maps that you have loaded.

Here is the original post found at: http://www.linuxquestions.org/questions/showthread.php?s=&forumid=14&threadid=181375
Quote
I had this problem earlier and after much searching found an answer. I think it applies to any game using the quake3 engine under linux, and doesnt seem to affect any windows versions running on windows (and presumably wine).

I started the game up and it errored and crashed. So i ran it again from console to see what the **** was up with it and got this error message:

_X11TransSocketOpen: socket() failed for local
_X11TransSocketOpenCOTSClient: Unable to open socket for local
_X11TransOpen: transport open failed for local/Shadow1:0
_X11TransSocketOpen: socket() failed for local
_X11TransSocketOpenCOTSClient: Unable to open socket for local
_X11TransOpen: transport open failed for local/Shadow1:0
Received signal 11, exiting...

After much searching, i finally came across the answer on a q3a forum.

Its to do with the number of maps/pk3 files you have in your etmain/baseq3 dir. If you have more than 300 maps, paks, etc. It will crash. i havent found out why yet, but removing a few maps solved the problem.

So i thought id post this up on here since others will no doubt come across it from time to time.
Worked for me. I only had 297 maps when it stopped working. I pulled out 5 of them and it works great.
