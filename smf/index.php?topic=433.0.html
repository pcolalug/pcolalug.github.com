Ok, here's the thing.

The Forum software has a timer. If you don't do anything after a certain amount of time, it logs you out. This is because some people, rather than logging out, just close their browser. If they come back and try to log in from a different IP it will show them as already logged in. So, PHPbb has chosen to log out stagnat connections.

This is just the way things work.

My suggestion: If you aren't actively browseing the forums, Log Out.

My problem is definitely looking like a cookie issue.  When you write session code in PHP it checks to see if your system supports cookies.  If it does, then your session id gets stored in a cookie.  If not, it rewrites all the url's to include &sid=948234bbsafd23523 or whatever your session ID is.  

If cookies are working, then the login page will read the cookie and auto-log you in (if you checked that option).  But without cookies when you first go to the login page it has no idea who you are 'cause it hasn't written an &sid=xxxxxxxxxxx onto the url yet.  

At least that's how I am perceiving the problem.  Maybe the code it's using to check for cookies isn't working for certain IE browsers?  This is 6.0.2800.1106 128-bit SP1 on NT4.0 SP6a.

For those of you who the auto-login feature is working, do you have &sid=xxx in the address bar of your browser?

I'll play around with it some more on my end and try to verify this.
