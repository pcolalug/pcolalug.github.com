A Very Much Biased, but Brief Vista Review
« on: December 31, 2006, 12:40:22 pm »
I got my grubby little hands on a copy of Vista and installed it this morning. Underwhelmed sums it up. Yes, it has some pretty little added animations, translucent window borders and a few other things, but for the most part, I haven't found anything making it worth the upgrade from XP. I was constantly nagged about running this application or installing another application. When I need to do something such as add a user or make a minor change, I would be asked if I wanted to do that. Conversely, IE7 prompted me to install add-ons, but wouldn't tell me what it was wanting to install. The settings are spread out far and wide, so finding network settings requires a trip through multiple windows. That is pretty much the case for most everything. Application compatibility is another thing that got me. I install Azureus, but when I ran it, I got a missing dll error followed by a notice that it wasn't allowed to access something in the JRE. Oddly enough, Eclipse ran fine. I was able to get release level drivers from Nvidia, but Creative only has beta drivers for the Audigy ZS line.

All in all, it really left me with a feeling of disappointment and relief. Through their over complication of doing most anything useful with your system, constant nagging and less that impressive UI, I am relieved that the linux community may grow even more with desktop users. There is so much more impressive things coming out of the OSS community that I'm comfortable to say we're already ahead of MS in the OS/UI arena. The current Linux kernel, Xorg/AIGLX, Xgl, Beryl/Compiz, Gnome/KDE already provide all that Vista has to offer and more.

We've been using a copy of Vista beta 2 (or recently 3?) for quite a few months now at work.

Underwhelmed is an understatement.  I've yet to see any real benefit from using vista.  My biggest complaint is the ram usage of vista.  We have it trimmed down, but it still uses 350-400 meg of 512 ram on the system.  This leads to "vista bugs" in the software we test, where our code crashes simply from the lack of ram on the system (yes, our code is in java  )

IMO each release of an OS should not be significantly more bloated than the last.  However, with windows this seems to be the case.  Windows 2000 and XP required about 128 and 256mb of ram respectively to run decently on a system.  Vista on the other hand seems to require 800 to a gig to be usable.  Given that I've seen no *real* usability improvement since 2000 (with the exception of wireless drivers in XP), how in the world can they justify this?  I currently have windows server 2003 nestled on a small partition on my old hard drive.  The last time I checked, it used 128mb of the 512 I had on the system.  This is proof that microsoft *can* produce a system that is both functional and light weight, but they choose not to.  It seems they are in collusion with the hardware industry to boost sales of new computers.  Of course, given the fact that a computer from 8 years ago can still happily chug away checking email, and writing documents, I can see where the hardware vendors would be concerned.

OTOH, I have a linux / fluxbox install on my parents machine.  Ram usage?  A whopping 30mb  .
