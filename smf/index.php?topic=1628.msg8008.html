I know the nvidia video drivers work in Slackware 9.1 because I've installed it myself.  (You are on slack 9.1 right?)

From the console, run uname -a and make sure the linux version of the running kernel exactly matches the sources you're compiling against.  

You might have a symlink pointing to the old kernel sources something like:

/usr/src/linux -> /usr/src/linux-2.4.22

Remove that symlink completely, that's what I do and it's never caused me trouble.  

But I suspect there's a mismatch in your running kernel and installed sources version, because the driver installer should detect and find the correct sources installed without you having to specify.

Quote from: SiliconFOXX
Anything I can put up that might make it easier to understand where it's screwing up or what's going on with the system?

When you run the Nvidia installer, does it complete successfully?  Do you get any error messages from it?  What it should be doing is telling you that there is no precompiled module available, and offer to compile one for you.  When you say Yes to that, it should take a few seconds (about 10-12 seconds) to compile one and it'll tell you that it was completed.

If this doesn't happen, you probably don't have the source code installed.  When you do a "uname -a" to find the running kernel version, it must match EXACTLY with the contents of the source code directory in /usr/src.  If it doesn't you don't have the source installed (or it's the wrong source).  If you'd like to, post the output from "uname -a" and from "ls -al /usr/src".
