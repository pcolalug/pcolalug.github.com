Title: Comparison of XFS, JFS, ReiserFS and EXT3
Post by: rugmonster on April 22, 2006, 10:27:18 am
I found a recent benchmark (http://www.debian-administration.org/articles/388) of the top four Linux filesystems, XFS, JFS, ReiserFS and EXT3, at Debian-Administrator.org. Their conclusion is that XFS is the best overall filesystem for a home or small-business file server.

    * It uses the maximum capacity of your server hard disk(s)
    * It is the quickest FS to create, mount and unmount
    * It is the quickest FS for operations on large files (>500Mb)
    * This FS gets a good second place for operations on a large number of small to moderate-size files and directories
    * It constitutes a good CPU vs time compromise for large directory listing or file search
    * It is not the least CPU demanding FS but its use of system ressources is quite acceptable for older generation hardware

What is your experience with these or other filesystems? I personally use EXT3 for my root, boot and main storage partitions (mostly media files). I then use Reiser3 for usr, var, tmp, opt and home since it has always been touted as been best for small files. This article suggests otherwise, though.

Title: Re: Comparison of XFS, JFS, ReiserFS and EXT3
Post by: Brent on April 23, 2006, 02:44:13 pm
I use XFS.  I have alot of movies and mp3's floating around, and XFS *does* handle them faster than other file systems.
Overall, XFS and rieser feel pretty equal.  XFS mounts faster, and handles bigger files better, but otherwise there is little difference.

Those of you who use multiple partitions, or obscess about your pc booting in 20 seconds (like me  ::)) might find it useful.

As for EXT3, It feels painfully slow to me, but it is easy to recover data.  If I was hosting a server with priceless data, I'd go EXT3 all the way...BUT, if you're relying soley on your file system to recover data and excluding golden rules like daily backups, raid, etc...your data must not be that important. 

For a home desktop, imho your better off going with a faster alternative. 


Title: Re: Comparison of XFS, JFS, ReiserFS and EXT3
Post by: green_dragon37 on April 23, 2006, 03:21:56 pm
I've got a single XFS partition, the rest is ReiserFS.  The XFS houses my qemu images (files weighing in at gigs and gigs apiece,) and runs really well.  The ReiserFS works really well on the /usr/portage stuff, but NWN actually has/had issues with ReiserFS, you had to mount the partitions with a special option to make the game load faster than 2-3 minutes!

As for ext3, it is tried and true, I use ext2 for /boot, never gets mounted anyways.  I have the same opinion on using a filesystem solely on the idea that you can recover data easier.  If it's important, back it up!
