Title: /usr/local/bin versus /usr/bin
Post by: rburt3 on October 07, 2004, 11:56:44 am
This (http://www.pathname.com/fhs/) may answer some of your questions.

One thing to keep in mind is that Linux, like its inspiration, Unix, is designed from the ground up as a multi-user OS. This would imply that there's an administrator over-seeing one or more boxen that have one or more users. In most of our cases, we're both the admin and the user. Anyway, the file heirarchy is set up to facilitate permissions and restrictions on who can do what to the machine. In the case of /usr/, this is where "userland" stuff should go. Programs and data that all users should have access to go somewhere here. I think that as time went by, the heirarchy got so granular that /usr/bin was for user-space programs that facilitated other programs, while /usr/local/bin was the place for applications (which could in turn use the software in /usr/bin). May be wrong, and certainly not a clear explination, but my memory ain't what it used to be.

Title: /usr/local/bin versus /usr/bin
Post by: danc on October 07, 2004, 02:46:10 pm
My understanding is this:

/usr/local is where stuff goes that is installed by the distro during initial installation.  Also stuff that is installed later by whatever package manager you're using (RPM, TGZ, etc).

/usr/local/bin is where stuff goes that doesn't fit the above description.  Good examples include game installers like ET or UT2004, which are not packaged into RPM or TGZ packages.  Also stuff that you compile yourself will usually want to go here.

On a related tangent, the /usr/sbin directory is normally for "system" related programs/utilities, many of which can normally be run only as root.

That's how my systems seem to be organized, YMMV.

Title: /usr/local/bin versus /usr/bin
Post by: rugmonster on November 23, 2004, 02:17:22 am
As I took it from the Filesystem Heirarchy...

/bin is for the essential programs for all users and admins to use. These are the programs required to basically have a functioning system in single user mode with only / mounted. It should only contain things to get your system up and /usr mounted.

/usr/bin is for all of the rest of the programs installed by your distribution. Upon initial setup, and packages installed after initial setup, the actual binaries should go in here. Think of a grouping of like architecture systems sharing /usr. You could mount a single /usr and then mount a local filesystem on each one of those systems as /usr/local.

Enter /usr/local/bin. This is where locally installed binaries are installed. Say there isn't a binary package for your distribution for a program, so you roll your own from source. Generally, you will find the --prefix default to /usr/local, and all libs, manpages and binaries are placed in their respective directories.

Hope this helps.
