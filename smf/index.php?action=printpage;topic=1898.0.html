Title: 64-bit Linux
Post by: steve on October 06, 2004, 07:06:36 am
I'm running SuSE 9.1 64 bit on an Athlon 64 bit  machine. Here are the pros and cons.

Contrary to previous posts, speed is the number one benefit. With the kernel compiled for 64 bit and the apps compiled for 64 bit, it flies.

However, because the kernel, libraries, etc are all compiled for 64 bit, it's a real pain in the butt to get a 32 bit app to run on it. You usually have to recompile 32 bit apps to point to the new locations for the 32 bit libraries since they are in a different location or have a different name. (64 bit libraries and 32 bit libraries can't be located in the same directory and have the same name... :D ) Also, many 32 bit apps will not compile at all on a 64 bit system. Oh, and except for the most popular apps, it's hard to find 64 bit versions of apps right now.

My recomendadtion is, that until 64 bit becomes more popular, go with a 32 bit version of Linux. You may sacrifice some speed because you don't have the 64 bit version of Mozilla or Firefox (Which can't work on a lot of sites because there's not a 64 bit flash plugin and other proprietary plugins) or the 64 bit version of XMMS or Evolution, but you will be able to run pretty much anything you want.

Steve
