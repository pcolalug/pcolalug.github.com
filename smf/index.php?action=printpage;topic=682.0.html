Title: ? about apache 2.0

You should be able to use the "Listen" directive in your configuration file like so:

Listen 8000

This tells Apache to listen on port 8000. There should be a Listen directive somewhere in your config file(s) already that is telling it to listen on port 80. If you want Apache to listen on your alternate port instead of the normal 80, you can simply change the existing Listen. If you want to listen to your alternate port in addition to 80, just add a second Listen directive.

Once you update the config file, make sure you restart Apache for the changes to take effect.
