Sounds like you installed LILO to the MBR?  You can try installing LILO straight to your root partition.  Then use MS-DOS's FDISK to recreate your MBR

In Linux (maybe you can do it from dos) set your root partition as the active / bootable partition.

Is your active partition the one that linux is on?  I think even dos's fdisk can tell you that, just go into it and hit 4 (or whichever key lists your partitions out), and one should be marked active.  Same if you're in linux, run cfdisk and see which one is active, or fdisk and type 'p' I think to list your partition tables and it should tell you.  Anyways, you need to be sure that your root partition (or whichever one lilo is going to) is active.  Assuming that is done correctly you should be able to boot into linux from lilo.  Can you do that or is that not working as well?

Assuming that is working, and you have rebuilt your MBR following the directions above from a dos boot disk, you might even need to restore your windows system files if something got screwed up, that's easy enough.  Assuming you have a boot disk and your windows is installed to C: drive type

A:\> SYS C:

If you still can't get anything working then I give up.  It sounds like all your data is still on the drives so I doubt your partition table is screwed up.  

If you go the reload route and you want to dual boot install windows first and then install linux, that's usually the easiest route.  When installing linux tell it to install lilo to the ROOT partition NOT to the MBR.  That will save you from having to troubleshoot this again (I hope).

I don't know that I've ever had to reload to fix a problem, but I know from experience I would sometimes rather just spend an hour reloading instead of 3 days figuring out how to fix something
