Title: Script function
Post by: bhoff on May 20, 2003, 08:28:22 pm
Have you ever had problems keeping track of everything when you are making a compile. Or ask somebody what they where doing and how they where doing it. Well here is the answer you have been waiting for.
before starting that next make or have that other person do this before making a mess.
type script at the command prompt.
you get script started , file is typescript.
you now have a handy file where every thing is echoed...every key stroke.
every return, every error or message.
after you are done just ctrl-D to stop and you have your file in that immediate directory called typescript. I have already used it several times to help me in documenting several projects.
Give it a try and see!

You can give the file a name when starting by using
script filename at the start.

Title: try this
Post by: bhoff on May 21, 2003, 10:07:58 am
#!/bin/sh

# Public Domain

# Put  CTRL-M in $m and CTRL-H in $b.
# Change \010 to \177 if you use DEL for erasing.
eval 'echo m=M b=B | tr 'MH' '\015\010'
exec sed "s/$m\$//
:x
s/[^$b]$b//
t x" $*



Hope this works for you...May need a little tweaking
