Here's a dinky little program that will tell you the size of variables according to your environment/compiler.


Code:
/********************************************************************
sizeof.c -- Program to tell the size of the variable types available
to the compiler.
*********************************************************************/

#include <stdio.h>

main()
{
        printf( "A char is %d bytes", sizeof( char));
        printf( "\nAn int is %d bytes", sizeof( int));
        printf( "\nA short is %d bytes", sizeof(short));
        printf( "\nA long is %d bytes", sizeof(long));
        printf( "\nAn unsigned char is %d bytes", sizeof (unsigned char));
        printf( "\nAn unsigned int is %d bytes", sizeof (unsigned int));
        printf( "\nAn unsigned short is %d bytes", sizeof (unsigned short));
        printf( "\nAn unsigned long is %d bytes", sizeof (unsigned long));
        printf( "\nA float is %d bytes", sizeof (float));
        printf( "\nA double is %d bytes", sizeof (double));
        printf( "\nA long double is %d bytes\n", sizeof (long double));
        return 0;

}


In my case, since I'm on an AMD64 system, a few of mine are bigger/different than they are on an x86:

Quote
A char is 1 bytes
An int is 4 bytes
A short is 2 bytes
A long is 8 bytes
An unsigned char is 1 bytes
An unsigned int is 4 bytes
An unsigned short is 2 bytes
An unsigned long is 8 bytes
A float is 4 bytes
A double is 8 bytes
A long double is 16 bytes

Quote from: jaymz_720 on February 17, 2006, 12:46:51 pm
      I don't think pi was calculated on a 32b either.

It can be calculated on anything from 4bit+. Keyword is "calculated", not stored in a variable. For your project, don't rely on the precision of the variables, but examine the method of how you would determine the correct answer. You'll probably find that all you need are simple integers and allocate space to store the result. As you keep iterating through the process refining the answer to higher precision, you'll more than likely either dump the result to screen, or store in a char type location. With this method, your precision is only constrained by the amount of memory you have.
