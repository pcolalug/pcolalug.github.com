Title: Sound Problems in Debian Unstable
Post by: imnes on December 04, 2004, 01:47:06 pm
For the error:

alsamixer: function snd_ctl_open failed for default: No such file or directory

when trying to run alsamixer, on debian unstable with 2.6 kernel.  Other symptoms - sound drivers are loaded okay but no sound plays back, no mixers can access the cards / or can see any cards.


Okay my fix is this:  

First edit /etc/group and /etc/group- and add your user accounts and root account to the audio group.

Then stop alsa with /etc/init.d/alsa stop

Now run /usr/share/alsa-base/snddevices

Now restart alsa: /etc/init.d/alsa start

Now you can use alsamixer (maybe any other mixer) to unmute your channels and adjust the volume.  


This needs to be implemented in a startup script to be redone every boot up, pretty easy: edit /etc/init.d/alsa   In the start() section, right after the printf "Starting ALSA..." add a line

/usr/share/alsa-base/snddevices

Reboot and test.  Make sure sound is unmuted in alsamixer if you have that (there should not be MM over any of the channels), or use your KDE or Gnome mixer, they should work now.
