Post by: Brent on January 31, 2005, 11:50:59 pm
Woah, take it easy.

In windows package management is handled by each individual developer.  The maker of the software includes their own 'install' and 'uninstall' programs.  Thinks are a little different in linux.  Each distrobution has a package management system which handles the install, uninstall and upgrade functions of the software.

The package manager for your distrobution is called the Synaptic package manager.  You should be able to access it under your kde menu.  If that doesn't work I believe you can do the following as root.

Code:
apt-get remove package

Title: uninstalling xine
Post by: imnes on February 01, 2005, 10:40:29 am
Totem can be built against xine-lib or gstreamer.  If it's built against xine-lib do not uninstall xine-lib because that would break totem.  (Also if you try to uninstall xine-lib synaptic should uninstall totem also just to keep things in order, it will try to keep you from breaking dependencies).

You can find out for sure by going into totem, help menu,and select About.  It should tell you if it requires xine-lib or gstreamer, here's mine:
