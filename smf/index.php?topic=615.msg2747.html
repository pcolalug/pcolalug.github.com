Mandrake has done many things very well in this release. A couple of the downfalls could be simply overcome if I had a clue as to what to do.

I never got 9.0 loaded so I don't know what the differences between .0 and .1 are. I have come up from 8.2.

First, the install.
It went very well. I think it was shorter than some of the previous installs that I have done. I believe from start to finish it was 20-25 minutes, though I did not actually put a watch on it.

In Mandrake they let you choose what type of machine this will be. You can choose one or all of the different types of environments and it will install software to give you that. For example, you can tell the install that you are wanting an Office Workstation, Server, Game Station, Development machine, etc. It will then put on their choice of the needed software. I did not install everything in the initial install. Just games, and internet machine. I figured I wanted everything in those options.

You can choose individual packages as well. I just let it bulk install these. After the initial install, I loaded on specific packages that I wanted.

I believe Mandrake to have the simplest, straightforward partitioning interface for a computer that you are dual booting. Since I already had a partition for Linux, this was simple. But I wanted to change some of the partition sizes, so I was able to do it during the install. If you have installed Mandrake before, none of this is really different.

Once you get into the system, you are presented with a nice theme, but I like to change mine around a bit. Before doing so, I looked at KDE and Gnome. They are configured to look pretty much alike, same desktop and window dressings. The launch menus look different, and the desktop icons look different, but they are layed out and ordered the same in both desktops.

After using them both a little, I think that Gnome looks better (prettier icons) but I still like KDE. I guess I am just used to it. All the programs are avialable in each. If you like Gnome and are only going to use that, then you can just install it. I installed both to get a look at the new versions. KDE is 3.1 (which I think was in Mandrake 9.0), and Gnome is 2.2.

Here are some of the programs that I use and have loaded.

OpenOffice.org is 1.0.2, that is very new, it has been out less than a month.
GnuCash (which I am just looking at possibly using) is version 1.8.1. A new version was released in just the last couple of days, ver. 1.8.2.
Gaim v.0.59.8
Evolution 1.2.2
Let me know if you want to know the versions of anything else.

The software manager is much polished since 8.2. They introduced a new look back then, but the functionality was not there. I really struggled to get some software to install. They have continued with the new look. I think it is much simpler and easier to understand than before. And it works now. WooHoo!

The software installer, as well as the whole system, seems noticeably snappier than before.

The CD burning software is much improved. K3B, which is part of the KDE setup, is at the current new version. Which has only been out for 2 1/2 weeks. So it must have been hot off the presses when they rolled it into Mandrake 9.1. I must say that I am impressed with this jewel. I have been a serious critic of CD-R/RW software in Linux. Previous to this, everything I have tried was worse than burning from the command line. This works as expected. I have not had one single problem in burning a CD with it. When there have been problems, it has a very detailed explianation and helps you get it on track. I have only made one coaster, and that was when I told it to burn an ISO as a single file and not as a (whatever you call it when you want it to burn the image to the disk) thingy.

The one thing that I was disappointed to see was that Mandrake did not clean up the launch menus like many of the other distros are doing these days. I would have liked to see a more simplified menu. Fewer options per category and allow you to add whatever software you wanted later.

That is my first impressions (about 3 days old now). I will update this as you ask questions or I find something that I really like or hate about it.

Revised and Updated Post

Dragon,
I usually try and use the Mandrake Package Manager before resorting to the command line. It works out all the dependencies for me.

You can get to the "Software Sources Manager" either through Mandrake Control Center, or from the launch menu|Configuration|Packaging. After you download your programs to a direcotry then set the source directory in the Software Sources Mangaer (this is a little misleading since it is not the source code that you are installing, but a binary).

This is done by opening the source manager and clicking "Add." Tell it "Local File" and give it a name if you would like to name this source. I usually just dump all my binary downloads into one folder so I don't have to go through this again in the future. Then tell it what directory everything is in. You should be set.

When you set this as a source directory, you can then install through the Package Manager and then you will be able to easily satisfy the dependencies. If there is something else you need and it is on the CDs then it will just tell you which CD is needed.

Let us know how it goes.
