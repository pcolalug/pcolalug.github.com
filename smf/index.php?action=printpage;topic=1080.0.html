At the command line (might have to be root), just use:  ifconfig

This should produce something similar to:
Code:
[root@palantir danc]# ifconfig
eth0      Link encap:Ethernet  HWaddr 00:0C:76:12:9F:19
          inet addr:192.168.254.104  Bcast:192.168.254.255  Mask:255.255.255.0
          UP BROADCAST RUNNING MULTICAST  MTU:1500  Metric:1
          RX packets:58424 errors:0 dropped:0 overruns:0 frame:0
          TX packets:75981 errors:1 dropped:0 overruns:1 carrier:0
          collisions:0 txqueuelen:100
          RX bytes:14669149 (13.9 Mb)  TX bytes:6583349 (6.2 Mb)
          Interrupt:23 Base address:0xd800
 
lo        Link encap:Local Loopback
          inet addr:127.0.0.1  Mask:255.0.0.0
          UP LOOPBACK RUNNING  MTU:16436  Metric:1
          RX packets:716 errors:0 dropped:0 overruns:0 frame:0
          TX packets:716 errors:0 dropped:0 overruns:0 carrier:0
          collisions:0 txqueuelen:0
          RX bytes:35800 (34.9 Kb)  TX bytes:35800 (34.9 Kb)

The "eth0" device is my network card (connected to router/cable modem), and you can see on the second line that my IP address is 192.168.254.104.

Not sure what a dialup adapter would show as, maybe "ppp0" or "tty0"...

Let us know if you have any more questions.

Title: IP address ?
Post by: dpeach on October 19, 2003, 01:49:59 pm
The ifconfig (if you find it and get it working) should tell you both LAN and WAN. That is assuming that you are running this from a computer connected directly to your DSL modem/gateway. Otherwise, it will just tell you what address is assigned to that computer from your internal DHCP server.

BTW, in Windows the command that gives you similar results is ipconfig. I am sure there are switches that will give you more information.
