I'm having a hard time getting my new wireless adapter fully working. I got a Linksys WUSB54G ver4, which is supported by the ural driver in OpenBSD. I want to get this thing setup to run as a hostap, and it works fine as an 802.11b ap. The problem is that it should be able to work as a 802.11g ap. Here's the ifconfig I'm using to configure it.

Code:
# ifconfig ural0 10.20.60.1 netmask 255.255.255.0 media OFDM54 mode 11g mediaopt hostap chan 10 nwid ruggy

That should make it work, but this is what I get with the ifconfig ural0 output.

Code:
ural0: flags=8843<UP,BROADCAST,RUNNING,SIMPLEX,MULTICAST> mtu 1500
        lladdr 00:0f:66:75:fa:65

        status: active
        ieee80211: nwid ruggy chan 10 bssid 00:0f:66:75:fa:65 100dBm
        inet6 fe80::20f:66ff:fe75:fa65%ural0 prefixlen 64 scopeid 0xd
        inet 10.20.60.1 netmask 0xffffff00 broadcast 10.20.60.255

The thing that caught my eye was the fact that on the media line, autoselect mode 11g hostap are in parenthesis. Don't ask me why. If you guys have any idea, please let me know. While I got the adapter as a hell of a deal (< $20.00), I would hate to think I got it and it won't work for what I had intended it to do.
