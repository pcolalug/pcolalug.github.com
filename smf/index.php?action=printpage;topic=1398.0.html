Title: Abit KV7 Bios Issue
Post by: hitechwizard on June 03, 2004, 09:18:13 pm
If you have the SATA interfaces operating, anything with a 2.6.x kernel will identify them first as HDA-HDD, then the EIDE drives as HDE-HDH. If you were to disable the SATA interface in the BIOS, then your EIDE drives would have started at HDA like you were expecting.

Title: Abit KV7 Bios Issue
Post by: steve on July 06, 2004, 09:54:36 pm
I haven't paid much attention to this thread since I don't have the motherboard but thought I'd give the research a quick shot. I did a google on "KV7 Suse 9.1" (My current distro of choice) and it came back with this link (http://forums.suselinuxsupport.de/index.php?s=8ee9b7fb1b1e3e8f8543d22d72b626a4&showtopic=1808) as the first hit in google.

Basicaly the solution says:

" OK - fixed it. At the splash screen type into the command line:
ide=reverse
then select "installation" and off you go!

I guess this had the effect of detecting the SATA controller *after* the onboard IDE
chipset, thus giving the appropriate device labels to your hardware.

SuSE Sys Admin manual to the rescue again"

I hope this helps....
