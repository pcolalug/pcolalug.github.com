Post by: rugmonster on November 23, 2004, 02:30:10 pm
Not to turn this toward a Windows support board, but what you need is a logon script and something to act as a domain controller. Samba will not act as an Active Directory server, and unfortunately, you can't mimic one using LDAP and Kerberos since MS's implementation has some minor differences. If you don't mind using Samba as a PDC such as you would find with an NT Domain Controller, then you could go that route.

I will tell you that I have had very little luck setting share permissions and what not on Samba. Noticably more trouble than compared to Windows. To be quite honest, I'm not sold on using Samba in a Windows domain setting.

From what I have heard, you can use Samba in an Active Directory, but it cannot act as an AD server, meaning either you would have to use an NT domain running off of Samba, or have a window server 2000 or 2003 system to provide your AD. Like I said before, I'm not sold on using it in a domain environment where share permissions would be a concern.

What sort of email do you want to employ? The best you are going to get away with Linux is IMAP and SMTP, unless you go for more expensive products from SuSE and others. It depends on what you are using now and what you plan to use in the future.

Finally, I noticed you said something about the Air Force. I do Information Protection for the Air Force. You may be stuck using SuSE or Redhat. I think those are the only two distributions which have been officially approved for use on Air Force systems.

I have been working with Windows Server 2003 and Exchange 2003 since April. In all honesty, I'm very impressed with it. The ease of administration is almost to cry for and I have yet to find a security issue that really worried me. Since it sounds like you are afraid of keeping your job, this desire to bring Linux into your network may be a terrible mistake. I think you need to tread very carefully if you do propose this becuase it's a very difficult thing to do and I would say in this case, it would be smarter to run Server 2003 and be done with it. I can say I would not recommend this to someone in a production setting. Try it out with a Linux server acting as your PDC/email/file server and get a couple of clients. Try out the user management and all of your functions and see how long it takes you to stand up. I think you will find a lot more issues making it all work like Windows Server would than if you were actually running Windows Server. I don't even know if I would trust myself to pull this off and be truely happy with the results.

As for a logon script, I've got a nice little WSH logon script written in VBScript that does a lot of mapping of network drives based on group membership that I'll send to you if you would like it. It's very simple and probably lacks a lot of error handling, but it was something I had to cobble up at the last minute when I found out we couldn't use KIX32 on our systems. Yay, AF!

Post by: hitechwizard on November 23, 2004, 04:49:45 pm
I just converted an entire office from Netware 5 to a Mandrake 10 system. It can be a painless process, depending on what the Netware server is being used for, and what added functionality you are trying to obtain (I mean, if your not going to gain anything, why upgrade? I have another client that still has a Netware 3.12 server, and 8 DOS 6.22 workstations running!).

To give some background on the project I just completed, the Netware server was being used for just a file server, and there was a Sybase NLM running for their contact management system that consistently crash every other Monday. Among other problems was with the Netware client software, as they continued to be upgraded, they began dropping support for (or at least we were unable to continue to access) Netware 5 and IPX. Long story short, they now have 2 Mandrake servers, and an OpenBSD Firewall. One of the Mandrake servers acts as the office server that  runs as their DHCP, Samba (both smbd & nmbd with WINS support), SQL Server (using MySQL) and intranet web server. I ended up writing a custom contact management system using Apache/MySQL/PHP for them, that has performed consistently and flawlessly since installation. The other Mandrake server acts as their web, email and primary DNS server. In their case, all files on the shared drive are public in nature, so permissions were not an issue. (And yes, permissions are a PAIN when using Samba). The linux server shows up in everybody's Network Neighborhood just like any other Windows server, and since the shares allow anonymous login from the local private network address space, there's no permissions issues.

I have heard (not confirmed) that the NDS (Novell's LDAP implementation) is now in the public domain and available in both Linux & Windows flavors. Not knowing what kind of authentication you are needing, this example at least gives you an idea of what's possible.
