Title: Encryption Question
Post by: on January 05, 2003, 10:02:16 pm
IIRC, once the client establishes a secure connection with a server, it generates a new master key and transmits it (encrypted) through the secure connection. Since the server is theoretically the only holder of the private key, only it can decrypt this new master key. All subsequent transmissions are encrypted and decrypted using the new key.

Hope this sheds a little light on the subject.

Title: Encryption Question
Post by: altomo on February 21, 2003, 12:54:50 pm
To answer your question about can and IDS system montior the encrpyted traffic:

It depends on the IDS. If the IDS is host based and lives between the OS and the application then yes since the traffic is decrypted at some point it can monitor and or stop an attack. If the IDS lives on the host at the same level as the application then no since it is really just a network sniffer at that point.

Now there are some exceptions to the rules of course. Some encryption is weak, base64, so an IDS could be writtent to crack base64, check the traffic and then respond although this would have to be done quickly.
