->The VP of Tech has asked the security team to make a case for Linux in other areas.

The VP of Tech needs to get a clue.  When it comes to security, the operating system itself bears very little impact.  What is more important is how the operating system is configured/updated/used.  I have been in the security business for a while and have seen Windows systems that were severely locked down, and Linux boxes that were swiss cheese.  One of the exceptions to this is OpenBSD which is relatively secure by default.  

My main point is this:  the operating system choice in and of itself “usually” has little bearing on the security.  A case can be made for anything by anyone, what the VP of Tech should have asked was this:

“Let’s review our current infrastructure and make sure that we are using the best tools for the job.”

Just my $.02

Kyle
