36  Distributions / Security / Re: Is a firewall necessary for Linux? on: June 18, 2007, 06:52:05 pm
Assuming that things haven't changed with the most recent release(s), an install of Ubuntu should have no public services running by default.  Installing and configuring a firewall*, such as Firestarter or Lokkit is still a good idea, in the event that you (either knowingly or otherwise) should ever install a server program that opens up a port by default.








* Actually, you already have a firewall installed -- iptables.  Firestarter, Lokkit, and the like are just GUI frontends that allow you to easily configure iptables, which by default comes with no pre-set configuration.

43  Club Stuff / Library / Book Review -- Programming Ruby: The Pragmatic Programmers' Guide, 2nd Edition on: November 12, 2006, 07:02:22 pm
I found this book to be very well written and concise, though clearly intended for someone with some level of prior programming experience.  My only real complaint is that some of the chapters, while covering a wide amount of material, seem to be lacking in depth.  Aside from that, however, I found this book to be more than adequate to give a solid a foundation in Ruby, and after it serves as an extremely useful reference for the classes and modules that are built-in to Ruby.

All in all, if you're interested in learning Ruby, this book should definitely be in your collection. 5 stars out of 5.

(This review also posted to oreilly.com and barnesnoble.com)
