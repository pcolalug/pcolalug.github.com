Post by: lloydr on May 25, 2004, 01:22:01 pm
in slackware chances are its already enabled unless you disabled the start script

but just in case:
as root
Code:
cd /etc/rc.d

then
Code:
chmod 755 rc.sshd

that will enable the ssh server to enable on boot, or if you want it to start and you don't want to reboot just do(after chmod 755 rc.sshd)
Code:
./rc.sshd

while your still in /etc/rc.d and after making sure the permissions allow it to be executed....in slackware to enable services in the /etc/rc.d directory to start on boot you chmod 755 their rc.x start script, to disable them you chmod 644 their rc.x start script(where x is the service in particular your wanting to enable or disable)

default config probaly will work just fine for you for the most part, but in case you do want to edit the config(i recommend first thing is disabling root logins through ssh and force use of ssh2 only), the configuration file is in /etc/ssh, for the sshd server in particular, the file your looking for will be sshd_config

hope that helps ya out  :wink:
