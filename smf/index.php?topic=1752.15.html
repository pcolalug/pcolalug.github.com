Another thing I do recall is that on the Nvidia driver website, it gives specific instructions just for Suse (there's a link for it).  Suse seems to do things somewhat differently than other distros.

Dan


OK, not to beat this into the ground but I just installed the latest Nvidia drivers on my SuSE AMD64 machine without a hitch. I followed the 5 steps in the how-to and here's what they said. See if there's anything different from other distros. (Remember, these steps are written for a failsafe install so they may seem redundant in some cases.

1. cd /usr/src/linux (This takes you to the installed kernel source directory)

2. make cloneconfig (This builds a kernel config file based on the current running kernel. This is pretty smart if you ask me.)

3. make prepare-all (Steps that need to be done before building the kernel which nvidia does)

4. sh NVIDIAxxx.run (This is the Nvidia install script)

5. sax2 -m 0=nvidia (This automatically configures X to use the new driver. No editing of xf86config is necessary)

So, it seems to me that SuSe is just making it alot easier and "safer" to install the Nvidia driver. If the instructions are followed, it should cut down on the number of problems people run into as I saw in my previous distro "Mandrake".
