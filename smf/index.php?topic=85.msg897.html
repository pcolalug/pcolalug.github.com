I have used many different versions of slackware.  Right now slackware is powering my router / web server.  

Seems to be a nice little distro.  Weak package management though, I think they should adopt RPM which is sort of a de-facto standard now anyway.  That would allow Ximian to release their desktop for it.

Releases are very slow compared to other distros, expecially in the last two years.

If you're used to the nice graphical installers of mdk / rh / suse you probably won't like slackware.  The install is text based (ncurses) and not very user friendly.  A little cumbersome.  About like installing FreeBSD if you've ever tackled that.  Definitely not for a beginner since you should know how to partition your drive with linux's fdisk.

On the plus side it has a lot of nice tools for the console user (why I'm using it for my router).  Nice utilities like NetWatch which other distros don't care to include.  And a text editor called "pico" that a friend likes to use (if you ask me, you never need more than vi for quick editing jobs, and emacs for web browsing / authoring )    

They also seem to be sort of a purist distro.  They don't seem to molest things like KDE and GNOME as much as the others do  That could be a good thing or a bad thing depending on what you like.

If you know what you're doing, don't mind compiling your own apps from source, (and know how to customize the kernel, non-modular last time I checked) ,then you might just like it.

Quote
As for FreeBSD, I found the install process logical and intuitive. It *was* my first "real" OS install, and it went smoothly. Thanks to the great docs, I had done my first custom kernel by the end of the first day with no probems.


I have to comment on this.  Maybe if I hadn't gotten used to the way Linux handles things this would have been true for me.  But I found some of the install process very disorienting.

Like the idea of slices, disklabels, and all the other stuff.  DOS lets you setup Primary, and Extended partitions.  Linux takes it a little further by letting you create filesystem and swap partitions (but still, you're dealing with just primary and secondary.)  The terminology used in FreeBSD confused me when going through the installer.  (Maybe because I'm not one to read a users manual before trying something.)  Maybe you can refresh my memory, I haven't installed BSD in a while.  But first you setup partitions, then you setup slices or something like that.  Anyway, it was just a little confusing, I got through it.

The startup scripts were arranged much differently than I was used to with RedHat.  And actually, I found them easier to deal with.  (Talking about the stuff in /etc/rc.d/... here)

Another unique feature was the whole deal about customizing the kernel for the installer.  I wasn't quiet sure why I was given those options.  Was it just to include modules I needed while installing?  Was I customizing the kernel that would be installed on my system?  From what I remember there wasn't much explanation in there.

And the last point is the text based installer interface wasn't very intuitive.  After using the various text based utilities in BSD for a week or so I became very comfortable with it, but for some reason during the install I couldn't keep track of where I was in the menu system, and kept running through the software instaler when I thought I was done and trying to exit (made that mistake 3 times).

But once the system was up and running, I was in love with it.  It seemed (at least in my view) to be faster and more responsive than Linux on my system.  Consumed far less RAM.  But just lacked some of the nice software I like to use in Linux (namely Ximian - which I don't use anymore anyway, VMWare, and Kylix).

Oh yeah, another thing that threw me.  The hardware naming is different. I'm so used to /dev/hda for my hard drive, took me forever to figure it out in BSD.  (Not even sure if I'm remembering this correctly, but I think my drive was /dev/asd0a or something).
