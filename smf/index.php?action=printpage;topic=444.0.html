Title: vnc/ssh

Did you ever get this resolved?  People sometimes forget that if you tunnel stuff through ssh, you can block all other ports on your firewall except port 22 - everything else goes through the encrypted/compressed ssh pipe.  Makes security a bit easier to deal with :)

I use it all the time.  If you copy over your public certs to your target machine, yuo don't even have to type a password to get ssh to do it's magic.  I use it all the time when I'm out in Milton - I run UUCP through ssh and never have to type a password, it just works :)

Title: tightVNC/SSH tunnel problems
Post by: rburt3 on February 03, 2003, 10:37:08 pm
I did, BTW get to do what I needed to do with the following command:
Code:
ssh -X -C username@remote.host
Once you do this, type the command for any of the X-apps on the remote machine at the prompt. You may have to wait a bit, but soon enough said app will pop up on your screen but be running on the remote machine via ssh. Not the most responsive, but works in a pinch.

If anyone else is interested, here (http://www.linuxworld.com/2003/0203.barr.html) an ok article on how to set up VNC and use it with ssh. Its by Joe Barr and so is light on the technicalities, but it can get one started...
