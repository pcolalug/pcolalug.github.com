Post by: imnes on October 29, 2003, 08:23:32 am
Okay I'm not a BSD expert, this just just what I've noticed on my own system.

My newer hardware works great in FreeBSD before it works well under Linux.  That's one of the main reasons I like it.  APM/ACPI support actually works (I can suspend and resume without a problem), something I still can't do under linux.

Linux binaries work find under it, most games I've tried work.  In fact, Enemy Territory had strange audio problems for me under Linux and Windows XP, but works flawlessly under BSD.

If you have an nvidia card, then you get official drivers from them just like under Linux.

The sound system of FreeBSD is also great, it can mix multiple sound streams in the kernel (something you'll appreciate if you've ever had the joy of running artsd or esd along with another program that wants to grab the sound card and have the program hang).

I like the package system of FreeBSD.  If you want to install a binary package, like Gnome 2, you just type:
pkg_add -r gnome2

It will fetch the packages off of the FTP and install them.  I think it automatically grabs dependencies too.

If you want to compile from source you go into the 'ports' directory, find your application, and run "make install", it grabs the source code from the project's ftp page, applies any patches from FreeBSD, compiles and installs.  You can also do make deinstall, make upgrade (I think).

The only downside, Wine quit compiling under FreeBSD for a while but it's now working again.  Sun didn't release a native java SDK for it (but the BDS folks have implemented it themselves and sun has taken the project and made it official), it's only 1.3 though, but should be okay for most people.  Those are the only two apps I really had any trouble with, otherwise everything seems to be available.

You have two compiler systetems to deal with I think, because sometimes I grab some source code that's not in the ports system to compile and install myself, running 'make' doesn't work, I have to run 'gmake'.  That's just a minor annoyance.

Right now there's two versions of FreeBSD, the older 4.x releases and the beta 5.x releases.  I like 5.x better, once it becomes the standard release and the java port to it is working I might make the switch back to FreeBSD on my desktop, right now I just have it running on a mail/web/ftp server.
