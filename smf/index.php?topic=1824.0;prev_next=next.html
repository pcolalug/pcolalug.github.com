I'm having one hell of a time trying to get Samba working on a Mandrake 10 paranoid install. However, when I try to attach to the share, I get something along the lines of "Not set up for transactions". Found my problem w/SWAT - helps to have xinetd running.

Anywho, once I got it going life was easier. My problem turned out that I had not created the smbuser for the guest account. Here's the final smb.conf that I ended up with:

Code:

# Samba config file created using SWAT
# from 10.0.0.95 (10.0.0.95)
# Date: 2004/08/11 19:24:22

# Global parameters
[global]
        workgroup = *********
        server string = Linux Samba Server %v
        map to guest = Bad User
        null passwords = Yes
        smb passwd file = /etc/samba/smbpasswd
        guest account = ********
        log level = 4
        log file = /usr/local/samba/log.%m
        max log size = 50
        name resolve order = wins bcast
        socket options = TCP_NODELAY SO_RCVBUF=8192 SO_SNDBUF=8192
        os level = 65
        preferred master = Yes
        domain master = No
        dns proxy = No
        wins support = Yes
        ldap ssl = no
        default service = shared
        remote announce = 10.0.0.255
        hosts allow = 10.0.0., 127.

[shared]
        comment = Shared Storage (public files)
        path = /home/*******/public_html/shared
        read only = No
        guest only = Yes
        guest ok = Yes


Client information changed to protect the innocent.

Anyway, thanks for giving me a hand.
