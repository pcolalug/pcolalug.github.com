I talked about Hamachi at the November 2008 meeting. Hamachi is a zero config VPN solution that works through firewalls without having to open any ports.

Here are the notes from the meeting:

Networking with Hamachi

Hamachi = Log Me In: www.logmein.com

If you go to the download steps, it asks for name, etc. You do not need to give it any information. Just click through. https://secure.logmein.com/products/hamachi/list.asp
(Or grab directly from here: http://files.hamachi.cc/linux/)

Quick steps at Hamachi Wiki: http://logmeinwiki.com/wiki/Hamachi:Install_on_Linux
1.  Download the hamachi Linux package
2.  Extract the package to a temp directory
3.  Change to that directory in terminal
4.  Run 'make install' and then 'tuncfg' from under the root account
5.  Run 'hamachi-init' to generate crypto identity (any account).
6.  Run 'hamachi start' to launch Hamachi daemon.
7.  Run 'hamachi login' to put the daemon online and to create an account.
8.  Run 'hamachi join <network>' to join the network.
9.  Run 'hamachi go-online <network>' to go online in the network.
10.Run 'hamachi list' to list network members and their status.

Ubuntu
http://www.supware.net/HamachiUbuntuHowto/
May need tun-source and build-essentials

usage: hamachi [-c <cfg_path>] [command]

-c         overrides default configuration directory (~/.hamachi) command specifies an action. Can be one of the following -

            #  start [debug]
                stop
            #  set-nick <nickname>
                login
                logout
            #  list
                get-nicks
            #  create <network> [<password>]
                delete <network>
                evict  <network> <5.x.x.x>
            #  join   <network> [<password>]
                leave  <network>
            #  go-online <network>
                go-offline <network>

If no command is specified, hamachi displays its status including version, pid, online status and the nickname
