Title: Audigy2 Not Working
Post by: imnes on December 03, 2004, 07:51:49 pm
Please go into your BIOS and disable the onboard audio.  Linux will arrange your sound cards in the order they show up (I guess in order of IRQs maybe).  But anyway, if your onboard card takes priority then it will get sounds from KDE and other stuff, not the audigy.

Also when you went into KDE it might have tried to change the mixer settings.  I don't remember the exact channel you have to unmute to get it working, but it's somewhere in alsamixer at the command prompt.

I'll try to find the post I mentioned the fix in and add it here.

No no, I got it working, but you're going to laugh at how.  The only thing I could think of was that SOMETHING went wrong when running xdm since I had to be root and that might've modified something in a strange fashion.  So I booted up the standard 2.4.22 kernel and surprise, the Audigy2 works.  So I went back to the 2.6.7 kernel, tried to modprobe the card, nothing happened.  I even went into the module directory and tried it, no dice.


So, one fresh kernel rebuild later it's working again, just as good if not better.  Same kernel config except this time I totally removed support for onboard audio (who uses it anyway when you've got another sound card?).  What I found most interesting is that with the "stock" kernel it worked fine even though it detected both cards.  Now it's only got the one and it works.


Basically the philosophy idea was, "If I'm going to have sound, it's going to be through ONE DEVICE ONLY." With that, problem's fixed.




I DO like Slackware and have had much more luck with it on all sorts of things that I haven't posted about (no problem, no post) whereas I've had issues with other "automagick" distros.  The number of issues I've had with Slack (soundcard itself was a Linux thing, not a Slack thing) on this box, Anubis, has numbered two, maybe three if you count the confusion about the graphical login.  Given that few number of "issues," both of which have been fixed, I'd say it's been a stellar distro.  With Mandrake, while I still heavily recommend it to all my friends, I didn't get the feel that it was "right" for me.  Fedora...just didn't feel like Linux to me, don't know why.  Linspire feels the same way, not anything "wrong" but not really "right for me."  Debian I'd like if I could get over the installation, and Gentoo took too much installation.
