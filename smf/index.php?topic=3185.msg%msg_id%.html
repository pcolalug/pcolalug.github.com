Quote from: linuxnewbie
After reading some tutorials, I tired using the aptitude install firestarter command to install firestarter.  The installation failed because I was not logged in as root.  Can anyone tell me how to do this?

First, let me introduce to you the Ubuntu 7.04 Official Documentation. Next, let me introduce the Ubuntu Community Documentation. Start at the first link and follow on with the second. I'm not telling you to go RTFM at this point, but in the future, look here first.

As for executing commands with "root," or administrative permissions, Ubuntu opts for the sudo approach. The way it works is that the root user is disabled by default and the first user you created has the ability to "sudo" as root. If you are familiar with the runas command in Windows, then you are familiar with sudo, but the syntax is easier. So let's take your aptitude command. Here's what it would look like using sudo.

Code:
$ sudo aptitude install firestarter
Password:

You run the command and then you are prompted for a password. You put your password in here and then the command runs as root. If you are in Gnome, you will get a window that pops up asking you for your password and permission to run an application with administrative privileges. Same thing happens there, except it is the Gnome equivalent of sudo, gksudo.

Quote from: linuxnewbie
I'm also not too happy about having to go online to install a firewall, so I tired to download a copy on my Windows machine.  I found the firestarter website, and when I clicked on the download link a tutorial web page opened up, but the download never started.  Can you copy programs to a Linux drive and install them off-line?

Well, I think you're being a bit paranoid here and as a security professional, I rarely say that. Like we said before, there are maybe one or two listening services, if any, on a default Ubuntu desktop install. You don't really need a firewall because a firewall's purpose is to block open ports to listening services. I'm assuming that if you have more than one machine, you are at least behind a cable/dsl router, in which case, I really think you're being paranoid. Your box will not get owned in the time it takes you to download and install Firestarter through apt/aptitude/etc.

If you feel you absolutely must have a firewall, you already have a firewall included with your system, called iptables. Firestarter is just a frontend for iptables, so if you want, you can go on your Windows machine and look up how to setup iptables.

As for installing applications without using apt/aptitude/etc, Ubuntu, being a Debian derivative uses the .deb package format. If you obtain a particular package you want to install, you can do so with the dpkg command. Here's what the install command would look like:

Code:
$ sudo dpkg -i example-pkg-0.1-i386.deb

By using this method, you may run into what is lovingly referred to as "dependency hell". Since most applications in Linux depend on shared libraries and other programs, it is more efficient to package up just the application and mark the other dependencies as such and leave it up to the user or the distribution to sort those dependencies out. Apt/aptitude/etc handle dependency checking and fulfillment automatically for you, but they all use dpkg to do the actual install and dpkg has no dependency fulfillment capabilities.

There is always the option of installing an application from source, but you shouldn't need to do that unless a particular app you want isn't in an Ubuntu or third-party repository. If you find yourself there, you can seek the help of someone more experienced if reading the installation documentation isn't clear.

To help you out, might I suggest you hook the system up to the network and get it online. Once you're online, get Automatix. Follow the installation instructions and start it up. From there, you will find Firestarter and ClamAV for your firewall and antivirus pleasure, along with many other great apps that don't come installed by default.

I would also like to suggest some reading for you. Ubuntu for Non-Geeks, 2nd Edition is a great read. It is available through O'Reilly and I believe that means you qualify for the LUG discount. I think that will help you get a firm footing and serve as a starting point for further exploration.

As you can see, I'm not going to tell you to go away and RTFM, but I do encourage everyone I help to doing some research on their own. I'm of the camp that believes if one finds the solution to a problem on their own, they will retain that information more than if someone gives it to them. With that said, we are all here to help where we can and we'll flex our mighty Google-fu if there is something we don't know. Best of luck and let us know if something doesn't make sense here or if something else comes up that stumps you.

Cheers!
