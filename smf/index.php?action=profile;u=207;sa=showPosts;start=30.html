A distro's "intended use" can only be defined by the user who is using it, wouldn't you think? You may say that Slackware or Debian are "intended" to be used as servers (and I would agree to a point).

The distro's intended use is determined by whoever releases the distro, either explicitly in the release notes or on a website, or implicitly in the way they package the distro. I use Slackware as a desktop as well. Good for me. I would not recommend Slack as desktop to a newbie, because that is not Slack's strong suit. I don't mind typing "startx", not having fancy buttons, and not having GUIs do everything from configuring my printer to connecting to wireless. However, Pat V has implicitly targeted an audience by setting up Slack the way he did. Fedora implicitly determines its intended use by making it easy for any monkey to get a GUI interface for everything, allowing root SSH logins by default, and opening more unneeded ports by default than Slack or Debian. Any Linux distro can be made into a good server, but the actions of the people releasing a distro implicitly determine the intended use. A final example is *BSD. BSD makes a fine server, but takes a lot of work to make it into a desktop. I have used BSD as a desktop, but the amount of work put into doing so tells me that BSD was not intended to be a desktop.

37  Distributions / Other Linux Distributions / Best way to update Slack on: July 16, 2005, 07:02:05 am
I'm not trying to be elitist. What I am trying to point out is that there a hundreds of distros available, each with an intended use and audience to suit a multitude of personal preferences.

When you try to force one distro to act like another, it introduces an unknown element. For example, if you run emerde on Slackware, you will spend some time fixing broken apps because emerge was not designed for Slackware--it was designed for Gentoo. Slackware uses the System V init scheme, whereas Fedora, Ubuntu and many other distros use the BSD style. Gentoo uses an init scheme similar to System V, but uses proprietary tools to administer those scripts (rc-update). What all that means is that config files are not in the same place between certain distros, yet you are using a third-party app which circumvents the differences, but is not accurate in keeping track of those differences, thus resulting in a broken system.

If you use distro X, use its native package manager, or build from scratch. Or, know how to code, and fix the problems along the way, because there will be a lot of problems.

Slackware is a no-nonsense distro. One should administer it with the same philosophy. Otherwise, you are defeating the purpose of the distro's philosophy, and using the philsophy of another distro. Use what you want--there are hundred distros available. Just don't complain if you take distro X, try to make it work like distro Y, then complain about it not working.
