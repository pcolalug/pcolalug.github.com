Title: Starting X without a password
Post by: dpeach on February 04, 2004, 05:23:36 pm
On my Mandrake machine I can boot the computer into my standard user without requiring a password. There is a password on the user, but it will do the initial boot without asking for it. If I log back out and try to log in, I have to supply the password. When I do XFDrake to make changes to my XF86Config file it asks me if I want to use this feature. Therefore, I assumed that it was something handled by XF86Config. But it is not, which I could have figured out based on the fact that simply restarting X does not allow to bypass the password, only restarting the whole machine.

I am trying to do the same on my Gentoo system w/ KDE. I am supposed to be using kdm, though I don't know since I never have to manually start X. Is this something handled by kdm and how would I go about finding this?

Thanks.

Title: Starting X without a password
Post by: Madwilli on February 04, 2004, 05:53:35 pm
I'm looking at a redhat 9 derivative here and I see the kdmrc file in
/etc/kde/kdm

If you look in the file, there are 'AutoLoginEnable' and 'AutoLoginUser' fields.
Was this what you meant by logging in automatically?

Not sure if this helps - here's hoping.

Title: Starting X without a password
Post by: dpeach on February 05, 2004, 12:09:43 am
Ahh, once again. Right where I expected it:
/usr/kde/3.1/share/config/kdm/kdmrc

I should have thought to look there first.

But all rants aside, that did the trick. Thank you. I was able to find the answer on the Gentoo forums, but only after I knew what to look for. I appreciate the direction.
