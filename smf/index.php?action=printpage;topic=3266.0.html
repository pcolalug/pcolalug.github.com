Post by: Skreyola on November 28, 2007, 12:00:25 am
I've been working with Linux for a long time now, but I haven't learned as much as I should have.
For a number of years, I was fascinated by the plusses of Linux but unable to get myself to really work with it, because I hadn't figured out how to do the important things I needed to do.  I started out with Slackware, and it installed great, but then I didn't know what to do next.  It wasn't until, oh, three or four years ago that I got a distro I could really work with.  It was Red Hat 7.3, and it was the GUI, ironically, that got me to the point where I could do most of the things I normally do with a computer, only under Linux.  Ironic because I am, at heart, a command line person.  I learned computers on the C16 and C64, then under MS-DOS.  The reason the GUI helped me is that it provided sort of a map for important applications.  On the other hand, it allowed me to use things like web browsers and file managers.  Still, I do most of my non-visually-oriented functions with the command line.
I currently run Debian stable on all my Linux boxen. I use IceWM for my X window manager. I generally use Opera for browsing. Are there other preferences I should mention?
I'm interested in learning more. I want to support two things simultaneously: education for computer users so they can be free to choose what to run on their machines, and efforts to make the installation of Linux and its applications easy enough that people who don't know computers can get a basic system running.
