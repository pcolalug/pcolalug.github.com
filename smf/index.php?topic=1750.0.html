Quote from: Rugmonster
A quick way to kill your X session is to hit ctrl-alt-backspace. It dumps you back out to the console and you can start X back up.

This won't take you to a console if you have X starting automatically at boot (runlevel 5 in most distros).  It will exit and restart X, but you don't get access to a console.

Unless the keyboard is frozen hard, you can get a console with Ctrl-Alt-F1 (and/or Ctrl-Alt-F2 through Ctrl-Alt-F6).  To return to X, use Ctrl-Alt-F7.

One nice thing about Linux is that it almost never needs to be rebooted due to a lockup. (Those compiling their kernels for the first time, ignore that last sentence. )

If you're in X, whether it be Gnome, KDE, or some other Windows Manager, and your mouse is still working, you can try right clicking on an app in it's title bar to kill it if it's locked.

There is also a mini program out there that you can run and it will allow you to point at a locked up app and kill it. (I don't remember the name)

If X is locked up and your keyboard is still working you can try "ctrl-alt-Backspace" to kill and (hopefully) restart X. Also, "Ctrl-Alt-Fx" (where Fx= any function key between F1 and F6) will put you into a console where you can have all kinds of control over the system. "man" is a good command to learn in the console.
(From a console "Alt-F7" will put you back into X if is running.)

If all else fails and you can't get to a console you can try "ctrl-alt-del" to see if the system will safely reboot. (In a console try typing "shutdown -r now")

One important thing is to know why you're having to fix a lockup or reboot the system. It doesn't help to just have to reset the system once a week. This means there are problems and they need to be fixed. Once a week reboots are for other Operating Systems.

Disclaimer: The above instructions may not work, depending on your particular configuration. They've worked for me, though.

I guess it depends on how badly you are locked up.  Often times if you just let it sit there awhile linux will recover itself.  This is often difficult for people coming from windows to get.  If you are impatient like me you can:

<ctrl>+<esc>  this will give you a skull and cross bones that you can put over the offending application and kill it.  I like this one.  This only works in KDE.

<ctrl>+<alt>+<f1> to get a prompt, login, then a ps -aux or top to get the PID if you need it, then do a "killall <application>".  If you don't like <f1>you can use other <f#> keys to do the same thing.  If you are not familiar with virtual terminals I highly recommend getting to know them.  Just remember <f7> is reserved for the X-server and is how you come back to X if you can recover this way.

<ctrl>+<alt>+<backspace>  this will kill your x-server.  If you have init 5 in your /etc/initab (on most OSs) the X server will exit and restart.  I believe that is the way SuSe does it.  If you are init 3 you will get to a prompt.

Then there is the I really locked this baby up hard core recovery.  Not well documented and hard to find. I am not sure it is still in the newer kernels.  I haven't needed to ever go this far.

<alt>+<SysRq> hold down these two keys.  The <SysRq> should be part of the <print screen> key.  Then with your other hand...yeah this is hard to do, type one at a time S (sync) U (unmount) B (reboot)

Also if you have another computer that you can login to the locked up machine and kill the program that way.  Not always a option from a home system.

Hope this helps.  I use a spiral notebook to write down little things like this.  Comes in handly when and if it happens.  Just a suggestion.
