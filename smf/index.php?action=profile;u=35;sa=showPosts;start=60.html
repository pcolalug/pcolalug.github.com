63  Distributions / Debian Based Distrtibutions / Re: w32codecs on: March 14, 2006, 08:50:38 pm
Another option, download this archive (it's 12MB compressed)
http://www.mplayerhq.hu/MPlayer/releases/codecs/all-20050412.tar.bz2

Extract the contents to /usr/lib/win32 by running
Code:
sudo mkdir /usr/lib/win32
cd /usr/lib/win32
sudo tar jxvf /path/to/all-20050412.tar.bz2
sudo mv all-20050412/* .
sudo rm -rf all-2005412

That will give you the latest win32 codecs (incase the ubuntu package isn't available, or you just want something newer).  Also you'll have the best luck installing totem-xine for getting all your videos to work. 
Code:
sudo apt-get install totem-xine

And for your web browser
Code:
sudo apt-get install totem-xine-firefox-plugin

If that browser plugin gives you any trouble try the mplayer one.
Code:
sudo apt-get remove totem-xine-firefox-plugin
sudo apt-get install mozilla-mplayer

74  Software / Programming/Development / MPlayer on Solaris 11 on: January 29, 2006, 01:21:45 am
Compiling MPlayer under Solaris 11 (and Solaris 10) works, but win32 codecs cause it to crash.  Here's a fix (don't know if this will cause other problems or not, but it's working so far for me).

Edit the source file: ldt_keeper.c

Change:
#define NUMSYSLDTS 6 

to:
#define NUMSYSLDTS 7

Recompile and win32 codecs should now work fine.
