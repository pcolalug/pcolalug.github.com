Title: Compress
Post by: crowe on May 11, 2003, 03:21:38 am
Try doing one of the following:

"tar -cvzf mygames.tar.gz games"

This will tell tar to use the gzip compression on the .tar file before exiting, resulting in a file that is both tar'ed and gzip'ed.

You would un-compress this sort of file by doing a:


"tar -zxvf mygames.tar.gz"

Another option is using bzip2 in place of gzip, such as:


"tar -cvjf mygames.tar.bz2 games"

This is the same as before, but uses bzip2 rather than gzip.  You'll probably want to use this one, since bzip2's compression is more or less superior to that of gzip's when used in this fashion.

Similiarly, this is extracted with:

"tar -jxvf mygames.tar.bz2"


Seems simple enough, no?  8)

Just for grins, I tried an experiment with this.  I used my /usr/local/games/wolfenstein/main directory, which contains quite a few LARGE files which are already compressed (.pk3 map paks).  Because of that, the compression results were not spectacular, as expected.  Here's what I got:

wolfmain.tar = 752 Mb
wolfmain.tar.gz = 685 Mb
wolfmain.tar.bz2 = 665 Mb

While the .bz2 ended up the smallest, it took MUCH longer to do (10-15x) than the .gz

I'm sure that with some text files, or a file type mixture, the compression would be much more noticeable.  Anyway, it's interesting.
