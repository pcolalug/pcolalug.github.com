Here's an interesting programming idea for all the developers.  This site is driven by PHP and MySQL.  How difficult would it be to make a script that would send an email reminder to all members the day before a meeting to remind them about it?

I know some systems like AOLServer + TCL, I think ColdFusion, you can tell the server to run a script every so often, or on specific dates.  Is there a similar capability in Apache / PHP?  

Another alternative would be to make a crontab entry to trigger a script to do it, but I was wondering about having it fully contained within the web server software.

Another side-effect of figuring it out would be that I could modify the library app to send reminders to members who have books out to bring them back.

Topics like this have been discussed before among the Steering Committee. Though no real concensus has been reached, the main issue is the whole "unsolicited" email vibe.

That being said, if we want to explore this possibility as a group, I would like us to consider either:
1. In the general "disclaimer" section, informing the user that by creating/activating an account, they consent to the "reminders"

2. (my preference, but more involved) Add the "opt in" options to the user's profile (i.e. "Send me meeting reminders" or "Notify me when my account is about to expire"). This would likely break any easy upgrade path between versions of phpBB, but not impossible to do. If its a general enough implementation, though, we could contribute it back to the project.
