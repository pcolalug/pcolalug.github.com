Camp Fire K12LTSP Status Update

Status so far:
- Printing works
- 21 User Accounts setup (camp1, camp2, ... camp21)
- /usr/bin/autologin script setup
- PXE and Etherbooting works (Mileage may vary, depending on specific NIC's)
- Sound does not work, however, I think Ed is the "sound man".
- The Gateway Camp Fire client will not boot from the server.  Neither PXE or Etherboot seems to work.  All three of my computers were able to boot from the server.  Two by PXE, one by Etherboot.
- Password for camp1 ... camp21 is ________.  Password is not needed for auto-login.  However, if you log directly on to the server with one of the camp user id's, you will need a password.

To be done on site:
- Setup each client's MAC address in the /etc/dhcpd.conf file.  The MAC address for ws001 is for my computer at home, so it needs to be changed.
- Add memory to the K12LTSP server.
- Possibly add memory to the clients.
- Fix sound
- Setup firewall and filter server.  Seems like we discussed using a separate box for these functions.  The firewall is disabled on the K12LTSP server.  Also, SELinux is disabled.
- Install Firefox browser plugins, such as Flash and Java.  Since the kids probably won't be able to access the Internet right away, this can be done later.  (IMO)
- Verify that all clients can boot from the K12LTSP server and run various applications.
- Identify and resolve remaining issues.

Final Details:
Documentation and training for Camp Fire personnel.
Documentation of all configuration files that were updated by us.
Success Story (hopefully a success story) submitted to K12LTSP and LTSP web-sites.
Pictures of the lab.
Perhaps notify the Gulf Coast IT members of this project. 
Other thoughts?

BIG thanks to Ed for finding a resolution to the auto-login problem.  I wish there was a solution that didn't require using the MAC address, but at least it works.

Auto Shutdown for Crontab:

type:
Code:
sudo crontab -e

then add this to the end of the file:
Code:
* 22 * * * /sbin/shutdown -h now

That will shut down the computer at 10pm everynight. If you want a warning type system there is a good guide here:

http://martin.ankerl.com/2008/01/24/howto-get-enough-sleep-despite-stumbleupon-with-ubuntu/

Hope that helps!
