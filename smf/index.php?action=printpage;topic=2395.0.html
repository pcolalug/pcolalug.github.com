Title: SuSE 9.3 Pro Review
Post by: imnes on April 21, 2005, 07:17:15 am
(Work in progress, will change update this as I play with it some more)

My copy of SuSE 9.3 finally arrived.  I was worried about it not working because the LiveDVD version doesn't work on my system (USB keyboard and mouse never initialize).  Luckily the installation went just fine and everything seems to be working great.

I got the Student DVD version (which includes two DVDs and five CDs, a big green / red SuSE + Novell box, a printed User's guide, and a printed Admin guide).  It was $59 + shipping.

For a Gnome user, the default install is kind of annoying.  In the installation you can choose to install Base Graphical System + KDE or Base Graphical System + Gnome.  What it should have said was do you want KDE, or Gnome + KDE.  I chose the Gnome install but every menu has a mix of Gnome and KDE applications.  Under system utilities I have Gnome Terminal, Konsole, and other choices.  Under Internet apps I have Firefox, Mozilla, and Konqueror.  (The Gnome menu layout and icons are pretty much identical to the kde version (http://madpenguin.org/images/reviews/suse93/organization.png))I would have really preferred a Gnome-only install.  I was previously using their Novell Linux Desktop 9 and that's the way it worked, a mostly clean KDE or Gnome install.  I haven't installed the KDE option, but I'm guessing it doesn't have all the Gnome stuff mixed in (will try it and let you guys know).

By default you don't have MP3 and other multimedia codecs installed.  That's easily fixed durring installation when the Online Update runs.  You get a choice to install some Multimedia Packs, Macromedia Flash, and the commercial NVidia driver.  They all went on fine.  But after first booting into the system and running Yast to install some additional software, I was bitten by RPM-dependency hell.  There were lots (about 5 pages) of unresolved dependencies, I ended up putting the Install DVD back in and installing them all by hand.  I think the 32 bit versions of everything got installed but not the 64 bit, after about 30 minutes of installing RMPs by hand, everything was happy and I could get into Yast without errors.  

One thing I've gotten used to with Gnome is auto-detecting my printers.  My USB printer in SuSE 9.3 doesn't show up though (neither in yast or in Gnome CUPS manager).  Not sure what's going on with that yet.   (It did work in Novell Desktop 9, Ubuntu, and Solaris 10 though).

I did a 64-bit install of the OS, and was surprised when prompted to install Macromedia Flash.  But when I loaded up firefox and tried it out (http://www.homestarrunner.com/sbemail.html), it worked perfectly.  I verified the build of firefox by going to help->about, they've got a 64 bit build of FireFox running the flash plugin.  I don't know how, everything I've read says this is not supposed to work.  But it works fine.

They're running the Industrial Theme (which is also used in their Novell Desktop) in Gnome, but have included their KDE icons and menu layout which will really throw you off if you've used Gnome for any length of time.  

You can find some KDE and Gnome shots at http://madpenguin.org/cms/?m=show&id=3851&page=1

For developers, they're including KDevelop, Eclipse 3.0, and MonoDevelop.  By default only Java 1.4 runtime is installed, the JDK requires a trip to Yast to get it installed.  (I wanted to remove Java 1.4 and install 1.5, but they've got OpenOffice dependant to the 1.4 version, so doing that would require removing OOo, so I stuck with 1.4 for now).  They have chosen to stay with GCC version 3.3 instead of going to 4.0 (which is where the GNU java stuff really does well).  Going to try installing the java specific parts of GCC later and see how well that works.

I really prefer a less-molested Gnome install than SuSE 9.3 gives you - but it does work very well.  KDE fans would probably be especially happy with SuSE, but for Gnome I would rather stick with Novell Desktop.  (My only problem with that is they're on Gnome 2.6, which is too old for some of the development stuff I use).

And apparently making free copies is OK - so if anyone wants to try this out, let me know and I'll bring them to the next meeting.
