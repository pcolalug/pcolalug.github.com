I used RH 8.0 for just a short time. I did not like it. There was absolutely no compelling reason for me to make the move. I like Mandrake very well. (I have stayed with them long enough that I feel like I am finally learning a distro).

The only thing that I wished Mandrake would borrow from RH 8 that I saw was the launch menu. RH really simplified things (some people saw it as dumbing down, I thought it was a step forward). Instead of having 8 browsers avialable to you when you said that you want to browse the internet, there was just one. If you wanted to install others, they were available, but not installed by default. Same with just about every other category on the launch menu. I think that would make it nicer for the average user that just wants to do a task, not necessarily have to know the different advantages of each one. If there is one that I do like better, then I could install it. If you know enough about linux to know that you like Evolution over KMail, then you know enough to install it from the CD.

I like what RH has done with the menu, but I don't think they should be the ones doing it. RH should be offering choices and Mandrake would be the ones to spoon feed their target market. RH has made their clientel the server and sys admins. Those who should know more about all of this than the average user. People that would want a choice for a specific tool.

Mandrake has targeted the end user and consumer desktop market. They are the ones that don't need as much choice when first diving into Linux. RH has done a good job with the menu, but Mandrake should be the one implementing it.

I was disappointed when I installed Mandrake and found out that they did not follow along with this simplification. Other distros have.
