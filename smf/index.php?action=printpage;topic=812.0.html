Welcome to the boards Chris!

Quote from: Starraisin
I am looking for the perfect distribution of linux ( runs 60% of my windows software plus detect all my hardware.)


I don't think you are going to find this, but with a little googling and a few well asked questions, you can achieve a system that will hopefully perform up to your expectations. I'm not going to lie and tell you linux is easy. It takes work to run well and a willingness to learn new things, but that is why most of us run it. The 60% of windows software thing...I would suggest leaving that expectation behind and start looking for open source replacements. Wine, the reimplementation of the Win32 API used to run Windows software under linux, isn't that good. Also, why would you want to run Windows software under linux when it will run better under Windows? Hardware detection doesn't always work right under Windows, so once again, that may be an unrealistic expectation. If you come into the world of linux without the expectation of it being Windows, you will appreciate it a lot more.

One thing you'll find is that Linux users are never short on advice! I sugguest you get started with Knoppix (http://knoppix.net/). Its a cd-based distro, and by that I mean the whole thing runs from a cd. You pop it in, reboot from cd and blammo, you have a fully functional Linux desktop (complete with apps) without installing anything. It doesn't touch your installed system and mounts your local hard drives read-only. It has some of *the best* hardware detection around. If you like it, you can even install it on your hard drive. Its a great way to get the "feel" of Linux without spending time on partitioning, installing, and configuring.

Post by: imnes on June 25, 2003, 02:11:11 pm
Slackware (http://www.slackware.com).  If you're into doing things the old fashioned way.  

Mandrake (http://www.mandrake.com).  If you're into bloatware :)  (Sorry, I know like 90% of the group uses Mandrake but I couldn't help myself).

SuSE (http://www.suse.com).  If you like using graphical tools to do everything.  (Unfortunately those same tools cause too many problems for  me).

RedHat (http://www.redhat.com).  If you like to be standard.  Since most everything available for linux is written on or for RedHat, and several distros are based off of it.

hmm...

Debian (http://www.debian.com).  If you dare.  I hear apt-get is nice once you get used to it.  So is emacs, but I haven't bothered with that either.

Gentoo (http://www.gentoo.org).  If you like to watch compiles.

You can grab cd-based "live" copies of several different distros to try them out without having to actually install.  And they work d**n good.  Knoppix is one, but SuSE and some of the others offer it also.

When it comes right down to it though, any distro will offer you the same functionality and access to the same programs.  So it doesn't really matter what you choose.  

You have a few choices to get your windows apps going, and they're all wine.  If you have a good mix of programs to run, use the generic Wine (http://www.winehq.com).  If you're mostly into getting MS Office and business apps to run, might look into the latest sources from Crossover Office (http://www.codeweavers.com/products/office/source.php).  And if you're wanting to get games working, I guess the best bet would be the latest wine sources from Transgaming's WineX (http://www.transgaming.com/).

Wine is a bit tricky, some programs work perfectly well, some require major tweaking to the wine configuration.  If you let me know which apps specifically you're trying to run I can let you know if I've had success with them.

Good luck!

Nick

Quote from: imnes
RedHat. If you like to be standard. Since most everything available for linux is written on or for RedHat, and several distros are based off of it.


Dear lord, that is a scary comment. I really don't see most developers using Red Hat for some reason. Red Hat is in my mind more of the "linux for the masses" than any other. Yes, Mandrake is geared toward desktop use and beginners, but Red Hat seems to try to do everything for everyone. Also, the reason so many people use RPMs for packaging is because so many distros use it and since it is the Red Hat Package Management system, the developers and package maintainers default to Red Hat as being the distro to base the RPMs on.

And by the way, RPMs suck bad. Besides dependency hell, I don't know if any of you have looked at making an RPM from a source package. It isn't fun. Debian's .deb isn't much better, but at least you don't have to deal with dependencies thanks to apt. And portage with Gentoo is just sweet. Yes, compiles can take a while, but that is why you run several systems and put distcc on them. Ah, distributed compiling is so nice.

Finally, apt isn't hard to use. I think I'll write an article when I get home on the simplicity that is apt. If you can use the command line at all, you can use apt.
