Right now I am working on getting all of my linear algebra (matrix and
vector-called BLAS in NA speak) subroutines installed and optimized. It has become a bit of a pain in the ass.  But I am learing quite a bit as I go regarding cpu architectures. I hope to start testing routines soon (next week) to see which different routines run faster. All redhat boxes come w/ the GNU Scientific Library which has much of this stuff (rpm -q gsl for your version) , but I have installed ATLAS and LAPACK both of which build architecture dependent (optimized) BLAS routines. I want to run timing runs for the same problems and compare times and times to flops (floating point operations) for the problems so that I will know which routines run with the greatest accuracy and speed. If you want to check out some cool websites which discusses all of this and more go to
1> http://gams.nist.gov or
2> http://www.netlib.org
3> http://www.gnu.org has the source code for maxima (computer algebra system (you need lisp to compile) and algae (NA language developed by Boeing) or octave.
