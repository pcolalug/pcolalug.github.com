In case you haven't heard of GG, this is a company started by a couple of ex-Dynamix folks. They continue the development of the Torque engine that powered Tribes 2. They sell this very capable engine at $100 a developer (*mega* cheap for a commercial game engine) and have built a community of independent and hobbiest game makers. Lots of tips, code, and other support for the budding hacker hoping to start the next Id to the geek just messing around late at night. Also, should your game, once completed, show promise, GG will partner with you as publisher to help you get your game to the masses. Lots of neat (and not-so-neat) projects going on over there. Take some time and browse around the "Make Games" section if you have a few.

Our requirements, loose as they are, are only that a native Linux binary be (legally) atainable. If the game in question has a Windows version, you are more than welcome to game on the platform of your choice.

The Pcola LUG does not descriminate against platforms. As Linux users, we kinda know what that is like and should not push any platform bias on anyone. Pcola LUG will not endorse nor espouse, in any official capacity, "bashing" of one platform over another. Debates on technical merits are fine, but bigotry is frowned upon.

You make the technology choices that are best for you. We only insist on the above guideline so that those of us who have "taken the plunge" and gone OSS-only can play. Also, as an OSS advocacy group, this shows support for the companies that make cross-platform games.

Quote from: Starraisin
I would like to switch to linux but have not yet. I am looking for the perfect distribution.


Firstly, there is no "perfect" distribution. If such a thing existed, you would know about it by now. Distributions are sort of like butt holes, everyone has a favorite...wait that didn't come out right. Anyway, everyone has a favorite and the only way to find out which one you like the best is by using one. No, you don't have to take the plunge and run it all the time, but if you are interested in learning about linux or any of the other unix work alikes, pick one and get it installed. I found that I learned best by having a particular goal in mind. You first goal should be to get a fully functional system up. In doing that, you will find other goals...well, hopefully.

Quote from: Starraisin
I don't think matters what OS as long as there is a client for it.


Think of the game clients as just that clients. The server and the client don't have to have the same underlying OS. Think of a web client or browser and a web server. The majority of the webservers out there on the net are running some form of Apache on some form of unix or unix work alike and the majority of the browsers run on some other OS. There is an underlying protocol that the two talk to each other and that is what matters. When we say we only look for at least some version that is native to linux, we want there to at least be a client available that has been compiled to run in linux. I don't know a single instance that a game will not interact with itself because one of the players is using linux and the other is using windows as long as the two are of the same version.
