Title: Re: Home networking
Post by: rugmonster on April 27, 2006, 04:02:54 pm
Depending on the type of partition he stored the stuff on under Linux, he has options for copying the data back inside of windows.

For ReiserFS, there is YAReG (http://yareg.akucom.de/) and for Ext2/3 there is Ext2IFS (http://www.fs-driver.org/). YAReG is an application that is slow to copy large files, but works. Note that it is only for ReiserFS. Ext2IFS is a driver for Windows so that Ext2 and 3 volumes show up as native drives. I have not tested the Ext2IFS, but have read good things about it.

As for copying between Samba, NFS, etc. Yeah, it will work. Probably the easiest thing would be to use scp between the linux boxes and then have him boot into Windows and make them both on the same Samba workgroup. There are numerous tools out there to set Samba up. Distros like SUSE, Fedora and Mandriva should have GUI based tools for it included.

Title: Re: Home networking
Post by: danc on April 27, 2006, 10:21:49 pm
I've always found Samba to be a pain, but admittedly have not really put much effort into figuring it out.

What I would do is to use "scp" to copy the files from his Linux partition to your Linux machine.  Then, after he reinstalls XPee, I would just install "putty" and "pscp" (from same place as putty), and use "pscp" to copy stuff back into the windoze machine from your Linux box.  This way there is no need for setting up servers and thrashing around with Samba/NFS.

As usual with Linux, there are many ways to do the same thing, but I would find this to be the easiest.
