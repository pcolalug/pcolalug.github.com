Title: Question about server Apache
Post by: imnes on April 12, 2003, 10:43:01 am
That's correct.  

And actually I would leave off the / and just link from where your page is to where the next page is.  So if they're in the same directory, don't have and / at all.  If it's back one directory do ../

If it's in a subdirectory called users then do users/


That way even if you're script moves to a new subdirectory directory on another site all the links will still work.

Here's a snippet of how I link files

Code:

Backup one directory, then under a users directory...
<a href="../users/login.html">Login</a>

File is in the same directory as this script...
<a href="change.html">Change your password</a>

File is in a subdirectory under this one...
<a href="pics/thebighouse.gif">A Big Ole House</a>

Title: Question about server Apache
Post by: dpeach on April 12, 2003, 02:30:36 pm
All this should work as imnes has stated. Apache should be configured so that the default html directory is (depending on the system) something like:
/var/www/html

This is where you will put your pages so that what Nick is saying is true. If you have chosen to put your pages in another directory like /home/pages (which would not be best, but doable) you would need to tell Apache where the "site root" is. Also, by doing it Nick's way, you will be able to do all of your building and testing in one directory and then just move into your main directory when you are ready to go live and it should all work.
