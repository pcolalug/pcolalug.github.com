Okay, the K12LTSP server for Camp Fire USA is up and running.  I downloaded and burned the 6 CD's for K12LTSP 6.0.  I ran into several problems of my own making that took me a while to figure out.  Apparently, I turned off a few services that are needed by LTS.  Anyway, finally got past those problems.

To perform a basic test, I used my home computer as a client.  It is PXE enabled, so I didn't need to make a floppy boot disk.  The client booted across the network from the LTS.  Currently, I am using the client to post this message.

Sometime next week, I will try to boot the Camp Fire client using an etherboot floppy, which I still need to create.  Once I get that to work, I will create copies of the boot floppy.  I wish the clients were PXE enabled, it would simplify things, but we have to work with what we have.  Also, I need to try and setup network printing and sound.  Lucky me.

A few things to think about:
- On the client machines, we probably should either remove the hard drives, or at least disconnect the cables.

- If the kids are going to be able to save their files on the server, then they probably should have their own user id's.  Of couse, this adds administration overhead.  If we use generic user id's, like user1, user2, etc., then the kids won't be able to save files that only they can access.  Also, is there a way for the kids to use the clients without requiring a user id and password?  Of course, this eliminates the possibility of saving personal files.

- Just a reminder that the server can use more memory, if anyone has some extra.  Also, we need a switch.

- Need to think about the best ways to lockdown aspiring hackers.

- Need to filter internet content for inappropriate material.

- Need to setup the firewall.

- Other ideas?

It was great working with you guys today.
