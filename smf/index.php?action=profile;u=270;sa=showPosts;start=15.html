Okay, now I need to add another    for mispelling kmix.

I did do some reseach on the problem to see why it was the way it was. I have an emachines labtop, and finding out exactly what sound you have is pretty tough...well, for me it was. The docs that come with the computer didn't have it, tech support doesn't either, just that it is AC 97 compliant. Since most of the time you never really hear good things about people, here's one...the guy at tech support was also a linux user and even though they don't support the OS he was pretty helpful. In fact, he give one of the better "newbie tips" that I have heard. Find a live cd that is totally supported and everything works and keep it handy to use for troubleshooting. If you load another version of linux and it doesn't totally work, check the live cd setting to see if something is different. I didn't have to do it, but it seemed like a great idea.

After reading the kmix handbook, the SuSE documentation, looking through forums all over the internet I didn't find any type documentation anywhere that covered it. The Kmix handbook did briefly explain the sliders, but not indepth. What I finally concluded was that the external amp switch affects the system the same way a headset does when you plug them in. Basically its telling your system to bypass the speakers because if you're using an external amp you're probably using external speakers anyway. I don't have an external amp and speaker set to test it out on so I don't know if they would have worked had I hooked it that way to start off it. I am also not sure if this problem pertains strickly to labtops, but it seemed to me like the majority of the people having the problem did have labtops. All of the sound works now though. Not a very technical answer, but it makes sense to me.
