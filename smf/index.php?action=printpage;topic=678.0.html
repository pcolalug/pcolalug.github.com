Title: What is a runlevel?
Post by: rburt3 on May 06, 2003, 07:44:09 am
Yes and no. Runlevels define a set of functions and services. When you boot, say into runlevel 2, that's typically "single user" mode, which means just enough stuff is running to call it a computer. Runlevel 3 brings in the networking services and typically any servers are started up. Not sure about runlevel 4. Runlevel 5 typically includes windowing services (X and the like). Runlevel 6 is reboot while runlevel 0 is shutdown.

You can think of runlevels as "stages" in the boot process. Only with the Unicies, you have more fine grain control over what services get started when.

A tip. You can change run levels from the command line with the
Code:
telinit
command. See your fine man pages for details.

Title: What is a runlevel?
Post by: imnes on May 07, 2003, 08:20:31 am
If you look at the file /etc/inittab it will describe the runlevels used on your system, for mine this is (I'm just including part of it):

Code:

# These are the default runlevels in Slackware:
#   0 = halt
#   1 = single user mode
#   2 = unused (but configured the same as runlevel 3)
#   3 = multiuser mode (default Slackware runlevel)
#   4 = X11 with KDM/GDM/XDM (session managers)
#   5 = unused (but configured the same as runlevel 3)
#   6 = reboot

# Default runlevel. (Do not set to 0 or 6)
id:3:initdefault:


So if I were to edit that file, and change that last line to id:4:initdefault: my computer would boot straight into X Windows.
