It's been said before...      
by Anonymous Reader on 2002.12.14 21:40 (#36726)
but I have to say this again in my own words. I don't consider myself one of the choir boys for OSS, but I do support it. It seems that you propose we get behind one distribution and one user environment and support that. What then would make that one distro with that one user interface any different from the lack of choice we get from MS?

The point of open source software is freedom. The freedom to choose which software you will use and the freedom to change that software if it fits your needs better to do so. If one distro does something right as a whole, then people will get behind it. Gnome and KDE have become popular and share a majority, but don't steal the thunder from WindowMaker and Blackbox and all the other WINDOW MANAGERS which provide desktop environments. Not everyone likes KDE and not everyone likes Gnome, so they use an alternative if they even use X to begin with. That is their choice. If we eliminate that choice, then what have we gained? Another company or non-profit organization dictating to us what the "standards" will be.

It makes me thankful that we have the developers who are responsible for these fine open source products who think differently than you and believe in choice. Standards are good, but don't eliminate the choice to use something else or develop new standards.

I'm gonna play Devil's Advocate here. (So, what else is new?)

Anyway, I like the choice that Linux gives us, but,  it's the choice that's holding Linux back from the mainstream and from truely competing with MS. While choice is good, it comes at a price. The price is compatibility.

I run Mandrake at home using Gnome as my desktop. Whenever a neat, new application comes out that I'd like to run, I have to ask some questions before I install it.

1. Has it been packaged for Mandrake? Each distro uses it's own set of libraries and other things that make it slightly different from other distros. This means that in order for apps to run on your distro they have to be "packaged" for it or you have to become a "Linux Guru" and learn how to install from source. I would never expect my Mom, brothers, or Sisters to learn to install from source. Windows, while not giving the choice, is compatible with all windows software. (Granted you must be running the right version but the same holds true for the Linux kernel you're running.)

2. Will it run in my Window manager? Many KDE apps require the KDE libraries in order to run. Granted, the KDE libraries can and usually are loaded even when you run Gnome but this is just a lot more overhead. Besides, if I'm running a KDE app in Gnome, it runs slower than it does under KDE. (That's because it has to load the KDE libraries in the background) The same holds true the other way around.

3. If it has been packaged for Mandrake and does run under Gnome, then I still have to ask the killer question. What dependancies does it have? Will I have to find and  install a dozen other apps in order for this one to run?

So, because I have choices, I also have a lot of work when deciding on loading an application. In the MS world a program written to run on Windows XP installs without any decisions needing to be made. Can you say "Next"? The only questions you need to ask yourself with Windows apps are: Am I running a compatible version of Windows? Do I have enough memory? Is my processor powerfull enough? Each of these questions also holds true for Linux as well as the other questions I mentioned about.

By giving me all of the choices that Linux does, it requires that I be a lot more knowledgable about my system and be a lot more carefull when installing software. Thus, it isolates the average Joe Windows user from a very reliable and powerfull operating system.

Okay, you have all the good points as to why linux isn't making it on the desktop and I agree for the most part (except the one on how it takes a "guru" to install from source). Choice does have its price, but isn't it the distribution makers job to provide their users with packages and a package manager that is intelligent enough to solve dependency problems? Who is to say that Redhat and Mandrake and SuSE and everyone else should bow down to apt or portage because they do the job better?

Every distro has a focus and some, like Lindows and Lycoris, focus on the desktop user. They should be the ones coming up with ways to make software installation easy and user-friendly, ie asking as few questions as possible and giving default setups. Then there are the distros like Mandrake, Redhat and SuSE who are sort of middle of the road. They want to focus on usability, but still give the user more choices up front and plenty if you dig. Then there are distros like Slackware, Debian, Gentoo and the non-distro distro of LFS who are focused on the "power user" who want choices and like to customize things to their liking. Let's not forget the specialty distros either. The fact that you have a choice is the key that I'm getting at. Let the desktop distros dumb things down so Joe Sixpack can discover the joys of OSS and make their desktops pretty and frilly, but don't merge all of your distros together and settle for just one way to do it. The author seemed to me to be saying that we should all unite and form one distro and one package management system and one standard for the way "Linux" works. This can easily be thrown to the argument of whether Linux is an OS or just a kernel. Is the OS what the distributions put together or is the OS the combination of the GNU projects endevours combined with the Linux kernel? That is a whole other ball of wax.

But yes, your points are valid, but it doesn't apply to how would we implement this "united" front as a Windows killer? To me Linux just isn't suited well as a Joe Sixpack friendly desktop solution by default. It is too modular and has too many options. It takes some thought to setup! Let the Lycoris and Lindows folks make that easy solution though and answer some of the questions for good ol' Joe and make it easier for him to enjoy the benefits of OSS, but don't take away choices from those of us who appreciate them. If you decide to use a distribution that doesn't answer those questions for you, then it should be expected that you learn how to handle dependencies (if you have an inferior package manager ) and to read the documentation and to "roll your own" occasionally. You will probably end up with a better system in the long run than Mr. Sixpack, but you still have to go through the long part of the run to get there which takes patience and thought.
