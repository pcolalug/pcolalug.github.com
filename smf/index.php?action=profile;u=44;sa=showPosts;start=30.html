31  Distributions / Debian Based Distrtibutions / Re: Ubuntu 7.04 (fiesty fawn) Released on: April 21, 2007, 07:47:53 am
Further notes on Feisty upgrade.

I have an AD, internal DNS domain, rugmonster.local, that I use for authentication. Since Avahi uses a .local namespace, I could not resolve any of my internal machine's names by FQDN due to Avahi's mdns being checked prior to dns in nsswitch.conf. To fix the issue (and since I don't use Avahi for crap), I changed my nsswitch.conf to the following:

hosts:          files dns mdns4_minimal [NOTFOUND=return] mdns4

The impact of all of that went beyond not being able to resolve names properly via DNS. I could not authenticate against the AD on my Ubuntu machine because Kerberos could not talk to the KDC since it depends on DNS for name resolution.

So if you're having problems where you can resolve a host by hostname alone, but not by FQDN and have a domain with .local as the TLD, then you may want to try this.

Other than that, it's been smooth sailing. The home file server is getting the treatment now, followed by the business's web server.

38  Software / Programming/Development / Re: Mono Project on: February 24, 2007, 12:33:49 pm
Have you used Java?

The whole .NET thing is sort of the same thing. The major difference is that you can write your app in one of several languages, the most popular being C#. When you compile the source, it is translated into the Common Runtime Language and then compiled to bytecode. The bytecode, like in Java, is then handled by a Just-In-Time compiler which is platform specific. Microsoft has their official .NET JIT compiler in the form of the .NET Framework, just like Sun has the Sun Java 2 Standard/Enterprise/etc Edition Runtime Environment. Mono is attempting to bring the standard libraries of the MS .NET Framework, as well as some additional libraries, with their own JIT compiler, much like the various third part Java Runtime Environments from GNU, IBM, etc.

So the idea is that you can write an app that uses the supported libraries in Mono and run that app in Linux, Solaris, Windows, Mac OS X, etc, and use any of the Mono supported languages, C# being the primary one at this point. The idea is that you can bring any other language to the framework provided you provide translations to the common runtime language.

My impression of it is that the libraries are expansive. The classes have too many properties and methods to remember all but the most common, so you'll likely be doing a lot of reference searches for a while. This is true of anything .NET. I've never become a fan, but I'm not a developer. I know folks that love the .NET stuff. I know Java guys that find things in .NET "interesting", but nothing especially new or exciting. Generally, the focus is different with the Java crowd. I wish I could tell you more, but my experience with it is very limited because I was turned off by it.

If you've ever used Beagle, F-Spot, Tomboy notes, or Blam RSS Reader, you've used a Mono app.

44  Distributions / RPM Based Distributions / Re: CHMOD FOR FC5 on: January 24, 2007, 10:40:28 pm
http://www.analysisandsolutions.com/code/chmod.htm

First, create a group that you want to have access to the folder.

As root:
Code:
# groupadd <groupname>

Then, add the users to the group.

As root again:
Code:
# usermod -a -G <groupname> <user>

Repeat this for each user.

Explanation:
usermod - change a user's account settings
   -a  append the user to the secondary group(s) listed after....
   -G list of secondary group(s) you want to add the user to

Why would you do that? You can give the folder the desired permission to that group. If you need to give the same permission to someone else, all you have to do is put them in the group you created above.

BIG NOTE! If you add a user to a group, they have to log out and log back in before their session will register that they are part of that new group!!!

So to set the permission on the folder in question, you use chmod. Here's the quick and dirty of it. You have Read, Write, and Execute for your file permissions. Those file permissions apply to three classes of permissions: User, Group, and Other.

A note on execute on directories. If you want someone to be able to list the contents of a directory, they need both read AND execute permissions on that directory.

So let's say you own the directory. We'll call it /usr/share/files. If you did an 'ls -l' in /usr/share, (assuming no other files or directories existed in /usr/share), you would see something like this.

Code:
drwxr-xr-x   7 bigtoss users        94 2007-01-21 16:22 files

RWX...that's Read, Write and Execute. Notice that there are three sets, RWX for User (bigtoss), R-X for Group (users), R-X for Others (everyone else on the system). Now, let's say you want to make the group 'files-users' have read-only permissions on that directory. You need to use 'chown'.

Code:
# chown bigtoss:files-users /usr/share/files

chown changes file and directory ownership, where the user comes first and then group, separated by a colon. Okay, so now the output of 'ls -l' will look like this.

Code:
drwxr-xr-x   7 bigtoss files-users        94 2007-01-21 16:22 files

To give files-users Write access to the directory, you use chmod. Here's what it would look like.

Code:
# chmod 775 files

Alternatively...

Code:
# chmod g+w files

The numbers represent RWX.

Values for the math

R = 4
W = 2
X = 1

So if you add up all three, you get 7. If you add the values for R and X, you get 5. See how that works? The other method is to add or subtract specific permissions (r, w, or x) for a particular class (user, group, other). So g+w means the group is added write permission. If you then did g-w, then you would take write permission away from them. u stands for user, g for group, and o for other (or everyone else).

So there you go. Quick and dirty. If you still can't figure it out between this and the link posted at the top of this, give us the group name that the users belong to, the directory and what you want them to be able to do in that directory (read/write or just read). We'll give you what you need to put in to make it happen.
