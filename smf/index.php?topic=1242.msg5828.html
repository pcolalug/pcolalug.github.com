I use Linux for most stuff (95%) when I am home. I use Windows for most stuff (99.99%) when I am on the road. Difference? I have an Internet connection that works in Linux at home, but not when I am traveling.

My gammut of "must have" apps include: OpenOffice.org, Firebird, Thunderbird, Gaim. These all run in Windows and Linux. I have one database that currently lives in Windows only. I use it once a month. I can export the data and work with it in Linux, but it looks nice in Windows, so that is where it stays.

There is nothing I need Windows for except...GAMES. Currently, since my favorite game is ET I could do anything in Linux that I needed to (even get to my database if I really wanted). But for the most part, the games I like to play are Windows based.

I use Linux mostly out of practicality. I like to mess with stuff (my wife thinks I just like to mess up stuff). Because of my desire to change stuff all the time, Linux has put life back into computing. I cannot afford/don't choose to buy the new version of Windows and Microsoft products every time they come out. With Linux I can install a new distro every week for the rest of my life and never run out of options.

I don't think Microsoft is an "evil monopoly." In fact, I think it is great that Bill Gates can make a bazillion dollars off of the general public by putting out inferior products. But he is not going to get my money if I can help it.

I think Linux is far superior in one area: stability. Other than that, open source guys really need to work on making their software work like the OS model says it should. Really, much of the OS projects out there are just that, a project not a useable program. They cannot seriously think that businesses are going to choose their project over a commercial product that does the same thing, but in a better way.

Examples of good OS projects:
OpenOffice.org
Mozilla
Firebird
Thunderbird

Examples of bad ones:
The GIMP
Any Quicken replacement
Countless scores of really half baked projects

The GIMP is great, but it has been in it's current state with minor adjustments/developments for years. Years. YEARS! Not months. Yes, it works, but the UI is awful IMO. Any company that is serious about bringing in the best designers is going to choose Photoshop over The GIMP. Who wants to teach that awful UI?

Quicken replacement. There have been many that have tried and failed. Not because they don't do the math right, but the UI once again. Sure there are some improved ones out there. But this is a program that my wife would use. Why would I make her go back to Windows 3.1 days of UI to do something that she currently can do with nice looking software?

UI is the biggest problem I have with many Linux programs. The ones I like are mostly because of their UI. Firebird is far better than IE. That one is a no brainer. If someone would give Firebird a try for a week and only learned how to use tabbed browsing, they would never want to go back to IE. I have tried other browsers that use tabbed browsing, but none of them quite do it as well as Firebird. Plus all the other features that Firebird gives you. UI will keep me using this product.

OO.o is a wash on UI. I have not used Microsoft Office enough to know anything about it in 14 years or so. Therefore I really cannot comment about the UI in it. It works well for me. I  understand the program and think that the UI is at least workable. The UI may not be the same as Office, but who says Microsoft has the best UI in the world?

Thunderbird is a wash as well to me. I really like the program, especially since the 0.4 version came out. I don't know that it is necessarily superior to OE, but it is at least as good as. The UI is useable.

Other programs may work, but the interface is horrid. There are some things I really hate about every window manager I have tried in Linux. My environment of choice is KDE. But there are some things that are just awful there. Linux suffers because there is not enough emphasis placed on UI. Sure the OS program may be better and be able to do more, but if you cannot figure out how to do what you want, or you can't stand looking at the ugly interface, what is going to keep you using the program?

I use Linux for several reasons. I like the OS model. But until some work is done on the UI, it will not be on the desktop of every computer in the world.

EDIT: I have to agree with Lloyd that Linux is absolutely necessary for any kind of server application. I would never consider a Windows based server of anything. Even just a simple file server.

Why have I moved more toward Windows for the desktop? I started thinking about that the other day. I have had no problems with my Windows XP desktop. I remember XP prior to service pack 1 and thought it was terrible. Since then, it's actually grown on me. I got used to where things moved to and I found there was a lot more I could adjust to my liking than there was in 2000. Believe it or not, it's stable! The system I have running it is a beast, but I get things done and I don't have to worry about messing with it or poorly written beta software.

As for my software, I am using a mix of Firebird and IE 6.0. I use Firebird for the tabbed browsing. I use IE when Firebird won't render a page correctly. Blame the page author for not making a standards based page, but regardless, I have to look at those pages from time to time to get my job done.

For my office solution, I use Office XP. Let's face it, OOo is slow and god is it a resource hog. I refuse to use it because it just doesn't get the job done like Office. I've got Office 2003 on my laptop and I like it even more than Office XP. The features are mostly the same, but there are little differences that make it awesome, most notably the improvement of Outlook. I use Outlook every single day for about 50% of my job.

We use Exchange for our mail servers and for as large a scale system as the AF has, Exchange works very well. Yes, we have services fail, but there isn't as much administrative overhead for maintaining the Address Directory for the ENTIRE AIR FORCE as would be required for a system based on LDAP or something similar. Administration of Exchange is pretty nice in fact and I have yet to find a product that does the complete job Exchange does. I would like to see the product by SuSE, but I'm not going out of my way for it.

Now, where do I use linux and why? I can't think of a better use of linux as a web or database server, a pure email server (not groupware like Exchange), a simple file server (not functioning in a Windows domain...more later), or ftp server. And who can deny dedicated game servers. I use my linux machines for programming (now) and I do occasionally like to use the cheap alternative software to mainstream products that would cost me extra...as long as they don't give me too much headache.

Why don't I like Samba in a Windows domain? Right now, we are trying to get a Samba server setup in our NT domain. What we are trying to do is extremely easy on a Windows machine; control who has rights for reading and writing to the share. Samba 3.0 is supposed to work with the Windows management tools for shares. You know what? They don't. The server will barely acknowledge our configurations in the smb.conf. Now I use Samba at home to share files between my linux boxes and my windows box, but here, it sucks. I would hate having to use it as a Domain Controller here considering how many users we have. It may be faster, but not from an administrative standpoint.

Why do I like the solutions I do? Because once they are setup, they run. I don't have to mess with them. Initial work is at least twice as much, but in the long run they save me time, but those are servers. As a user, where I have to interact with the environment, I spend more time making things work, usually work like they would in a Windows environment, than I do getting done what it is I'm trying to get done.

That leads into another huge point, most Linux software is just doing something that someone else has already done! How many open source projects are there that are unique? How many times do you see someone writing another solution to a problem that's been solved five times before? If you don't believe me, go to Freshmeat and look around for a while. Also, there is this geek bandwagon that any time you write something, it needs to be opensource and needs a project behind it. Brave souls, they may be for putting their crappy code for everyone to make fun of, but sort of wasting their own time if you ask me. Shouldn't there be more to life than getting included in a Red Hat release along with four other products that do the same thing?

I also want to address the misconception that linux is magically more secure than Windows. Yes, Windows does have a large number of exploits out there, but have you seen how many exploits there are for BIND or WU-FTPD? Almost every system has some remote exploit of one kind or another. The only way to truely make a system secure is to disconnect it. A good administrator will keep up with his patches and updates, thus avoiding vulnerabilities. Most worms come out months after a vulnerability is found. How many root exploits do you know of for Windows and how many are there for Linux? I know of a lot more for Linux server software and even more for local exploits. I believe the whole misconception comes from the fact that fewer people target linux and there are fewer people running linux than there are windows users and windows targetted exploits. As more people start using linux and don't keep up with their updates, their systems will too become hosts for the latest DDoS or worm.

This may seem like a huge anti-linux rant, but it's not. I'm just getting fed up with the state that most Free/Open Source software is in and a lot of the zealot mentality of some of our fellow users. I've had to look at software solutions from a point of view that I need something that works and works reliably and Linux just doesn't fit that bill as often for the tasks I need to get done. Sorry guys. I still love to use it for what I love to use it for and it is fun. I hope that some day I can be a Unix administrator because I have more fun doing it, but there are just some things done better in Windows.

By the way, it's W-I-N-D-O-W-S. It sounds rather ignorant to call it Winblows as well as elitist. Regardless, Windows has a place just as every other OS. It really doesn't make you look smarter or more l33t by calling it Winblows. It only scares off people who might think of trying Linux out for the first time.

Well wine has come a long way.  It can run most windows apps flawlessly, and winex runs most games at or near native speed.  So (unless you have some special case apps) you could probably get by, running the same software, on either.  So the choice of running windows / linux is less about which apps and games are available, and more about which OS you feel more comfortable in.  After using 100% linux for a while now I have a hard time working in Windows, things just don't work the way I'm used to.  Not saying anything bad about windows, XP is stable if you have good hardware and a decent machine.  For me, device support is a little better in Linux, I've got a windows-only printer and USB camera that only work in windows before XP (or maybe even 2k) but do work fine in Linux.  

After playing with Gimp and Photoshop a little today, the interfaces seem nearly the same?  The only problem I have with both is I like my programs to use MDI (I think that's the term), where the toolbars and open-file windows and everything are contained within a window, photoshop and gimp just let you drag the toolbars around the screen wherever you want.  But that's the way the Apple style guru's say that everything should be made...
