Title: freebsd calcru negative time problems
Post by: something on July 02, 2004, 11:23:48 am
I just installed freebsd 5.2.1 yesterday on my AMD k6 2 500mhz machine.  The install went well (to my knowledge), after I bootup everything seems to be running fine except whenever a process needs to run I get this message on the screen: 'calcru: negative time of -582613 usec for pid 672 (sh)' , of course this is not the only one, there are many different ones, but one appears on the screen at a time.  The time it's always negative, from what I understand, whenever the kernel sees something weird going on it will output that on the screen.  I googled for a while, and found out many other people are having this problem too, but noone really talked about a way to fixing it.

I did find this fix: 'sysctl -w kern.timecounter.method=1', after I typed that on the console it didn't help (I actually got some error, but I can't remember what it was, yes, I checked the syntax many times), I have read some posts saying that might fix it but with some performance lost.  I would like try to fix it without any performance lost if I can avoid it.

If anyone has this problem before and know a fix please let me know, thanx in advance.
