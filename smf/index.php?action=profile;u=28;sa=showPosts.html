iptables -A INPUT -p ALL -i lo -s 127.0.0.1 -j ACCEPT
iptables -A INPUT -p ALL -i lo -s 10.0.0.1 -j ACCEPT
iptables -A INPUT -p ALL -i lo -s xx.x.xxx.xxxx -j ACCEPT



change to:
iptables -A INPUT -p ALL -i lo -s 127.0.0.1 -j ACCEPT
iptables -A INPUT -p ALL -i eth0 -s 10.0.0.1 -j ACCEPT
iptables -A INPUT -p ALL -i eth0 -s xx.x.xxx.xxxx -j ACCEPT


Nothing but 127.0.0.1 should be on the lo interface. I am assuming you have two nics (eth0 & eth1)

Realize I'm not that great with using iptables as a firewall as I use pf & openBSD instead. Something else to consider is what DSL adapter are you using? Most of the ones I have seen you set the outside IP address on the router, and everything behind it is 192.168.0.x

12  Club Stuff / Club discussions / Re: First Annual hack attack on: June 08, 2006, 07:39:36 am
I don't have time to tinker with this, but if I did, here's how I'd attack the problem:

Don't look directly at the box you want to attack - of course its locked down completely from the outside, otherwise this would be a short contest. I would start by sniffing the entire slackwolf.com subnet and find a vulnerable server, and literally attack it from behind. Just a quickie glance I found DNS servers on 66.80.196.243 & .244, a mail server on .245 and I'm sure there's an exploit for those somewhere. (Lloyd, you're brave for doing this on you own network) The idea is to break into the box indirectly through a service/IP/port that it trusts to be safe from within. (As a bonus hint, look for exploits that use IRC, since we know Lloyd uses IRC, there's probably not a rule blocking outbound IRC (6667) traffic. Once you have a hole via IRC outbound, chances are the firewall will keep state, and then let you inject directly through the firewall via port 6667)

From the black hat perspective, breaking into a firewall nets you very little in comparison to breaking into the systems that the firewall was meant to protect. 

For a future contest, what you should do is setup a typical office setting (for me, that's an OpenBSD firewall with Linux and/or Windows servers behind it, and allowing typical services such as ssh, remote desktop, web, mail, dns, etc). Put a file on one of the servers named: "content_winner_read_me" with some code in it so whoever discloses the contents of the file first wins.
