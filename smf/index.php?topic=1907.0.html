Quote
I am still having a hard time understanding why people are so hesitant to try Gentoo.


Ruggy, no offense, but I've tried installing gentoo.  I consider myself a fairly competent linux user, but I draw the line with gentoo's install process.  Frankly it is far more complex than it needs to be.

There is no reason a user should be forced to hand edit most of his config files in the process of a normal vanilla install.  There should be an ncurses program (like that in slackware) that steps the user through this process in a reasonably intelligent manner.  

I like the way slackware has a smart set of base libraries which you can use to install most anything.  I remember trying to emerge the emu10k1 sound module in gentoo.  Emerge said that in order to install this one tiny sound module, I should have vim, and to have him i really should have X windows, and to have X I really should have gnome, and to have gnome...you get the idea.

This to me is a sign that gentoo's deps aren't really any better than debian or mandrake, despite the fact that gentoo compiles everything, and therefore should have a much less trouble with deps than binary packages.  


I have read several independant studies that pitted gentoos 'optimizations' against standard binary packages from debian, slackware, madrake, etc.  For all the time and effort spent compiling every package they have not been proven significantly faster than binaries.

I would bet that dropline got slashdotted since this topic was on there last night.

As for Gentoo, perhaps I'm just different than most of the people here. I don't want my distribution to hide stuff from me. I want to get into config files. I always have, even when I had piece of crap buggy frontends for them. I'm just that kind of user. I like to get into the thick of it. I used to tell people that in order to become truely successful linux users, they needed to be comfortable with the command line, learn how to read manpages and welcome editing config files by hand. I guess I was wrong on that.

Gentoo is about making your install of linux run like you want it to. The reason you were asked to install X and Gnome when trying to install the driver was you had a USE flag set for X and probably Gnome. The nice thing is you can tell it to ignore those. Most binary distributions simple won't work if you ignore a dependency like that. You're stuck unless you compile on your own and then you're doing the work of Portage on a different distro.

In the past year, I've been able to follow the instructions in the Gentoo handbook, step by step to get a working install. Anything else I wanted to know, I got the answer in the forums. It takes work to get a truely optimized system, but it can be done and there are several great howtos in the forums.

Gentoo isn't that complicated, but there is a learning curve. I have found that while on that curve I've learned more about linux in general than with any other distribution. Gentoo is the right distribution for me because it doesn't hide anything from me. It takes all of the really tedious work out of building a system from scratch, but still allowing me to have an install that feels like my own. Linux is one of my hobbies, and Unix is part of my job. I love the details of it and I'm always wanting to know more about it. It has motivated me to want to learn how to program so that I may better understand the true inner workings of the system and surrounding programs and that I may some day contribute back to the community.

I'll keep my Gentoo and you guys can keep whatever it is you run. Since coming here, I've run pretty much whatever was unpopular at the time. At first it was Debian, now it's this. If you ever need help with getting Gentoo installed, or post-installation, don't be afraid to ask me or one of the other Gentoo users. We're here and ready to share the good word with you.
