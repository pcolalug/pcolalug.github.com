Best server distro?
« on: June 11, 2004, 01:16:28 am »
I am in the (un-)enviable position of having to replace a rather aging and non-supported RedHat 7.2 distro with something else.  This is a development server, so (1) it's gotta be Linux, and (2) it's gotta be small, fast, and no X!

The server is an AMD K2/300 with 192MB of RAM, a 10GB root partition (contains OS, home directories, email), and other drives for development and backup use.  It runs headless, so running X would be a huge waste of server time and resources.

Anyone recommend a distro that's (1) painles to install, (2) has a small footprint in terms of disk and memory space, and (3) doesn't have X , or at least is painless to *not* run and include X?  I'm not looking for Mandrake or similar, where the distro is designed for the desktop and getting X and it's associated huge suite of libraries and apps to not install is impossible, or almost so.  The other requirement is that it be able to be automatically updated over the net without under intervention.  This box is designed to sit in the corner and never be touched - it reboots itself once a week or so, early Sunday morning, just because

I'd go with FreeBSD for the server (it's otherwise perfect for such an application), but since the development server is used for producing Linux apps, it's gotta run Linux.  My other development server happily runs FreeBSD in about 150MB of disk space  (And yes, it's the Escapade development server, thanks for asking!)

Thanks in advance

The reason I recommend slackware is because it does offer excellent stability and Pat V. has an great track record of providing a stable server os for 11years coming this july. Another thing is slackware being the most "Unix-like" linux distro around and being based off the published linux standards....once you know it, you know linux.....admin is extremely easy if you know linux. keeping up to date to the latest stable or the latest development is easy and i've had my slackware machines run much better as servers than redhat 6.x-9/RHEL as far as stability and speed goes.

as far as the packages go, if you notice in a lot of places, most people don't consider things stable and ready for server use until Patrick Volkerding integrates it into the slackware -current packages. take for example, heard a lot of big linux sites  mention 2.6 isn't mature enough for server useage and when you see Pat V. bring it totally into slackware -current(even though slackware has been 2.6 ready for over 9months), then it's matured enough and ready for servers.

The way services and all are handled and things are started on boot is easier to keep up with and allows you keep good control over whats started on boot with a simple ls -atl /etc/rc.d/....chmod 755 rc.x to start it on boot, chmod 644 rc.x to disable it on boot....very easy to understand and easy to keep complete control of......
Also the command line adminitration usually goes much faster on a server anyways since I don't think I'd care to have X running on a server just to use the gui sys admin tools in distros like redhat   or if you need things to look at and click, there's always webmin.

plus it was designed with the twin goals of ease of use and stability as top priorities.....lots of good help for slack and its a distro that i've seen branded as anything from desktop workstation to machine room server, so why wouldn't I recommend it as a server os? also install on even lowend pentium/486 systems is under an hour if u wanna do full install but for a server u want to customize install so may take longer selecting packages but less time installin them(a bit quicker than a stage1 gentoo....hey u might be able to use your server in july if u do gentoo and start right now     )

As much as I hate RPMs, Redhat did provide a better solution than Slackware at that time. I'm scared it (redhat) may be bloated now, but a lot of things in linux are getting that way.
