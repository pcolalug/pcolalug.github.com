Hi FOXX---

This discussion is about your last question --- how to back up your secret key.  After you have installed GPG and generated a key pair they reside in your home/foxx/.gnupg directory, pubring.gpg  and secring.gpg.  You can copy them to a floppy or other media.

This may be too much detail, but here is an example:

Code:
cd .gnupg
cp pubring.gpg /mnt/floppy
cp secring.gpg /mnt/floppy


Let us assume the worst --- everything gets fried and you decide on a new installation.  After the installation install GNU Privacy Guard again and generate a new key pair.  You want to delete the new public and secret keys.

Code:
cd .gnupg
rm pubring.gpg
rm secring.gpg


Now you can copy your old public and secret key rings to the .gnupg directory from the floppy:

Code:
cd /mnt/floppy
cp pubring.gpg /home/foxx/.gnupg
cp secring.gpg /home/foxx/.gnupg


This will save you a lot of grief from exchanging new keys with you encryption buddies.

Hi FOXX---

Been playing with GPG.  No wonder you are having trouble.  Here is how I managed to get my public key in a fille I can send to others.  I had to jump through a few hoops but it works.

I did everything while in my /home/sandspur/.gnupg directory.

I'm going to create an empty file we will use later.  

Code:
touch chuckkey.asc


Eventually we will have an ASCII version of my public --- you can view it later with vi.

In a terminal and in my .gnupg directory I typed:

Code:
gpg -armor --export chucksteele@bellsouth.net chuckkey.asc


An ASCII GPG key appeared in the terminal, to my surprise.

How to get it in a form you can send to others?  I highlighted the GPG key and then clicked Edit > Copy.

I opened another terminal and typed
Code:
vi chuckkey.asc


I entered the insert mode in that file and clicked Edit > Paste.

The key was then visible.  Exit and save vi.  chuckkey.asc is the file I will send to other people.  They will import the file to their public key rings.

Hope this helps you.

Chuck

Ok, I was able to simply encrypt a file with the following:
Code:
gpg -e calendar.html


Of course, you'd want to pick a file on your own machine. When it asked me for a user ID (the key to use) I gave it the username part of the email address of my key (rburt3). This resulted in the file calendar.html.gpg. To decrypt, I used:
Code:
gpg --decrypt calendar.html.gpg


It prompted me for my passphrase and then spit out the plaintext. If you want to save the plaintext, you would have to do something like
Code:
gpg --decrypt myfile.gpg > myunencryptedfile.txt


Of course, the decrypt worked since I encrypted with my own key. If you are encrypting with someone else's public key, you won't be able to decrypt unless they have foolishly given you their private key as well.

So, if I had your public key (you can find mine at http://home.mchsi.com/~rburt3/rburt3.asc) and wanted to send you supersecretstuff.txt, I would do something like
Code:
gpg -r SiliconFOXX -e supersecretstuff.txt


I got all this from reading the man pages (shame on me, but I usually use kgpg and the Konqueror file manager for all of this stuff... ) .

Nope. This would be a HUGE amount of data and quite impossible to distribute in a practical manner. There are public keystores, though. Give http://wwwkeys.us.pgp.net a try. For others looking for your key, you can always export your public key to the free keyserv with the command
Code:
gpg --send-keys rburt3 --keyserver hkp://wwwkeys.us.php.net


Of course, you'd replace rburt3 with your own id and hkp://wwwkeys.us.php.net with the keystore url of your choice.
