26  Software / Networking / Linux to linux desktop control on: September 07, 2009, 07:32:07 pm
I have been looking for a good way to access and control a linux box from a linux box. My mother got her first computer at 79. It is a linux box I built running PCLinuxOS. She had never used a computer before but at 81 she is surfing the net on linux and really enjoying it. Her hands don't always work as she would like them to so I occasionally have to help her. The ability to remotely control her desktop saves me time an travel to and from her house and she gets instant help.

I have tried several things. TightVNC, VNC and Krdc. We don't have static ip addresses and I know I can register with one of those dynamic dns web sites but I chose hamachi to get a static ip address. Hamachi is totally free on linux. She has a dsl modem and I have a cable modem. Her connection is the dsl lite so it is somewhat slow. I found that Krdc is really slow updating her desktop on mine. TightVNC and VNC both are faster and work well except I could not find a way to set the screen resolution on my desktop and I had to scroll around to see all of her desktop.

I searched the net and found a much better solution. It is the free edition of NX by Nomachine. The core libraries are GPL'ed as FreeNX. The NX Free Edition comes with deb, rpm and source. I downloaded the rpm's. There are three components. The client, node, and server. To share your desktop you need all three installed. To log on and control a desktop you only need the client. You need to read the documation to get it configured for controlling a desktop other wise it just logs you into you own session on the computer. The desktop control terminology they use is called shadowing. You need to configure the /usr/NX/etc/server.cfg and the /usr/NX/etc/node.cfg files to get it setup. The reason I like this better than TightVNC and VNC is I can set a custom screen resolution on my desktop so I can view the other desktop in full without scrolling around. NX uses SSH to tunnel the connection between client and server also.

If I read it right the server is only supported on unix but you can control a linux box from a windows client as well but why? Anyway check out their site for the documentation and see the other stuff they offer.

http://www.nomachine.com/download-package.php?Prod_Id=1083
