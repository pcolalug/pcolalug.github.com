Title: Book Review: The Official Gnome 2 Developer's Guide
Post by: imnes on June 16, 2005, 09:11:42 pm
The Official Gnome 2 Developer's Guide

(http://www.nostarch.com/images/gnome_cov.gif) (http://www.nostarch.com/frameset.php?startat=gnome_foundation)

I got this book to use as sort of a reference / guide to the Gnome libraries while learning the Java-Gnome bindings.  This book was written by C programmers for C programmers.  It's usefulness as a guide / reference for other languages is kind of limited (unless you happen to know C).  This would probably be better if I knew C or the Gnome API coming into this, but knowing neither it can get tough to follow some parts.

I started off trying to learn the Java-Gnome bindings, but started toying with the Mono project early on - and found it to be a much easier platform to get started with.  (But I still think GNU's Java Compiler is the right way to go, I plan to pursue this some more).  The mono-GTK# platform will run on older software (Gnome 2.6) than Java-Gnome (Maybe Gnome 2.8, or 2.10).

As an overview of the Gnome / GTK platform, this book is great.  You can get a good overview of how the various libraries work together.  The book covers the basic GTK and GDK libraries (sort of the base libraries that Gnome is built on).  It also covers the Gnome libraries which cover services like Gconf (a registry-like service for storing program configurations).  

Gnome-VFS is covered briefly with some examples but I didn't have a chance to try any of it yet.  

The book also covers the Glade toolkit.  Glade is a program that allows the programmer to easily build a GUI by visually laying out components in the screen.  This is much easier than trying to hand-code windows in your programs source.  Glade stores your windows in standard XML files so you can easily open them and make changes at any time without touching your source code.  Once a window is built, the programmer can use the Glade Library (libglade) to parse the XML file, and render the window on-screen.  (It will also automatically connect your code to the window components, for example attaching code to a Quit button to shutdown your program).  

This book is most helpful if you use it to learn about the Gnome / GTK component you want to use, then supplimenting the material by finding examples (maybe on the web) showing details about how to actually code the component in the language you're using (Python / Java / Mono C# or VB.Net).
