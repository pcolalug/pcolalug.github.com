Title: Tips and Tricks for ROOT!
Post by: rugmonster on April 05, 2003, 11:21:53 pm
For the opposite situation, where you need to execute one or two commands as root, rather than using su, you can use sudo. This does the same thing as suing to root, but without dropping into a complete shell as root. Very handy, especially when you set it so your uid doesn't require a password to sudo.

Example:
Code:
$ sudo mount /dev/cdrom


You have to have sudo installed on your system and you have to add your user or group to /etc/sudoers with visudo, but you can either have the option to ask for a password or not. Check out the manpage for visudo and sudoers on how to set these up or ask me and I can help.
