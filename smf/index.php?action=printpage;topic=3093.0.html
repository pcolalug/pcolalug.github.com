Title: Re: No Sound in OpenSUSE 10.2...
Post by: OmnipotentEntity on December 25, 2006, 01:13:45 am
Suse runs on KDE which means, because you have a sound card and don't need software mixing, that arts shouldn't be running.  If you're getting a /dev/dsp error that usually means that arts is owning the handle to /dev/dsp and it's not allowing other programs to access it...  All you have to do is open the KDE Control Panel and turn off Arts. (kcontrol -> Sound & Multimedia -> Sound System -> Uncheck "Enable Sound System.")

(If you're not getting a /dev/dsp error that means everything is kosher with that usually.)

Then check your alsa mixer to make sure you've got the sound turned up.  Remember, after you disable the Sound System you have to restart all open apps that use sound in order to get sound.

You might also want to consider using lspci and lsmod to make sure you have all of your drivers installed too.  Sample good output is:

Title: Re: No Sound in OpenSUSE 10.2...

OK, for anyone still paying attention to this one... here's the final word. By default SUSE 10.2 doesn't add it's user to the audio group, this has to be done after the install through YaST. After this the sound works... MOST of the time. When I get a boot with no sound I restart the mixer, then the computer and everything is gold again. Title: Re: No Sound in OpenSUSE 10.2...
Post by: rugmonster on January 04, 2007, 11:23:54 pm
Group permissions are as simple as adding the user to the group.

Example:

/etc/group
daemon:x:1:
audio:x:29:user1,user2,user3
admin:x:108:user1,user4

You have <groupname>:<comment>:<gid>:<users>,<belonging>,<in>,<the>,<group

In Ubuntu, when users are added to the system they are automatically added to the audio group. I suppose it may not be something you want, but for a system geared to desktop use, I would expect it.

Here's a pretty good overview of groups in the linux world.

http://www.yolinux.com/TUTORIALS/LinuxTutorialManagingGroups.html

File permissions get really cool (and flexible) when you start using ACLs.
