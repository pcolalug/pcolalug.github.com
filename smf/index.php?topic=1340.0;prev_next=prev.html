If this is a X-based tool you want to start up each time you log in to X, you could use the .Xsession (.xsession, .xinitrc) approach.  This is very X build specific so it depends on the distro.  Basically, I start my windowmanager from my '~/.xsession' file:

Code:

#!/bin/bash

WM=/usr/bin/ctwm

# Basic startup cmds

## ssh agent
eval `ssh-agent` &> /dev/null

## turn off beeping
xset b 0

## turn on dpms
xset +dpms

# Call startup script
~/bin/x11_init.sh

# Load WM if possible else load an xterm
 if [[ ! -x $WM ]]; then
   xterm &
   waitpid=$!
 fi
 # Load wm
 $WM &  
 waitpid=$!
#fi

# Wait for leading process
wait $waitpid




Then I symlink as follows:
Code:

cd ~
ln -s .Xsession .xsession
ln -s .Xsession .xinitrc


Dunno if this is the sorta thing you needed.  This WILL restart the process every time you log in to X.

If you've got the program running from rc.local or where ever, you will need to have it executed as the user.

 e.g. su - myuser -c 'myxfrontend'

This will make sure that the logged in user's ~/.Xauthority file allows the program to have access to the display.  You may also need to add DISPLAY=:0.0 to your environment as well to make sure it tries the right place.

So putting it all together --

#!/bin/bash
USER=frontend-user
# this assumes bash is the defl shell for $USER
CMD='DISPLAY=:0.0 /usr/bin/myxfrontend <myargs>'
su - $USER -c "$CMD"
#EOF

As for my other solution, it just has to do with how X starts up.  By default it will look for an Xsession file in /etc/X11.  That file usually refers to the calling user's ~/.xsession file.  It then runs that and when it returns, exits X.  If it turns out you're interested, check out the man pages and online docs about xsession and xinitrc (as well as xinit).

I'm not sure if the above suggestion will work for you.  If it doesn't, you might have to resort to the xsession approach.

Good luck!
