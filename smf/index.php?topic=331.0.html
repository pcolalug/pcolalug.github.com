Quote from: phed
I understand Linux and Windows XP do not play well together on the same HD so each will be on its own HD.

I have run Linux and XP on the same drive without problems. I have had best results when I installed the Windows first and then Linux. You have to realize that Linux will not be able to write to your NTFS (the file system that XP runs on).

If you want to trade files between the two OSes, then you will need to set up another partition that is just a FAT or FAT32 partition. This will be viewable and writeable from both OSes. The other option is to install XP on a FAT32 partition. I will assume that it is already installed and working, so there is no sense in monkeying with it. Just set up another partition (FAT32) that will be viewable and writeable from both. Either way, you will be able to see your XP partition from Linux. It is completely readable, you just cannot write to it.

Quote from: phed
I had ordered the Debian GNU/Linux distro CD's which I guess was clueless newbie mistake #1.

Not a mistake. If you are anything like the rest of us, you will want to try them all (or at least a good handful) before you decide on the one that you will stick with the rest of your life, or the next 3 weeks when a newer one hits the servers.

Quote from: phed
Being a first timer, is it better to order their CD's or download the ISO's from their website?

Apparently, Mandrake is hard up for cash. So go ahead and send them the 12 and 1/2 bucks. They need all the help they can get.

Quote from: phed
As I said before, I am looking to eventually move away from M$.

Don't get your hopes up too high. I have been trying to do the same thing for 4 or so years. The problem is games. You need a small Windows partition just so you can have the game fix.

One thing that has helped me with a transition is to use programs in Windows that you will use in Linux. For example, I run OpenOffice.org for both Linux and Windows. This lets me keep the same software and file types in both OSes. I use Mozilla/Phoenix as my browser in both places as well.

As you get into this let us know some specific questions in the appropriate spots. Some guys live to be able to post on this board.

Welcome to the team. Look forward to meeting you.

Quote from: phed
From what I've read on these discussion boards, the favorite distro seems to be Mandrake 9.0. I had ordered the Debian GNU/Linux distro CD's which I guess was clueless newbie mistake #1.


Hey hey! Don't be thinking that Debian was a mistake. Debian is very popular and if you need any help at all with it, I'll be more than happy to help you. I am a huge Debian fan, but I will say it isn't the best desktop Linux solution. Also, you can get on IRC and any EFNET server and go into #Linux. There are quite a few Deb fans in there. That was my old hang out and there are some great folks in there who will be very willing to offer up some help. Just remember that you have several sources of great info in google (including groups.google.com), man pages, www.tldp.org which has a whole library of HOWTOs and such as well as the documentation that comes with your packages. If you can't figure something out after that, no one will ever fault you for not understanding something. It is an aquired art to read manpages and HOWTOs.

Quote from: phed
I see where Mandrake has an "Operating system refugee" program where you can order 9.0 for $12.50. Being a first timer, is it better to order their CD's or download the ISO's from their website? I do have broadband net access (cox.net) so downloading large files will not be a major problem. I am looking for any way to minimize problems on install.


Two things on this. The $12.50 offer is for a full set of Mandrake 8.1 or 8.2 or a /single/ cd edition of Mandrake 9.0. This won't give you a whole lot of software, but nothing is stopping you from downloading the packages or the other two cds from a mirror. You can also looking into Redhat. It is supposed to be pretty darn nice as well. At the present moment, I am using Mandrake 9.0 and am pretty happy, but I do miss apt (part of debian).

Quote from: phed
KDE looks pretty nice and hopefully will not be to difficult to get used to.


It is geared for easy migration from a Windows environment. The general use is very similar to the windows start menu idea. You won't have issues there. When you start digging deeper, which you will, you will find the differences and you can always ask any questions you have here.

Quote from: phed
Any suggestions from all of you would be most welcome. I'd like to hear from folks who have, "been there and done that" and have the benefit of your experience.


My biggest suggestion for anyone is once you get used to the general GUI desktop, dive into the command line. The true power of Linux (and any other Un*x based OS) is the command line. Learn how to read manpages and get comfortable with config files. You will be a better person for it...or something like that. I've got a friend who is learning command line right now. It goes quickly and is very rewarding, especially when you write your first shell script.

Quote from: phed
I like the open source philosophy and especially the price!


For the most part, most of the leaders of the OSS movement (besides the highly religious) want OSS to be used because it is the best tool for the job. Use what makes your life the easiest and makes you happy. Computing should be fun and Linux and the rest of the OSS movement has put that fun back into it.

I look forward to hearing more from you. Welcome and thanks for getting me one more step away from being FNG on the forum
