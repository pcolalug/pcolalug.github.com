From someone who used to run a linux firewall and has since moved to using a Linksys WRT54G, I think it would be best to use the router. You can configure your port forwarding through the router's web interface, usually found by going to http://192.168.1.1 and using the default password of "admin" and no username if you've never changed it.

Here are the instructions from Linksys on setting up port forwarding.

The reason I don't use a linux box as a firewall anymore is for the fact that you have to have the router box always on. A linksys router will use less electricity and you don't have to worry about it crashing unless you have a bad firmware flash. The positive side of using a linux box as a firewall is you get experience with setting up iptables, but you can do this anyway (and recommend it).

actually i've seen many linksys, belkin, d-link, netgear, netopia/cayman, etc consumer routers crash when under a load(seen a few actually crash very frequently, like every few days or weeks due to heavy transfers) or when you forward too many ports and it runs out of memory....on the flip side i've never had my linux router crash in the well over a year and some odd months that it's been running and considering we already have a good bit of systems running, 1 lowend stripped down pentium class machine that is even way overkill for a router really doesnt drag out much juice

but if you do have a old computer laying around with 2 network cards, i'd recommend actually making a dedicated linux/bsd router. Probaly one of the easiest to get up and running would be smoothwall (http://smoothwall.org), but you can do it with any distro you choose but smoothwall is nice for making a quick linux router, has a nice web interface for checking iptables firewall logs and snort IDS logs, changing settings, setting port forwardings, checking transfer graphs, checking for updates, etc.... ipcop is another that was originally a fork of smoothwall

best bet though is just use a router(whether it be a store bought router or a *nix based router) and make it simpler on yourself and use port forwarding...and dump the extra ip
I'm very happy with my linux router though, but might try a openbsd based pf router once I get more familiar with the inside and out of openbsd and possibly carp

p.s. we do have a linksys WRT54G wireless router, works quite nicely as a 802.11b/g access point once u disable dhcp server and bridge the lan ports on the unit into your lan    but i've heard that the linux distros for that particular model run well, u ever tried linux on your linksys 802.11g router? been wanting to try it for a while now actually, mainly just for toying with

Yeah, I put the really popular third party firmware on it to try out PPTP VPN functionality, but found it was relatively slow and somewhat less stable than the Linksys firmware. The one thing I do like is I can boost the crap out of the signal and get 54mb all over my apartment and on the balcony, but feel bad for anyone else who wants to use channel 11 around me.

Like I said before, I felt the linux router was a bit of an overkill since I was only using that box for routing. I didn't want to use it for anything else because I didn't want to bring it down. If you don't mind dedicating a box to it, go for it. I would put something like a folding client or SETI@Home on it because iptables doesn't take much to run.

I do like pf on OpenBSD, except the way they handle ftp...I couldn't ever figure out how to do the stupid ftp proxy for incoming connections. If you don't care about hosting an ftp server from the internal side of the router, then no worries there. I also like the syntax for pf so much more than iptables.
