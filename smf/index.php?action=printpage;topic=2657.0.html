So far I have built several bash and perl scripts to handle most of the management issues. So far processes migrate fine.
Single processes do not migrate but any and all multi process processes do migrate to avaliable nodes. I was using a combined 10mb and
100mb but was noticing really slow hand-off times. Now with a full 100mb network and proper switches I am getting full bandwidth.
I have built several test scripts that run large numbers forking the processes to fork as many times as there is nodes.
I am presently working on scripts to migrate single thread proccesses to available nodes.
I am presently running the stable openmosix kernel and openmosix-tools with the ram patch that gives me a total avaiable ram
that I have on all up nodes. Very nice.

Well this is where I am at right now putting in new shelf in rack next weekend for last two nodes and adding cisco switch.
That will be it for my cluster then I will hit hard and heavy on scripting and programming at kernel level. should be educational.
