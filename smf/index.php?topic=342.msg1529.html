Quote from: danc
1. Would like to automate some repetitive tasks, like changing directories. I think the way to do that is with the "alias" function, and I can get that to work. But how do I save the alias to always be there when I log in? From what I've been able to read, I should save it to the ".bash_profile" file, but can't seem to make that work. Do I just edit that file and put the alias definition in there? So far the alias I define only seems to "stick" for the current terminal session, and if I exit and re-open the terminal window, it doesn't work any more.


It all depends which shell you are using. If you are using bash, then .bash_profile will work, but if you are using zsh or ksh, then you will have another .profile to edit. If you want it to be system wide for all users, then take a look in your /etc directory for the appropriate profile file. If you do a ps ax, you can find what shell you are using. Then if you man <shell (sh,bash,zsh,ksh,csh,etc.)> look for the information about profile.

Quote from: danc
2. A similar question is about shell scripts. I want to have a script to start up my RTCW server, without having to type the long command line each time. The books I have say that I can do this either by:
A. Create a text file with desired commands, save, chmod to executable. This seems to me like the old-fashioned DOS batch file method.
B. Create a shell script file, where the first line is: #!/bin/sh (and other lines below), save, chmod to executable. What is the difference between these two methods???


They are the same thing. By putting #!/bin/sh as the first line, you are telling the current shell you are using what to use to interpret the script. If you don't specify this, the shell will go down the file line by line and try to interpret each line itself. If you were using csh as your shell and tried to run a script written for bash, parts of your script won't work. Or say you have a perl or python script. Those you just put in a plain text file. Instead of having to type our perl [appropriate switches] filename, you just put #!/bin/perl (or appropriate path to the perl binary) as the first line of your script and then you can type ./filename. Easy enough. I would suggest, just as good practice to always put the appropriate interpreter in all of your scripts. You never know when you might want to give your script to someone else who might not use your same shell.
