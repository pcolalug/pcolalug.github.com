Title: Print Server
Post by: rugmonster on January 03, 2005, 09:30:09 pm
Quote from: rburt3
Can you config this little gem from a Linux machine?


Of course you can! To initially configure it, you just need to plug it all up to your switch and then assign an IP such as 10.0.0.2 to your nic and then point your browser to http://10.0.0.1/. It's very simple from there. The web interface is very basic. You have to turn off the DHCP server that is setup to run by factory settings if you are using another.

Best bet is to assign the puppy a static IP outside of your DHCP scope for your internal network since there is no way to make it automatigically update DNS due to it's somewhat feature-lacking DHCP client. Ho-hum, but it works. What I would recommend is to give the static IP a DNS entry (if you have an internal DNS server...and what self respecting Linux user doesn't) and use that DNS name for all of your configs. That way, if you ever change the IP of the print server, you simply update the A record in your DNS server and all of your clients are back to using the right IP for the printer. Imagine having to change 5 or more IPs for TCP/IP printer sockets in CUPS and any other clients you may have.
