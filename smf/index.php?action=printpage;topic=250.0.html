Has anyone here had experience coding with the various toolkits for X?

Any opinions / suggestions?  Or possible another toolkit not mentioned here?

Post by: rburt3 on November 20, 2002, 05:50:37 am
Well, I haven't done anything with GTK(+/-), but you confirm everything I've heard about it being a bit cumbersome. Qt is really clean and pretty easy to work with. Signals and slots are a joy after the nightmare of callbacks. There should be application scripting with this latest release as well. (How long before KDE gets email viruses?)

As for java(TM)*, I hate awt/swing. It feels cludgey. I've been looking into XWT (http://www.xwt.org) a bit for a project at work. Seems pretty cool, if you're into the XML-as-gui-markup type of thing.

The sad truth of it all is that if you want to stay in the area and code for a living, you probably want to be proficient with Visual Basic and/or Visual C++. Java never hurts, and there is a lot of database (SQL) work to be had if the timing is right.  You should also have a sprinkling of web programming in your toolkit as well, for when times get lean. ;)

Don't get me wrong, though, these should be condidered "fallback" skills to pay the bills if need be. In addition, *nothing* can substitute for good general programming skills. Just my .0012 escudos.
