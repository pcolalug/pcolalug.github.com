Quote from: Brent
I'm not interested in learning to hand-code html when there are programs out there that could do better.

There is a reason people learn to hand code. It is like doing work on your own car. Sure it is easier to let the mechanic do the work, but you will pay a price for it. If I do it myself, I choose which parts go in there and how much I want to spend for each one. If the mechanic does it, you are almost guaranteed to get the lowest quality part for the highest price.

With FrontPage, you get the same end result, but you will pay a price for it. Not just the dollars that it cost, but the quality of the underlying code. Which results in larger file sizes and code that may not conform to the standards, therefore not all browsers will be able to read the pages as you intended.

Quote from: Brent
I ordered frontpage because it is well known, and cheaper than dreamweaver.  

And so is the quality of code it produces. I watched the Indianapolis 500 a couple of weeks ago. I was amazed at the cars themselves, but also at the size of the teams. When Tiger Woods wins a tournament he pays himself and Steve (his caddie). When David Letterman won the Indy 500, he had to pay a whole lot of people. But, it is worth it to him to pay those mechanics to keep his car running as opposed to picking a group of guys out of the grandstand that would love to work on one of those cars for cheap.

I have personally never used Dreamweaver, but if I was building pages all the time, or doing it professionally, I would consider it.

Quote from: Brent further
Now i would like to know how well a frontpage generated website can coexsist with apache.  

Coexist perfectly. Which is the heart of your question and the answer for which you seek. The quality of the final code will not effect Apache's ability to serve up the pages.

Quote from: Brent
I want to make sure my pages can be hosted on apache, and can be viewed by any modern web browser.

Now that comes back to the code. There is a standard for how code should be written. If it conforms to the standard, then modern browsers that conform to the standards will be able to read it. Unfortunately, Microsoft Internet Explorer does not conform to the rules.

So if you are wanting to write cutting edge code, you have to either write it for IE, the most used browser (I am assuming), or for browsers that follow the rules. Most day to day basic html will work on all of them (maybe), it is mostly the "cool" stuff where you will find your problems.

Now, with all of that in mind. Is it bad to go to a mechanic to have work done? No. Your car gets you down the road. You may have to pay more to keep your 4 year old car on the road than I pay for mine. But if that is your choice, then you make that decision. Personally, I will continue to drive my 14 year old car with 238,000 miles with confidence that if anything goes wrong, I am going to do all I can to learn how to fix it myself.

To use FrontPage is not a wicked thing. If that is your choice, then no one will fault you for that. Just realize that it deos not (and never will) do a better job than coding it yourself. Dreamweaver is worth much more in cost than FrontPage because of the better quality code that it produces and the time it saves. But still, it cannot compete with a person who knows the code well. Both products may produce a quicker result, but not a better code.

If you code poorly, then your results will be poor. If you mechanic poorly then your car may not run. So using a professional until you learn to do it yourself is a good thing.

As a side note, my cousin told me this last week that he is a certified web designer, but he had never heard of Linux.

I believe that if a web programmer writes code to conform to W3C standards, it will be displayd correctly in most current browsers 99% of the time, including IE. If that same programmer writes code in FrontPage, it will display correctly in IE 6+ 99% of the time, but will only be correct in other browsers half the time. In order to accomodate the other 25% of us (I don't really know the number), I would recommend programming with anything but FrontPage.  Those pesky 'msoe' and 'mso-ignore' tags and such placed by FrontPage really confuse other browsers and seem to have been put there only to discourage use of any browser but IE. That is one way that certain companies ensure that IE remains the dominant browser.
