I've settled back on slackware (again) for all my systems.  (Why does slackware have to be bunched into 'other', should be 'RedHat Based Distros', 'Slackware', 'Gentoo', 'Other')    

I was trying different distros to ease some of my hardware problems, but then I  got over it and just ran slackware with a newer kernel.  2.5.74.  The compile process is interested on the new kernels.  Runs great though, and at least as stable as 2.4.20 for me (with much improved sound).  They dropped OSS sound drivers (they may still be an option but they're moving away) and incorporated ALSA as the main sound option.

Also compiling this new kernel seems to be quicker on my system.  If anyone is planning to try it, make sure to get the new modutils too as you'll need them.  It tells you all about it in the readme (I only RTFM as a last resort so I didn't know).

If you got a 2 cd set you have the sources, otherwise visit ftp.slackware.com and get them..

Basically, download this file to your computer
http://ftp://ftp.slackware.com/pub/slackware/slackware-9.0/slackware/k/kernel-source-2.4.20-noarch-5.tgz

Login as root, and type
Code:

installpkg kernel-source-2.4.20-noarch-5.tgz

Incase you missed it, you should edit your /etc/X11/XF86Config file and make sure Driver is set to "nvidia" not "nv".  You'll know you have it setup right if you get that nvidia logo when your XWindows starts.

Also you want to comment out the line for module DRI and one other one, I can't remember off the top of my head for sure, it's in the readme though.
