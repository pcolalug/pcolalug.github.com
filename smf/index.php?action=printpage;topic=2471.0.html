Title: OpenBSD binpatch
Post by: lloydr on June 09, 2005, 08:45:30 am
I'm beginning to work on the Binpatch system for OpenBSD where Santana over at openbsd.org.mx left off. He left off a bit into 3.6, 5-7 patches ago due to lack of time. I've began building binpatches for 3.7 mainly as that's the latest release.

http://www.openbsd.org.mx/en/projects/binpatch.html

Basically they allow you to upgrade your openbsd system to any of the patches posted in the errata (http://openbsd.org/errata.html) without having to recompile the source of the system.

I'm doing this to keep from needing a compiler and sources on my router/firewall box, along with now any other machine I have running openbsd like my primary dns server. my binpatch build environment is a base openbsd 3.7 install running inside vmware on my slackware box.

I'm basically wondering if anyone is interested in testing this binpatches out or if you find a use for these, I'd be happy to upload them somewhere. I sent a copy of the 001_cvs errata binpatch to Santana to look at. So far my 3.7 binpatches are limited to only the 001 patch so it's not like much has been happening in the obsd errata area, but that's a good thing.

So far the patch applied just fine on my future router/firewall box, but it's not like the patch applied to anything in use like kernel updates or such, but I'm sure patches will come soon that do apply.

To install a binpatch just unpack it under /, e.g.:
cd ${PACKAGEDIR}"
tar xzpf binpatch-${REVISION}-${ARCH}-001.tgz  -C /"[/url]

I'm building them for i386, planning to use the GENERIC kernel config, so the install process would just be:
tar xzpf binpatch-3.7-i386-001.tgz -C /
tar xzpf binpatch-3.7-i386-002.tgz -C /
and so on down the line of patches.
