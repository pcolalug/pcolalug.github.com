Title: Setting up a proxy
Post by: phed on July 15, 2003, 11:37:24 pm
Hi Guys,

Here's my problem that I would like your help with.

I use a proxy to filter my Internet account.  It works under M$ Windows in such a way as to set itself up when the computer boots up.  All http and ftp requests are automatically routed to the proxy.  This cannot be disabled by fiddling with the proxy settings in any of the browsers.

Linux however is another story.  So far I have only been able to set it up by using the proxy settings within the individual browsers.  What I would like to be able to do is to have some kind of script run automatically each time a user logs in to their account.  The script would need to send the username and password to the proxy server and subsequently redirect all furthur http and ftp requests from that user to the proxy server.  I am currently using Mandrake 9.1 and Gnome 2.2 for my desktop.

The object is to make it less easy for the casual user to disable the proxy simply by changing the proxy settings within the browser.

The high speed proxy that I use under Linux is proxy.hedge.org on port 8080.

Any help would be appreciated

Title: Setting up a proxy
Post by: crowe on July 16, 2003, 12:02:23 am
Try adding this to your ~/.bash_profile (or such):

Code:
export http_proxy=http://proxy.hedge.org:8080
export ftp_proxy=http://proxy.hedge.org:8080


Log out, and log back in.  Then see if it works afterwards.

Of course, this would still be easy for the user to modify, if the user knew what they were doing..
