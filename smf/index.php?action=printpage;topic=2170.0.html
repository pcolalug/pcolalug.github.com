There is very little significant difference between FreeBSD, OpenBSD and NetBSD. OpenBSD has been ported to more architectures somehow, but the differences between OpenBSD,NetBSD and FreeBSD is not like the difference between Solaris and *BSD. Since I have used FreeBSD more than OpenBSD or NetBSD, I would have to recommend FreeBSD.

Actually, from what the user sees, the differences are subtle. It definitely feels like a professional Unix in that things don't always make sense where they are located. To me, Linux's filesystem layout makes the most sense of all the flavors of Unix-like operating systems.

I have been debating these differences. While there are some differences, I don't think they are major ones. In all truth, there a just a couple of directories that are used by the Sidewinder that you won't see on Linux, but it isn't that extreme. I think the most noticable difference is the fact that your default shell is tcsh. That's okay, I can put tcsh on a Linux or BSD box, no problem. Vi, tcpdump, etc are going to be very similar if not the same in their behavior.

What I'm getting to is that I'm beginning to think that I'm going to have to figure out the best platform for supporting multiple users running virtual servers. Linux has this capability with the very robust user-mode linux. FreeBSD gives us jail, which is apparently different from chroot. So at this point, I'm somewhat torn. From what I understand, UML requires a kernel of it's own, where jail does not, but I don't think jail will necessarily give me a full /dev filesystem. With UML, I can simulate hardware that I might not otherwise have and it all be residing inside a single file. So at this point, I'm leaning more toward User Mode Linux or FreeBSD. I'll have to play with the two and see what works the best.
