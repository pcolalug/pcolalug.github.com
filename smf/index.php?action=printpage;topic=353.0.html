Post by: steve on January 06, 2003, 06:08:26 pm
I had looked into this a while back. The way I understand Linux is that will bite off a chunck of memory when it first starts up to use as it sees fit. The more memory you have, the more memory Linux bites off. It willl free it if other programs need it.

However, I also found a couple of programs that I used to run that had memory leaks. One example was KDE with the Nvidia card. Everytime I switched from one desktop to another and back again, KDE would eat up more memory and wouldn't release it. Eventually it would freeze the system while it did a constant swap to the hard drive. This was about 8 months ago, though. Gnome did not exhibit this problem and that's why I switched to it and haven't looked back.

I do use the Gnome and sometimes KDE system monitors to track which programs are using the most memory. Remember to check program usage as root or you'll only see your processes.

Steve
