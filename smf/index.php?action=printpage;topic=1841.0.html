Title: Linux Doesn't Exist
Post by: SiliconFOXX on September 12, 2004, 09:12:03 pm
To my knowledge, Linux is a Unix clone written from scratch by Linus Torvalds.  I mean, written from scratch means no copied code.  Even if there's similarities in implementation it's a totally different beast that just happens to look similar.

If nothing else, the freakin' clone() function separates Linux from nearly everything else on the planet.  All calls to fork(), pthread_create() and like calls are all made into clone() whereas in everything else those are all handled separately.  The power and massive usage of clone() clearly defines a major area of usability and stability (for good or ill) from any other OS.



This is like the people who tell me Linux isn't an OS.  They tell me it's a kernel wrapped up with a lot of other applications.  Pardon me, but where did their brains go?  The definition of an OS as can best be described is "a piece of software that interfaces with the hardware of a system and provides an interface such that the user may interact with the aforementioned hardware." Well, that's what the kernel DOES, which by definition makes it an OS.  There's nothing about user-friendliness in there (man, if that was the issue I don't think Unix would've been around very long at all) or external apps, just that interface.



This is as stupid as those people trying to patent software concepts over in Europe.  You know, trying to say, "Yeah, I came up with a 'scroll bar' so you can't do anything like that." Total BS.
