Post by: dpeach on October 16, 2003, 04:13:46 pm
You should be able to plug your modem to the switch and the switch to the different computers. Without a need for two NICs in the main computer.

But...there is always a but, depending on your switch, this probably will leave you without a firewall.

I don't know how you want to handle giving out IP addresses to all the computers on the network and handle firewalling. I personally (did this once and it worked, there is probably a better way), would run your modem to ethX and set up firewall and dhcp server (and any other server that you want), then go out to the switch with ethY (X being one and Y the other, your choice, but I think standard is 0 going to the WAN and 1 going to the LAN).

There are security freaks that will say that this is potentially dangerous, having your Firewall being on the same box as any other servers. They are probably right, but I never had a problem with it. Ran like this for two years and did all the security checks that I could to find any problems. Never found any.

That is my recommendation.

I like what Bellsouth is doing to the DSL. Giving more options. I have read that several others are getting ready to do the same. The DSL lite ($29.95) that they are selling is as fast as the DSL that I used to have and I was paying $50 a month. Now for $39.95 ($10 more) you can get 6 times faster down and twice as fast up. That is great. I don't know if there service stinks (if it is as bad as their Cell service, it does), but it is a great price and I am glad to see it coming.

Title: DSL coming
Post by: imnes on October 16, 2003, 04:30:33 pm
When I signed up you could only get USB dsl modem for home users and ethernet modem for soho users.  FreeBSD drivers for the USB one sucked so I got a cayman DSL modem+router off ebay pretty cheap it works great.

It was nice, with the router and my laptop on UPS I was able to keep internet access and my computer going for over 4 hours yesterday durring a lenghtly power outage, I could have probably gone 5 more hours but luckily power came back.  Having a tiny hardware router was nice because it pulled almost no power off the UPS whereas a full-blown PC based router would have eaten it up quickly.

I don't have any problems running servers off my account (it is a dynamic IP though).  I have it tied to a domain name (homeweb.org) and have a script that keeps the DNS records updated when my IP changes.  I haven't really had any downtime yet.  I have mail server, web, telnet, ssh, ftp, and postgres database running on it, remotely accessible.  I haven't experienced any bandwidth problems / limits.  I usually get around 350k/s downloads and I move a lot of bandwidth for web-based videos I use for school.  I've pulled down close to 16gb worth within the last month.

Oh well YMMV.  Let us know...
