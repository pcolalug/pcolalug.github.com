Title: SAMBA set-up quick and easy.
Post by: bhoff on April 07, 2003, 05:59:20 pm
SAMBA

   Ok I have been putting it off for a while, But I had to much time on my hands this weekend and got tired of flipping back and fourth between my Linux Mandrake 8.2 and my Windows 98 boxes...oops did I say win98...(sorry about that but yea I still have a couple running for a project I am on.) Anyhow I decided to see how hard it was to install and setup samba so they could communicate and save me a lot of trouble.
   I have been putting it off because I heard it was very difficult to get up and running. Could have not been further from the truth.
I installed samba using the mandrake software manager and then did some research found out that there is this great tool called SWAT and let me tell you it took me 1 page of reading and about twenty minutes to get both of my win98 boxes setup and talking to the Linux box I can log in and browse my shares and read/write to them from my Win98 boxes. Another 8 minutes setting up my printer which is on my Linux box and I was good to go, And this tool SWAT comes on the mandrake disk. At least ver. 8.0 & 8.2.
It runs in your browser. It lets you setup everything from all your Global's to shares and printers with constant monitoring of who is connecting and where.You can start and stop the daemons right there on the screen!
   There are two things you will have to do in order for this to work thou.
1.) First edit the /etc/services files.
Add the following line to your /etc/services file.
swat 901/tcp
2.) Secondly add the following line to your /etc/inetd.conf file
swat stream tcp nowait,400 root /usr/sbin/swat swat

   These are needed in order for SWAT to work. After this just open your browser and type http://localhost:901/ or search your menu for Configuration/networking/samba configuration. That should do it a little digging in the included help and you should be on your way.

Here is a little help with those of you still using Win95 & Win98 Due to the way windows deals with passwords across networks it is best if you have the same accounts for each of your windows boxes and on the Linux box.
After setting this up go to your command console on the Linux box and type smbpasswd -a <username> this adds the user to the samba account (note: must have the account on the Linux box also!) it will ask you for the password.
You should be able to login to you shares on the Linux box no problem.

Any questions can be posted here for everyone's benefit.

Title: SAMBA set-up quick and easy.
Post by: danc on June 11, 2003, 10:52:50 am
Do you have anything more specific that we could answer...?  ipchains has nothing to do with Samba, although it could prevent it from working.  ipchains produces a set of firewall rules which might prevent other machines from "seeing" the Samba server.  (also, ipchains is sort of outdated, and has been replaced by iptables).

Try going to http://www.samba.org and see if there's anything there to help.  I have some other good links to Samba info at home which I can give you later when I get home.  You may also want to install something called samba-swat which is a configuration helper.  I can tell you that the main config file for this is called smb.conf and is located in /etc/samba if I recall correctly.  Have another go at things and ask some more specific questions here if you can...
Dan
