Title: ATI/Nvidia Binary Drivers with Modular X.org 7.0
Post by: rugmonster on April 19, 2006, 06:11:25 pm
I was very surprised this last weekend following an update of my Debian unstable box to find X.org 7.0 get installed. After a reboot, I was no longer able to get into X. At the time, I was using the ATI binary drivers (fglrx) and little did I know that with Xorg 7.0, comes a completely new filesystem location. Rather than living under /usr/X11R6, it now resides at /usr/lib/xorg. Unfortunately, ATI and Nvidia's installer installs things under /usr/X11R6 still.

To remedy this, I made sure /usr/X11R6 was empty and did an install of the ATI binary drivers. Then, I copied all of the contents of /usr/X11R6 to /usr/lib/xorg and I was back up and running. This same technique will work with the Nvidia drivers, as I have just confirmed this with my new card.

At this time, I know Fedora Core 5, Debian Unstable and Ubuntu Dapper Drake are running X.org 7.0, so heads up if you are using one of those and want to use the latest and greatest drivers from your card's manufacturer.

Title: Re: ATI/Nvidia Binary Drivers with Modular X.org 7.0
Post by: rugmonster on April 20, 2006, 04:39:09 pm
With the modular X.org 7.0, /usr/X11R6 should be gone completely, however, you could use symbolic links like this:

/usr/X11R6/bin -> /usr/bin
/usr/X11R6/lib -> /usr/lib/xorg

ATI also installs some headers into /usr/X11R6/include, and guess you could send those to /usr/lib/include, but I don't know for sure.
