Title: Using Dialog
Post by: bhoff on November 26, 2003, 10:46:38 pm
Using The Dialog Command In Linux.
You may asking why do I need the Dialog function if I have a GUI Desktop.
 Well what if it crashes or what if you are like me and prefer to work from the command prompt.
 I am hoping that the following examples will get you interested in this seldom used function and finding uses on your own to use it.

First you must have Dialog installed on your system.
You can find out by running
# rpm -q dialog
This should tell you the version number of Dialog if it is installed on your system if not you will need to install it to continue.

I will attempt to show you a few examples to get you right into coding with it to show you how easy it is.
At the prompt type
# dialog --fselect / 12 20
This will use dialog to create a box 12 high 20 wide with the window showing the contents of / .
You can adjust the path line to see other directories.

For our next example I was looking for a calendar when I came across this next app for planning backup's for the year.
At the prompt type
# dialog --calendar "2003 calendar" 15 20 26 11 2003
You now have a handy calendar for checking dates with days.
Use the tab key to move to the month or year to move up and down thru the calendar.
 " " is the text on the top of the calendar, 15=height 21=width 26=day 11=month 2003=year
feel free to change these as your heart desires.

You know I have showed you some examples but you know if you are like me you don't want to have to type
 this in every time you want to use it so how about we make a nice little script to run it.

# vi example1
use i to insert the following
# #!/bin/bash
# dialog --fselect / 12 20
#
Then save with
# :wq

Now for example 2
# vi example2
Again use i to insert the following
# #!/bin/bash
# dialog --calendar "2003 calendar" 15 20 26 11 2003
#
Then save with
# :wq
there you have it our first two examples saved as scripts in your directory.
be sure to make them executable before trying to run them by.
# chmod 700 example1
# chmod 700 example2
You now have 2 executable scripts. What you did with the chmod is make the scripts readable,writable
and executable by the owner being you.

Let's now add a little to the scripts with some input.
First
# vi example1
now change the script like so.
# #!/bin/bash
# echo -n " Enter directory path to view "
# read -e path

# dialog --fselect $path 12 20

Now save with
# :wq
you have now added the ability for the script to become pretty useful by having you ask for the path ahead of time to view.
Go ahead and run it and see what I mean.
What you have done is echo to the monitor the question, read you response and put this in the variable path
which we then use as a variable $path in the dialog function.

I hope this has been educational and got you thinking of some interesting things to do with it.
More information can be found in the man pages.
# man dialog

Ideas password tool for admin, User manger, script launcher just for a few. I use it to mange my Cluster with great success.
