1411  Distributions / RPM Based Distributions / RedHat 8.0 - First Impressions on: October 07, 2002, 10:23:47 am
Well, I downloaded RedHat 8.0 (psyche) last night and got it installed this morning.  It's 5 cd's (although my default workstation install only used the first two).

I must say, RedHat has taken GNOME to the next level.  This is nice.  Gone are the days of 5 different printer configuration programs and cumbersome menus.  This bad-boy is streamlined.  And works great!

Installation was a breeze, all hardware was detected perfectly.  I chose to do a default workstation install (which gives you Gnome, Evolution, Open Office, Mozilla, games, development tools, configuration utilities).

I ran into a problem when I first tried to login to my system, it would display the Gnome logo and lockup.  I'm not sure what the problem was, but I went to the console, installed my latest NVidia drivers and it still locked up.  So this time before I hit "ENTER" on my password I manually selected "American English" and "Gnome" from the options, then logged in, and it worked fine.  

So far I'm very impressed with this system.  The custom fonts provided by redhat give gnome a more pollished and professional look.  Much better (IMHO) than KDE on Mandrake.  Also Mozilla is working beautifully, no crashes, no mis-rendered pages.

I wanted to grab a screenshot to show everyone but GIMP is having trouble doing a screen-cap, so I'm gonna install another utility.  I'll post it here once I have it.

I've gotta say, RedHat 8 is definitely my new distro of choice.  Give it a try, I'm sure you'll like it.

1413  Distributions / RPM Based Distributions / Mounting NTFS on: October 05, 2002, 02:21:37 pm
I've successfully used NTFS mounts on my system without ever experiencing data loss or other problems.  However to avoid any problems (since NTFS support is still considered beta I believe), Mandrake doesn't support the kernel module by default.  You'll have to install the developer tools, kernel source, then compile a custom kernel to support it.

Alternatively you may be able to just compile the ntfs module, not sure.  If you want to try, it's something like:
cd /usr/src/linux
make menuconfig
Enable ntfs filesystem support, save and exit program.
make dep
make modules
make modules_install
depmod -a
modprobe ntfs (or maybe fs_ntfs, not sure exactly what the module name is)

Then you should theoretically be able to mount the partition.  I haven't tried this on mandrake, but if you like I'll take the time to compile an ntfs module and make it available for everyone here.  Just let me know.

I've run every distro I could get my hands on. In fact, I'm known for formatting my computer several times a week  

Right now I'm testing out SuSE 8.0 and I really like it. But generally Mandrake (at least up to 8.2) was my distro of choice.

The problems I've had with my latest computer though have given me reason to stop using Mandrake (and RedHat). First thing is the kernel they include gives me major problems with my sound card. I can compile a custom kernel to fix that, but also the compiler that comes with recent versions of RH and MDK give me problems. (XMMS, mplayer are almost unusable for me).

I've tried Slackware, it's okay. I've run ever version of RH since 5, and I like it. Mandrake just has the nicest installer (and most robust set of desktop apps IMHO). I must say though, I made the trip into FreeBSD territory recently. I downloaded and installed FBSD 4.6.2 and it is very nice. It uses a lot less resources than recent versions of linux, and seems to support my hardware better out of the box.. But it has it's problems too, mainly VMWare and Kylix don't run on it (two programs I like to play around with). If not for that, I would still be running it.

SuSE seems nice, but it's a pain to install. I couldn't find any iso images to download so I had to do a (2.6GB) install via FTP. Not too terrible over DSL, but it still took a few hours. SuSE seems to support my hardware perfectly, but I ran into a problem with it too. The custom kernel they use (it includes memory patches from the 2.5 series) causes VMWare not to work. So I had to compile a custom kernel and disable >4gb memory support. It actually took me about 4 tries to get the kernel compiled and still have all my hardware work, but it's up and running fine now.

Back on MDK though, I had some stability problems with KDE3 (especially Konqueror) that I'm not seeing in SuSE. I'm gonna take a guess and say it's the compiler that comes with MDK. Anyone else have problems with konqueror crashing frequently?
