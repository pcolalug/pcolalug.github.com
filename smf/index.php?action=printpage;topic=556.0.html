Title: Dual Booting Linux and Windows with LILO
Post by: imnes on March 16, 2003, 08:52:04 am
Here's a lilo.conf that I was using to dual boot Linux and Windows.  I don't remember who had the lilo problem, but maybe this will help you.

Code:


# LILO configuration file

# Start LILO global section
boot = /dev/hda
message = /boot/boot_message.txt
prompt
timeout = 60

# Timeout is seconds =timeout / 10,
# so a timeout=60 will wait for 6 seconds.

# Override dangerous defaults that rewrite the partition table:
change-rules
  reset

# Normal VGA console
vga = normal


# Linux bootable partition config begins
image = /boot/vmlinuz
  root = /dev/hda1
  label = Linux
  read-only
# Linux bootable partition config ends


# Windows bootable partition config begins
other = /dev/hda2
label = Windows XP
table = /dev/hda
# Windows bootable partition config ends




Should just have to copy that to /etc/lilo.conf, modify it to point to the correct partitions on your machine, and type 'lilo' at the prompt.  No guarantees though -- good luck.  If you have trouble ask questions here.

Title: Note on LILO!

Just remember anytime you make any changes to lilio you need to run /sbin/lilo this resets all the behind scenes stuff. You should also rerun /sbin/lilo if you make any kernel changes.
