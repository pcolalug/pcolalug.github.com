Debian has their packages structured like this....

Stable - The current main distribution. These are stable as stable come and are only updated beyond the release for security purposes. These packages are somewhat dated, but great for servers.

Testing - These are packages that are newer than those in Stable, but can be somewhat dated. These are what are being considered for the next release and are considered stable enough to be tested by the large group of Debian users who are comfortable using this set of packages for their desktops.

SID (Unstable) - Still in development... this is about as bleeding edge as Debian will get. These are the builds of the latest releases the package maintainers throw out there to the masses to test. Once they are found to be mostly stable, they go into Testing. SID can be quite hairy though.

Yeah, the dev or devel means it is a package for development. It is to seperate it from the normal package that most users need. For instance, most people want trolltech's QT library so that KDE can run or whatever other QT apps you have can use it. But the normal QT library doesn't include all the things you would need if you are using QT to develop an application. So those who want to use it for development install the extra stuff they need(the devel package) and everyone else doesn't have to worry about extras they won't use installed on their system.
