Pardon me while I pick apart your post, lloydr.

Quote from: lloydr
I find it kinda odd how someone in a linux user group finds windoze/winblows/winbloze offensive.....


Just because I'm in a linux user group doesn't mean that I have to hate Microsoft or their products.  Do I like their business practices, no.  That matters nothing to their products which are the topic of debate and how they stack up against their open source work-a-likes.  You miss my point entirely, in fact.  Here, I'll state it for you again.

Quote from: myself
It only scares off people who might think of trying Linux out for the first time.


This is a Linux User's Group, not a bash Microsoft Group.  As much as people would like to turn this into a Linux/*BSD/Unix vs. Microsoft world, it is really irrelevant.  People are going to use what they use and it is up to the developers to make the superior products.  

As I have stated already, I don't find it practical to use linux in a desktop environment.  The window managers are for all intensive purposed copies of three environments; Windows 9x, MacOS or NextSTEP.  The thing is, they don't do a good job of copying them and they don't add any compelling reason to use them.  More times than not, I spend more time getting a linux desktop setup than I do using it for whatever task I set out to do.  

It's one thing to play with this stuff at home, when I have the time for it.   In a work environment with missions having to be accomplished, I will take what is going to cause me the least amount of work, be it in the beginning or the long haul.  Those are the decisions you have to make when you are engineering a solution.

Quote from: lloydr
you've never been in the area of F*ckmicrosoft.com have you? its called microsuck now but i've been a member of the forums and had an email account with them for about 2 years..... check it out sometime.....theres a lot of nicknames for M$ and their products....


No, I haven't been there.  I'm sure it has some funny stuff.  Maybe I will check it out sometime.

Quote from: lloydr
also, for those who cant figure out any of the linux desktop systems UI, theres solutions for those certain cases called XPde.... http://www.xpde.com/


I have no reason to want to use the XP interface on a linux box.  I want to see some innovation come from the open source world.  At one time, it seemed like this was happening (I've been using linux for the better part of 5 years now), but has recently slowed down.  It seems there are so many people working on copy work and less on something new.  OSS is the perfect place to experiment with new ideas because there isn't the constraint of how the profit margins will look next quarter because it's made on volunteer time.

I'm lazy and I hold a motto of work smarter, not harder.  A lot of these copies add a consider amount of work to something that is easy in the original.  Once again, time and effort and when do you want the solution to work and how stable will it be in the long run.

Maybe I'm the only person who sees this, but I compile a lot of my programs under linux rather than using packages.  I see a lot of packages break on compile and I spend a lot time researching the errors and making changes to code to get it to COMPILE.  Regardless of using source or binary packages, the end products usually aren't as stable.  I'm not talking about the operating system.  Linux itself is ultra stable, hence my reason for using it for servers, but the OSS running on that operating system by and far is less stable than the stuff you usually find for Windows or Mac or commercial Unix.  The only exception to this is Explorer in Windows.  Explorer, the shell, crashes all the time, but it's not like when your Window Manager crashes and you have to kill your X session to start it back up.  Woops, there just went your work.

So, lloydr, you have yet to tell me why it is practical for you to always use linux and it's assorted OSS solutions over Windows, as you are obviously asserting.  If you use it out of principle, then say so, because it sounds like you are using OSS for the simple fact of using OSS, and blinding yourself from the other solutions out there that could possibly be better.

EDIT-

After thinking about this post, I feel I may be giving the wrong impression of my views on Linux.  I use it because I enjoy using it.  This debate I started because I wanted to get an understanding of why everyone else uses it.

At home, I use it for practicality and for principle.  Most of the server software I use at home would cost tons of money on a Windows environment and probably wouldn't be as complete a solution.  For what I do with my linux boxes, they are suited very nicely.  Also, those two linux boxes are old platforms and probably wouldn't run Windows very well.  On that note, however, I don't run X on them.  I stopped running a Linux desktop when I found the Windows 2000 actually ran better on my P3 550, which was at the time my fastest computer.  This made it a practicality issue, not to mention having to kill my X session at least once every day or two due to memory leaks in Gnome and Firebird.  Those may have been fixed by now, but I had problems, so I used what worked.

I love unix and there isn't much I can complain about from a console aspect.  I enjoy it and there are a lot of things I do from the console, but as a desktop environment, it just doesn't cut it for me.  So to sum up why I'm here, I have a good time using unix-like operating systems and they programs that run on them and I am here to help others who want help and get help when I need it.  I also love the unix world because of the community that we have, which is something you just don't get in the Windows world.  So, there you go.

Just caught this thread and thought I'd throw in my two cents.

I use both, Linux and Windows. I don't prefer either one as they both have draw backs. I first got into Linux because I was bored with Windows and wanted to try "something new". I bought my first computer, back in 1985,  with the primary purpose of learning about computers. My purpose hasn't changed. I used MS/PC-DOS through the Windows 3.11 era. I used Windows exclusively until 1995 when I started playing with SuSE. I started getting into Linux because it was different and I wanted to learn about it.

Now I use Linux on one of my home machines primarily as my internet desktop, Office Apps, and Money Management. I use another Linux machine at home for my router and firewall.

I use a Windows machine at home for most mutli-media application. (Eexcept I watch AVI and MPG files on Linux and use Gimp on Linux) I use my Windows computer with a program called showstopper. ShowStopper integrates my TV vieweing, TV Guide, TIVO like activity, and other multimidea activities into one, easy to use, easy to manage, easy to setup, interface. I also use my Windows system for capturing video from my VCR and Digital Camcorder, authoring it and making DVDs. Finally, I use my Windows system to play SimCity 4 (My current addiction)

There are some things I find easier to do and more polished in Windows. Other things I find easier to do and more polished in Linux. The bottom line is they are both running on the same hardware so should basically be able to accomplish the same things with the right application. The question comes down to which application do you prefer, the linux App or it's Windows counterpart? Think of it as choosing Gimp or another graphics program in Linux or a similar program in Windows. Which program do you like? Since all of this runs on the hardware, it's just a matter of choice.

Choice is good.... (Windows? Linux? Gimp? Photoshop?) It's all up to you.....
