Title: Moving Server from OpenBSD to Linux
Post by: rugmonster on October 03, 2006, 09:53:34 am
Good morning, all. I've got an OpenBSD file/mail server at home that I'm thinking about moving to Linux. Last night, I realized just how slow disk i/o on it was compared to my Linux desktop and the best I can come up with is the OS and the FFS filesystem. With a Athlon XP 2500+ and 512MB of memory, the thing should fly. When compared to my Linux box with the same model drive, I am getting about 1/4th the performance on disk I/O. I never got around to using OpenBSD for what I really wanted it for, so I think it's time to move back to Linux.

In doing the move, I want to make sure I keep all of my mail that is stored in my Cyrus IMAPD and the user information in the SASL database. What would be the best way to back this up so that I can transfer it in once I have the new system install up and running.

Finally, what distro do you guys think I should go with? I've run a bunch of them over the years and keep coming back to Debian, but I want to get as much bang for my buck as possible. Since it is an Athlon XP system, I would like to get away from i386 binaries, but don't want to go through the headache of Gentoo, though it's not completely out of the question. I'm not a huge fan of Fedora or any of the other RPM based distros, but once again, still a possibility. I'm mainly looking for something that will be fast, relatively easy to manage, and not be crippled without having X installed.

Actually, Ubuntu and most Debian derivatives are still i386. Fedora, SUSE and Mandriva are either i586 or i686 binaries. Slackware and its derivatives are the only ones I know of that are using i486. From what I hear, i486 actually runs slower than i386 optimized binaries.

Anyhow, I've decided to go with Ubuntu all around. It works well on my laptop, has newer packages and has a consistent release path. The server install of it is very minimal, but still provides the Debian style default config files that require very little tweaking. Additionally, apt is my favorite package manager and I just can't see myself using something else.

Title: Re: Moving Server from OpenBSD to Linux
Post by: rugmonster on October 07, 2006, 05:44:11 pm
I did it! I moved the server to Ubuntu and ended up setting up LVM for the drives. I used the alternate install disc and things went swimmingly. I gained 40GB of space for my file storage that would have otherwise been wasted on my homedir or something from the second hard drive in the system in a logical volume with the 200GB drive I had used for file storage. The system feels snappier. Also, I had very little to custom configure. I still need to get Postfix and Cyrus going, but that isn't a huge priority. Also, the base system was very small, but software installation was a breeze thanks to apt. One note, not even SSHD is installed with the initial base install. There are no listening services by default.

I also went ahead and reloaded my desktop. I'm running Ubuntu 6.06.1 AMD64 and loving it. When I first heard about the way 32bit binaries are handled in a chroot environment, I was a little turned off. Thankfully, Debian and Ubuntu provide utilities to set the environment up for you and a seperate apt environment making the use of 32bit apps mostly pain free. I can say that I am very pleased with the setup and in a couple of hours, have a fully functional system with a 32bit Firefox and plugins as well as Mplayer with w32codecs. I'm very happy with it and look forward to the official release of Ubuntu 6.10 in the next few weeks.
