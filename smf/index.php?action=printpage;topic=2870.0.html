Title: FreeBSD Aims for Desktop
Post by: rugmonster on May 13, 2006, 09:29:53 am
An article on ZDNet (http://news.com.com/FreeBSD+vows+to+compete+with+desktop+Linux/2100-1011_3-6071598.html?tag=nefd.top) with Scott Long of FreeBSD about their goals of reaching equal footing with Linux on the Desktop. In it, he talk *cough cough* complains *cough cough* of Gnome's gearing toward Linux development, as well as their goal to add a HAL to FreeBSD that supports hotplug devices.

What do you think; should FreeBSD really be aiming at the Desktop, or should their focus remain the server and let the folks at PC BSD and that other one worry about the desktop? To me, FreeBSD is sort of a middle ground between Linux and OpenBSD, which seem to be on opposite sides of the spectrum of desktop usability and server/security focus. That's not to say that Linux doesn't make a great server, but OpenBSD certainly provides a secure, stable server platform while sacrificing on desktop usability.

I might be wrong, but until I really get to know a distro, I judge it based upon my initial impressions on it.  My desktop experiences with fbsd weren't exactly smooth sailing.  Also, I really can't take it seriously until it gets current, timely java releases.  If you do a broad range of software development these days, you had better have a current java SDK. 

On the server side, I'll admit fbsd does it's job fairly well.  Then again, so does debian stable.  I wouldn't recommend either for desktop use, but if you wanted a rock solid server setup, It wouldn't be a bad way to go.  Some might recommend slackware.  That's fine, if they're willing to do the extra work to maintain it.  For me though, if I was a server admin, debian would be the way to go, as it would be the least work since they only update it once every 3 years or so.

Of course obsd has it's place, but I'm not sure how well it would handle special hosting needs...

Title: Re: FreeBSD Aims for Desktop
Post by: digger on May 17, 2006, 09:48:27 pm
I have installed FreeBSD desktops based on versions  4.8, 4.9  and 4.10.  As a desktop it worked OK, although my recollection of it reminds me of Debian Stable for some reason, older packages, perhaps.  Then I had nothing but problems trying to get the 5.x series to run, as I could never get X11 to work properly (I was missing an important post-install step: no .xinitrc file).  Finally got 6.0 installed with X11 & a KDE desktop, and sound to boot.  It still reminds me of Debian or even Slackware but with a dependency-aware (sometimes) package management.  FreeBSD tends to boot very quickly and it shuts down in a heartbeat,  It also  it seems to be very powerful, although that is just one of those gut impressions, that I couldn't back with an objective measurement if my life depended on it.

Actually, PC-BSD really is FreeBSD with a slicker installer, which leaves one with X11 and sound fully configured & other little niceties.  Package installation is very easy with the .pbi system (as long as they work) and one can still use the traditional FreeBSD pkg_add and ports system.  Be that as it may, for some reason, maybe because it was the first non-graphical install of any *nix I did that actually worked (or masochism?), I'm really fond of the Old Red Mare, & keep diddling with it.   I've downloaded install disks for FreeBSD 6.1 & will probably dance with it some more later this month.
