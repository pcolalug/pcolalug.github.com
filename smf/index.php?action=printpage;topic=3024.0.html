Title: Re: Hello to the group
Post by: lloydr on October 09, 2006, 07:21:31 pm
Quote from: mobilemail on October 09, 2006, 06:48:18 pm
lloyd,
verizon xv6600  1xrtt  (I could do tethered with cable or bluetooth), for data access.

hmm from what I hear getting it working with bluetooth is easy as long as you have bluetooth pairing working. I don't have a bluetooth adapter so just used the usb cable

here is my post on sprintusers forum that shows what I did to get it working using usb cable:
Quote
current kernel version I'm running is 2.6.15.7 due to crappy ati drivers in the case of my laptop. But, built custom kernel for laptop and made sure to include USB ACM modem support (CONFIG_USB_ACM). with this and using the usb data cable that came with my A920, the phone shows up as /dev/ttyACM0 under dmesg.

then setup the PPP by creating these 2 files:

/etc/ppp/sprint.modem:
TIMEOUT 5
ABORT '\nBUSY\r'
ABORT '\nERROR\r'
ABORT '\nNO ANSWER\r'
ABORT '\nNO CARRIER\r'
ABORT '\nNO DIALTONE\r'
ABORT '\nRINGING\r\n\rRINGING\r'
'' \rAT
TIMEOUT 12
OK ATD#777
TIMEOUT 22
CONNECT ""

/etc/ppp/peers/sprint:
noauth
connect '/usr/sbin/chat -v -f /etc/ppp/sprint.modem'
defaultroute
usepeerdns
/dev/ttyACM0
921600
local
novj

Then, I simply ran the command "pppd call sprint" and watched my /var/log/messages for the modem status..... and at first, all was well, until after 2.5 minutes later, it'd error out and disconnect with the message of no response to 4 echo-requests.....

soooo, to fix that, I added these 2 lines to my /etc/ppp/peers/sprint:
lcp-echo-failure 4
lcp-echo-interval 65535

and everything works flawlessly now. after fixing the echo-request issue, I stayed connected for around 30minutes before I disconnected the PPP session myself since I was done.

this works for both 1xRTT and EVDO in the case of my samsung A920. in your case I'm not sure if you will use the same #777 for data call out to use the phone as a modem. and in the case of my phone, I had to disable modem NAI since it's an EVDO phone, to keep it from showing that I'm using it as a modem and not web useage just from phone (don't have tether web plan heh)
