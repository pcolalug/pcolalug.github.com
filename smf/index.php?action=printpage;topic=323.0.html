Title: Telnet / FTP slow to respond
Post by: rugmonster on December 20, 2002, 02:38:45 pm
It isn't so much a matter of his dns server, it is the telnet and ftp services wanting to do a reverse dns lookup on his ip for logging. I'm sure there is a way for configure the daemons not to do a rDNS lookup, but adding the hosts to the /etc/hosts is the easiest solution.

I just happened to be looking for DNS server daemons earlier and ran across  this (http://www.thismetalsky.org/projects/dhcp_dns.html). It is a perl script for updating your dns server from info from DHCPD's lease log. I'm sure there are other solutions out there similar to this. The only thing I don't like is in order to use this particular utility, you have to run a BIND alternative called DJBDNS (http://cr.yp.to/djbdns.html). I haven't looked into packages or anything, so in order to install the dns server, you may have to compile from source. It seems to be very well documented, though. You may also be able to find a solution that works with BIND if you look hard enough, though.
