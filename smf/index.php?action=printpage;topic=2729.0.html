Title: Re: Command Line "cheat sheet"
Post by: rugmonster on December 12, 2005, 12:09:43 pm
First thing I noticed in that second "cheat sheet" is the switching between vty's. Unless you are in X, you don't have to use ctrl with alt + Fnum. If you are in X and you want to break out into the console, then you use ctrl+alt+Fnum.

Also, arrowing through the command history is only applicable with certain shells such as BASH and ZSH. CSH, KSH and good ol' Bourne Shell don't support command history or tab file name completion.

Shift+PgUp/PgDn doesn't always work depending on your terminal. This is especially true with a framebuffer console and when using GNU Screen. This also doesn't work on a lot of dumb terminals and traditional UNIX machines (Sun hardware for example).

Ctrl+Alt+Del may have no effect on your system. It is all in how the system is setup to interpret the combination. My debian system doesn't do a dang thing when you give it a Ctrl+Alt+Del...and that's how I like it. I like having to type sudo /sbin/shutdown -h now...and yes include the whole path because I don't use the sbin dirs in my PATH.

And who the heck uses pico and rsh these days?! No mention of VI or VIM?!

I guess you can't cover all of the caveats.

One more glaring error, not every configuration file uses # for comments. Also, a comment means that it is ignored by the program so you can type whatever you want there without causing any problems.
