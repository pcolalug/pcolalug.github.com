Thanks for the helpful link Steve.

I read all of that and ended up setting my fstab file thusly:
/dev/hda1 /mnt/disk ntfs umask=0,quiet 0 0

I added that to the end of the file. It had an error on boot saying that it was not defined as the proper type, or that there were too many mounts defined (both within the same error).

So I changed it to this:
/dev/hda1 /mnt/disk -t ntfs umask=0,quiet 0 0

This booted fine, but I still cannot browse that mount point. When I go into console and type mount, it does not show that I am mounted to that device. I can manually mount it from the console and see everything fine. After manually mounting, I can also see the drive as root inside the file manager, but not as a user.

I've successfully used NTFS mounts on my system without ever experiencing data loss or other problems.  However to avoid any problems (since NTFS support is still considered beta I believe), Mandrake doesn't support the kernel module by default.  You'll have to install the developer tools, kernel source, then compile a custom kernel to support it.

Alternatively you may be able to just compile the ntfs module, not sure.  If you want to try, it's something like:
cd /usr/src/linux
make menuconfig
Enable ntfs filesystem support, save and exit program.
make dep
make modules
make modules_install
depmod -a
modprobe ntfs (or maybe fs_ntfs, not sure exactly what the module name is)

Then you should theoretically be able to mount the partition.  I haven't tried this on mandrake, but if you like I'll take the time to compile an ntfs module and make it available for everyone here.  Just let me know.
