If you guys would like this server to be visible to the public, try this:  
1.  You'll need to open ports 27900 and 27950 (both udp) on the firewall.
2.  Add the following to your "public.cfg" (the first line may already be there):
Code:
seta sv_master1 "wolfmaster.idsoftware.com"
seta sv_master2 "master.gamespy.com:27900"
seta sv_master3 "master0.gamespy.com"

These are the master server "browsers" that rebroadcast the available servers to the world.

Should work, and I'll be checking into the server to see if anyone's looking to play.  Watch out for "Gamgee"...

You know, after more thought, I remember reading a lot (and experiencing firsthand) about problems with running a game server behind a hardware router.  The NAT translation / masquerading is the issue.  I'm assuming you're including something like "+set net_ip 68.99.60.104" on your command line (script) when you start the server?  I'm not sure what the answer is, or if there is even a solution at all.  Not sure how you have things set up there, but if it's possible to run the server outside the hardware router (either physically connected outside, or maybe the "DMZ zone" of the router), and just use a software firewall, that might work.  Maybe a little dangerous to the machine if you have anything else besides the game server running on it.  

In the long run, it's not a huge problem for us to connect to the IP address without the name showing up in the browsers.  Keeps the crowds down too.
