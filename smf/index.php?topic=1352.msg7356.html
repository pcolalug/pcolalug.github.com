« on: February 04, 2004, 02:00:09 pm »
After a very long month and a half, I finally have a very acceptable Personal Video Recorder (PVR) solution up and running.

It is called MythTV. If you are familiar with TiVo, then you will understand many of the features of MythTV.

Here are the abilities that Myth has that I am using:
Watching live TV with the ability to pause (I have it set to keep a 1 hour buffer)
Record/Playback TV shows
Commercial Skip
Basic editing of shows to make the file sizes smaller
Watch (pause/FF/RW/stop watching) a show that is currently being recorded
MP3/OGG ripper/player
Photo Gallery viewer
Weather/News feeds
TV listings through mySQL and XMLTV
Record modes:
-Only this showing
-This show in this timeslot each week
-This show on this channel in any timeslot
-This show in any timeslot on any channel
mySQL keeps track of your shows so that it does not record a second time an episode that you already have recorded
DVD ripper/player

Features that are available (and many more) that I am not currently using:
mame/xmame/snes/etc... frontend for a game box
Visualizations during music playback
Web interface so that you can schedule recordings from another computer on the network, or you can VPN from outside to schedule recordings
If you have more than one tuner card you can have PiP or record on two channels at one time, or watch TV while recording another channel
Frontend on other computers in the network to be able to do video playback at any workstation/TV from the main recorder/database

There are many other things to say about it, but that is a basic list. I am very excited about the project. There was a new release this weekend. It is up to 0.14. I started with 0.13 and had some minor performance issues, but these seem to be fixed in 0.14.

I do still have one problem that I am working on. I have not been able to get my TV out card to output properly. I am running an STB Velocity 4400 (GeForce 2? based card). It has TV out through RCA and through SVideo. Both put out about the same quality signal (which is nothing veiwable). So currently, we are watching TV on a 13" CRT sitting next to our 19" TV.  

If you do decide to jump into this, I really recommend thier mailing list archives for great information. Be warned if you sign up for thier list though. It has generated between 150 and 180 messages per day since I have been signed up! I set up a separate email account for this and am glad I did.

The TV listings are updated daily via a cron job and a connection to the net. This can be done statically and less often if you do not have an always on connection. It grabs 9 days of programming at a time. It will check the current days that you already have to see if there were any changes to the schedule as well as add the next day.

Several themes to choose from. I do not get much into theming but have checked out a few of them. I like the default.

Fairly detailed documentation on the web. There are a few prepackaged distros that you can install with MythTV included. Even one for your Xbox! There is a Knoppix based version. Several binaries are available which can be found at the site and on the mailing list.

Once getting Gentoo set up (what a pain!) it is a good distro for running Myth on. I DO NOT offer any Gentoo help. Anything specific about MythTV and Gentoo I might be able to help you with. Gentoo came upon high recommendation for this project (no rburt, you had nothing to do with getting me to run this). I think it is a great distro for this. I even hate admitting that I almost like it.

My setup:
AMD running at 1.4
KWorld Tuner card (BTTV 878)
STB Velocity 4400 video out card
On board sound
Gentoo 1.4
Kernel 2.4.22ish

To answer an earlier question, which you have obviously found an answer to:
Signup for mailing lists
Archive to mailing lists

XMLTV listing is AWSOME! Through the setup in Myth, you tell it your Zip code. It then will generate a list of all the cable providers and packages available for your Zip. You then select the provider and package that you subscribe to. That simple. Then you run a command (mythfilldatabase) and it will populate your mySQL database with all the listings in your package for the next 9 days. You can set a cron job to check for the new listings each day. It will also check to see if the current listing that you have is still acurate by checking to see if there have been any updates to the listings since you last updated.
